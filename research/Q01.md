# Q01 — Tier 3 Skill Catalog Validation

**Priority**: P0
**Status**: Open
**Owner**: giodl
**Resolves**: simulate plugin skill catalog (Tier 3 domain simulation skills)
**Blocks**: plugin-plan.md

## Question

Do the 10 proposed Tier 3 simulation skills cover what BIC developers across Dynamics 365 and Power Platform actually need? What's missing? What's redundant? What would they reach for first?

## Hypothesis

The 10 Tier 3 skills (request-trace, state-validation, contract-testing, deployment-trace, plugin-chain, permission-walkthrough, data-flow, component-trace, migration-testing, integration-trace) were derived from generalizing our internal compiler simulation patterns. We expect gaps in:
- Dynamics-specific workflows (BPF, business rules, SLA timers)
- Power Platform maker scenarios (canvas apps, model-driven apps, Power Automate flows)
- Copilot Studio / AI agent scenarios
- Multi-environment ALM patterns

We also expect some skills may overlap or be too granular for the target audience.

## Proposed Tier 3 Catalog (Under Test)

| # | Skill | What You Simulate |
|---|-------|------------------|
| 1 | `/simulate:request-trace` | Trace a request through service boundaries, APIs, middleware |
| 2 | `/simulate:state-validation` | Starting state → operations → expected outcome |
| 3 | `/simulate:contract-testing` | Expected vs actual output comparison |
| 4 | `/simulate:deployment-trace` | Trace what happens when you deploy, import, or migrate |
| 5 | `/simulate:plugin-chain` | Trace record operation through plugins, business rules, workflows |
| 6 | `/simulate:permission-walkthrough` | Trace who can do what through RBAC, security roles, teams |
| 7 | `/simulate:data-flow` | Trace data through ETL, pipelines, transformations |
| 8 | `/simulate:component-trace` | Trace user action through UI component tree, state, renders |
| 9 | `/simulate:migration-testing` | Expected vs actual after schema change, version upgrade |
| 10 | `/simulate:integration-trace` | Trace cross-system interaction through connectors, MCP, APIs |

## Experiments

- a: User testing with backend developers (Dataverse, Sales, Commerce, Customer Service, Finance, Operations) — present the 10 skills, ask which they'd use, what's missing, what's confusing → expected: gap list + priority ranking | method: design analysis
- b: User testing with frontend developers (Sales, Commerce, PowerApps, Copilot Studio) — same protocol → expected: gap list + priority ranking | method: design analysis
- c: User testing with developer subroles (Automate, Connectors, Copilot Studio) — workflow/integration-heavy roles → expected: gap list, especially around flow simulation | method: design analysis
- d: Cross-role synthesis — merge findings from a-c, identify consensus gaps and redundancies → expected: revised catalog | method: desk research

## Test Protocol

For each persona:
1. Present the 10-skill catalog with one-sentence descriptions
2. Ask: "Which 3 would you use first and why?"
3. Ask: "What simulation would you want that isn't here?"
4. Ask: "Which of these overlap or confuse you?"
5. Score each skill 1-5 on usefulness for their daily work
6. Record verbatim responses
