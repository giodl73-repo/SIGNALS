## V-02 — Severity-Tiered Ledger Violations

**Axis:** Severity-tiered ledger violations — Table 1B adds a Severity column classifying
each breach as BREACH-MINOR, BREACH-MAJOR, or BREACH-FATAL. Per-tier correction protocols
are declared in Phase 1 so the correction response is deterministic throughout.

**Hypothesis:** Generic ledger-violation framing (C-36) treats all violations with the same
urgency. Tiered severity creates proportionate correction pressure: fatal breaches halt and
restart; major breaches require exit-gate re-verification; minor breaches correct inline.
This prevents over-correction (restarting for a missing pitch) and under-correction (continuing
past a missing Ph6 Pre-Score column).
