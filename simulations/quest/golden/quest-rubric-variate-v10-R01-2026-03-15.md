# quest-rubric Variate — Round 1 against rubric v10
**Date:** 2026-03-15
**Rubric:** v10 (C-01--C-28; adds C-26: authoring instruction IS the contrast model rather than augmented with one; C-27: causal grounding defined constitutively by the neighboring-value test rather than asserted and verified; C-28: each blocking gate's condition is the existence of a named evidence record retained per criterion)
**Target criteria:** C-26, C-27, C-28

**Round 1 against v10 design note:** Rounds 1--15 in the global variate sequence produced mechanisms for C-01 through C-25. Round 9 (against rubric v9) is the most directly relevant predecessor: V-02 R9 introduced self-demonstrating instructions (GENERIC:/SKILL-SPECIFIC: blocks inline in the text instruction, satisfying C-24; constitutive biconditional for the causal grounding definition, satisfying C-25); V-03 R9 introduced phase-linked evidence records (Contrast Contract + T+1/T-1 records as blocking gate exit conditions, partially satisfying C-28's structural intent).

The three new criteria draw a sharper distinction at each level:

**C-26 vs C-24.** C-24 is satisfied when the text instruction *contains* an inline contrast example alongside generic organizing prose. V-01 R9 passes C-24 by appending labeled Generic/Skill-specific blocks to a generic-prose instruction. C-26 is not satisfied: the instruction's *organizing format* must BE the contrast pair -- the contrast structure must be the primary form, not an appended demonstration. The operative test: if the label words are removed, does the instruction still read as a two-part opposing structure? An augmented instruction does not survive this test; a structured instruction does.

**C-27 vs C-25.** C-25 is satisfied when the pass condition schema includes a neighboring-value falsification procedure applied after or alongside a causal-grounding assertion. V-01 R9 passes C-25 with a step-by-step T+1/T-1 procedure the author runs to verify a claimed threshold. C-27 requires the biconditional constitutive form: "A threshold T is causally grounded if and only if T-1 no longer catches the same failure mode." With this form, claiming causal grounding and passing the test are the same predicate -- there is no prior assertion to verify separately.

**C-28 vs C-13.** C-13 is satisfied by any blocking gate ("DO NOT proceed until [boolean condition]"). C-28 requires the gate condition to be the *existence* of a named, structured, retained evidence record -- not a property being true. A boolean gate verifies and discards; a record-linked gate produces an artifact that proves the gate fired. The distinguishing test: after authoring completes, can an evaluator find the evidence that each gate fired?

This round probes three single-axis mechanisms (V-01: inertia framing, V-02: output format, V-03: phrasing register) and two combined variations (V-04: role sequence + lifecycle emphasis, V-05: inertia framing + output format).

## V-01 — Inertia Framing (Three Near-C-26/C-27/C-28 Competitors)

**Axis:** Inertia framing
**Hypothesis:** Prior rounds named near-excellent competitors scoped to the criterion that most commonly fails. V-01 names three new competitors, each scoped to one Round v10 target criterion. The Augmented-Instruction Protocol passes C-24 by appending a contrast example but fails C-26 because its organizing form is generic prose. The Procedural-Grounding Protocol passes C-25 by running T+1/T-1 but fails C-27 because the causal assertion precedes the test as a separate step. The Boolean-Gate Protocol passes C-13 by including blocking gates but fails C-28 because gates fire on boolean properties that leave no auditable artifact. Each competitor is placed at its relevant authoring moment. Hypothesis: competitor framing naming exactly the R9 mechanism that passes the prior criterion but fails the new one produces reliable compliance because the gap is identified at the precise moment the builder is most likely to replicate it.

---

You are building a scoring rubric for a Signal skill. The rubric is the objective function for quest-golden. Your protocol has three competitors. Each passes C-01 through C-25. Each fails exactly one of the three Round v10 target criteria.

**Competitor A: The Augmented-Instruction Protocol**

Essential criterion texts follow the causal template: "Without [Y], [failure] occurs." The text instruction teaches this pattern and appends a contrast block -- a labeled Generic formulation followed by a labeled Skill-specific formulation. The contrast example is present and inline. But the instruction's organizing format is generic prose: "Write the criterion text using the causal template. Here is a contrast model: [Generic block] / [Skill-specific block]." The contrast is appended as supplementary documentation. An author reading quickly absorbs "use the causal template" and may treat the appended contrast as optional context. Your text instruction must not follow this pattern. The instruction's organizing format must BE a contrast pair -- GENERIC:/SKILL-SPECIFIC: blocks as the instruction's primary structure, not generic prose containing a contrast example. Removing the label words from your instruction should leave a recognizable two-part opposing structure.

**Competitor B: The Procedural-Grounding Protocol**

Pass conditions carry the causal-grounding clause ("the observable must be causally linked to the failure mode") and the neighboring-value test procedure (T+1/T-1, verdict boundary/measurement-choice). The procedure is applied after the author claims causal grounding. The conceptual sequence: claim causal grounding, then run the test to confirm. Both exist. But they are sequential operations: assertion first, verification second. An author can write "this threshold is causally grounded because..." and then run the procedure to confirm -- the claim precedes the test. Your pass condition schema must not follow this pattern. Define causal grounding constitutively: a threshold T is causally grounded if and only if T-1 no longer catches the same failure mode. With this form, claiming grounding and passing the test are the same predicate. The biconditional IS the definition; the test IS the claim.

**Competitor C: The Boolean-Gate Protocol**

The protocol includes hard blocking gates: "DO NOT finalize an essential criterion until its text matches the skill-specific pattern." "DO NOT finalize any criterion with a quantitative threshold until the neighboring-value test verdict is boundary." Each gate fires on a boolean property being satisfied. The property is evaluated and then discarded -- no artifact is produced. After authoring completes, an evaluator cannot determine whether any gate fired correctly: the condition was checked and the evidence trail is absent. Your protocol must not follow this pattern. At least one gate must fire on the *existence* of a named, structured evidence record retained per criterion: "DO NOT proceed until the [named record] for this criterion is complete." The record is the gate condition and the audit trail simultaneously.

## V-03 — Phrasing Register (Constitutive-Pair Structure Throughout All Instructions)

**Axis:** Phrasing register
**Hypothesis:** Round 9 V-02 applied self-demonstrating register selectively: the text instruction was a self-demonstrating contrast pair; the causal grounding definition was a constitutive biconditional. V-03 applies the constitutive-pair register uniformly to ALL definitional instructions in the protocol. Every property that must be defined -- what a skill-specific criterion IS, what causal grounding IS, what a gate IS, what aspirational territory IS -- is organized as an IS / IS NOT constitutive pair. This makes each instruction self-demonstrating by form: the instruction that defines skill-specificity IS a contrast between specific and generic; the instruction that defines causal grounding IS the biconditional; the instruction that defines a valid gate IS a record-existence condition. Hypothesis: when every definitional instruction is organized as a constitutive IS/IS NOT pair, the protocol propagates the required form to all three targets simultaneously without requiring the author to apply separate mechanisms for each -- the register does the work uniformly.

**FAILURE MODES**

A blocking failure mode IS:

```
BLOCKING FAILURE IS:
  An output property whose absence prevents the skill's downstream agent from completing
  its primary function. The function stops, not merely degrades.

BLOCKING FAILURE IS NOT:
  An output property whose absence reduces quality while leaving the function possible.
  Degraded function is not stopped function. That is a degrading failure.

DEGRADING FAILURE IS:
  An output property whose absence reduces output quality but leaves the downstream
  function possible at reduced effectiveness.

DEGRADING FAILURE IS NOT:
  A property whose absence prevents the function entirely. That is blocking.
```

**PASS CONDITIONS**

A causally grounded pass condition IS:

```
CAUSALLY GROUNDED IS:
  A pass condition whose threshold T is the exact failure boundary -- T-1 no longer
  catches the same failure mode. This biconditional IS the definition of causal
  grounding: a threshold is causally grounded if and only if T-1 fails to catch the
  failure. Claiming causal grounding and satisfying the biconditional are the same act.
  There is no prior assertion to make; running the test IS claiming causal grounding.

  Test procedure: apply T-1 to the failure mode. If T-1 catches it: T is a measurement
  choice; revise. If T-1 does not catch it: T is the failure boundary; accept.

CAUSALLY GROUNDED IS NOT:
  A threshold selected as a reasonable minimum or conventional count without testing
  whether T-1 still catches the failure. A stated causal claim with an unrun test is
  not causally grounded by this definition. A threshold is only causally grounded when
  T-1 is tested and found to fail.
```

A valid gate IS:

```
GATE IS:
  A condition of the form "DO NOT proceed to [next step] until the [named structured
  record] for this criterion is complete" -- the gate fires on the existence of the
  record, not on a property being true. The record is retained per criterion; after
  authoring, an evaluator can verify the gate fired by reading the retained record.
  The record's existence IS the proof of compliance.

GATE IS NOT:
  A condition of the form "DO NOT proceed until [property] is true" where the property
  is evaluated and discarded without producing an artifact. A boolean gate leaves no
  evidence after authoring; its correctness cannot be audited post-authoring.
```
