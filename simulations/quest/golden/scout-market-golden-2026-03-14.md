## What Made It Golden

**1. STATUS QUO as a blocking prerequisite with named outputs.**
The preamble declares STATUS QUO runs before scoring and names its two outputs -- do-nothing cost and inertia anchor -- as required inputs to three downstream sections (scoring cards, beachhead, sequencing). This forces the baseline before any fit rating, making inertia traceable from source to rank.

**2. Dual-view architecture: per-cards + cross-segment summary table.**
Per-segment cards deliver vertical depth (all dimensions for one segment in one block, revenue model co-located with WTP). The cross-segment summary table delivers horizontal comparison (all segments in one row). Neither view is subordinate. This is the only architecture that passes both C-13 (per-card co-location) and C-15 (unified cross-segment view) simultaneously -- and is the only path to C-19.

**3. C-17 quantification chain enforced at source and consumption.**
STATUS QUO "Do-nothing cost" includes a forward-reference that pre-commits to a quantified estimate (`Required in "Not building this means:" below`). The beachhead "Not building this means:" cites the STATUS QUO cost by format (`costing approximately $ or hrs from STATUS QUO do-nothing cost`). A reminder at consumption alone (C-12 standard) doesn't satisfy C-17; enforcement at the point of entry is what closes the chain.

**4. Explicit inertia-break per-block field forecloses time-based responses (C-16).**
The sequencing section uses a pre-printed key-value field -- `Inertia-break condition:` -- with an inline instruction that names a specific invalid response pattern (`not a calendar date or time period`). The label string is precise enough that "Year 2+" cannot be a valid fill. This is the field-level mechanism for C-16; the table column header mechanism (V-03/V-04) was confirmed equivalent but C-19 makes Path A the production architecture.

**5. Instruction economy: each constraint stated once, at enforcement point.**
Each structural rule appears where it acts, not as a preamble list to re-read. The STATUS QUO forward-reference sits at the do-nothing cost field. The inertia-break instruction sits inside the Y2+ block. The summary table constraint sits at the summary table header. No redundancy from development rounds survived into the golden candidate.
