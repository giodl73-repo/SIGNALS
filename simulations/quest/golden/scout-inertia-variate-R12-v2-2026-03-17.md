## R12 Design Notes

### C-38 mechanism -- five domain subtitle forms tested

C-38 requires every FAIL-FIRST heading to carry a structural role marker AND a domain-frame subtitle drawn from inertia vocabulary. R11 V-01 and V-02 failed C-38 because their headings read "FAIL-FIRST DECLARATION" with no subtitle. R11 V-03 passed with "-- THE UNNAMED COMPETITOR". R12 tests five subtitle forms across five variations:

1. **"NAMING THE UNNAMED COMPETITOR"** (V-01): Adversarial framing -- names the competitor as the analytical act
2. **"THE UNNAMED COMPETITOR'S FAILURE SURFACE"** (V-02): Vulnerability framing -- names what the analysis maps
3. **"THE WORKAROUND FAILURE INVENTORY"** (V-03): Artifact framing -- names the output of the FAIL-FIRST stage
4. **"DEFEAT CONDITION PREREQUISITES: WHERE THE WORKAROUND BREAKS"** (V-04): Causal framing -- positions failure modes as prerequisites for defeat conditions
5. **"NAMING THE DEFAULT COMPETITOR -- STRUCTURAL WEAKNESSES"** (V-05): Double-label framing -- competitor identity plus analytical focus

C-38 stress test: does any domain subtitle drawn from inertia vocabulary pass, or does the evaluator require specific tokens? All five subtitle families are structurally distinct. R12 scoring determines whether C-38 is content-agnostic (any inertia-domain phrase passes) or vocabulary-specific.

### C-39 mechanism -- five binary-question forms tested

C-39 requires the completeness gate heading to be Yes/No-answerable. R11 V-01 ("BRIDGE COMPLETION STATUS") and V-02 ("BRIDGE COMPLETION GATE") failed. R11 V-03 confirmed "-- READY TO PROCEED?" R12 tests five additional interrogative structures:

1. **"ALL BRIDGE ARTIFACTS POPULATED?"** (V-01): Artifact-inventory question -- asks whether all required artifacts exist
2. **"FULLY MAPPED?"** (V-02): Short-form domain question -- asks whether the mapping obligation is complete
3. **"ALL ARTIFACTS POPULATED?"** (V-03): Minimal artifact question -- same semantics as V-01, shorter form
4. **"HAVE BOTH BRIDGE ARTIFACTS BEEN POPULATED?"** (V-04): Explicit-count question -- names the exact count expected
5. **"PASS BEFORE ADVANCING -- ALL BRIDGES COMPLETE?"** (V-05): Imperative + question compound -- gate action named before the decision question

### C-37 mechanism -- body placement confirmed, three command label forms

C-37 requires `[BRACKET COMMAND]` in the gate block body, not embedded in the heading. R11 confirmed V-01 passing (command inside BRIDGE COMPLETION STATUS body). R12 tests three command label forms to determine whether C-37 is label-agnostic:

1. **`[BRIDGE COMPLETION COMMAND]`** (V-01, V-03): Canonical form from R11
2. **`[STATUS QUO BRIDGE COMMAND]`** (V-02): Domain-vocabulary label variant
3. **`[BRIDGE GATE COMMAND]`** (V-04): Abbreviated gate-reference label
4. **`[BRIDGE COMPLETION COMMAND]`** (V-05): Canonical form with COMMAND phrasing register

### C-34 and C-35 -- four label forms carried forward from R11

R11 confirmed that bracket-prefix, bracket-suffix, and domain-prefixed bracket-suffix all satisfy C-34/C-35. R12 carries two forms forward:

- **Bracket-prefix** (V-01, V-03, V-04, V-05): `[A-16 PRIMARY KEY RULE]`, `[A-19 CITATION INTEGRITY RULE]`
- **Bracket-suffix** (V-02): `STATUS QUO LOCK RULE [A-16]`, `STATUS QUO CITATION RULE [A-19]`

### C-40 -- confirmed retroactive from R11

R11 V-01 already carried C-01: through C-05: prefixes and confirmed C-40 pass. All R12 variations carry the prefixed checklist format. The stress test is whether variations with domain-vocabulary section names (V-02, V-03) still produce C-NN: prefixed checklist items without vocabulary contamination.
