**Formula**: `(essential/5 × 60) + (recommended/3 × 30) + (aspirational/37 × 10)`
**Golden**: all essential pass + composite ≥ 80 (requires ≥ 2/3 recommended)

---

**What C-43 adds over C-12:** C-12 tests that each phase has a "do not proceed until every cell is filled" instruction — establishing that a stop gate *exists* with cell-fill intent. C-43 tests that the gate's pass condition *names each mandatory column individually*, requiring a column-by-column enumeration rather than a structural row-presence check. Round 12 scoring revealed that all variations had C-12-style gate language but all gated on row/block presence ("all 9 namespace rows present; no namespace omitted") rather than cell fill per column. A gate can satisfy C-12's intent through wording while still passing when mandatory cells are blank — C-43 closes that gap by requiring the column names themselves to appear in the pass condition.

**What C-44 adds over C-26:** C-26 tests that the pre-signal phase has a named, numbered stop gate blocking artifact reads — establishing Gate-0 *structurally*. C-44 tests that the numbered label "Gate-0" (or equivalent) appears in the *section header* of that gate block, making it scannable as position-0 in the sequential chain without reading the gate body. Round 12 scoring showed V-01 and V-05 satisfying C-26's structural blocking requirement via "CONTRACT COMPLETENESS GATE" but failing to carry a numbered identifier — meaning the gate cannot be found by scanning headers for "Gate-0" and cannot be confirmed as the chain's first link without reading content. The numbered header label is a separate property from structural correctness.

**What C-45 adds over C-38:** C-38 tests that the schema-phase gate exists as a named gated precondition in the sequential chain, positioned before Gate-0. C-45 tests that the gate is internally structured as a per-schema checklist rather than a compound single condition. Round 12 showed that passing variations (V-01, V-05) achieved C-38 via a four-item checklist ([A][B][C][D]) where each item corresponded to a specific schema section — making gate completeness verifiable by counting items against the schema register. A gate with a compound condition ("all §ID schema blocks declared") satisfies C-38 but cannot be audited by item count; C-45 requires atomic per-schema items, so that a missing schema produces a detectable missing check item rather than a silently incomplete compound condition.
