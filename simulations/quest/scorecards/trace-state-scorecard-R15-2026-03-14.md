## Tie-Breaking Analysis: V-01 vs V-03 (both 99.1)

Both pass C-42. V-03 is architecturally stronger: application sentences are sealed in PART 0.0C before any data appears in PART 1, and each phase's pre-detection commitment references them verbatim (Line 2 of 3). V-01's sentences are inline-declared at phase runtime — a model can write a vague sentence for Phase 3D after seeing Phase 3A findings. V-03 closes this post-hoc adjustment window using C-38's seal-before-analysis mechanism. Under the current C-42 definition both score identically, but V-03 is the stronger architectural implementation.

## Excellence Signals from V-05 (Top Scorer)

**Signal 1: Pre-pass APPLICATION SENTENCE confirmation interrupt**

V-05 adds a blocking STOP instruction immediately before Pass 1 in every phase:

> `STOP — confirm APPLICATION SENTENCE is above before proceeding to Pass 1.`

This is structurally distinct from C-24's exit gate checkboxes (end-of-phase) and C-31's LOCKED/OPEN phase-level gates (cross-phase). It is an **inline entry gate** — a named confirmation act embedded at the exact moment the model would begin detection, before any finding is recorded. Without this interrupt, the APPLICATION SENTENCE can be written after findings begin accumulating (within the same phase's generation window). The STOP creates a hard ordering constraint: *application sentence committed → confirmed present → Pass 1 begins*. This is a new structural position: the **application sentence entry gate**, distinct from the exit gate checkbox.

**Signal 2: APPLICATION SENTENCE as post-sweep calibration artifact**

V-05 PART 5B:

> If score < 0.6: identify which APPLICATION SENTENCE was most misaligned with the actual evidence shape encountered. Revise the sentence for the domain scenario used — do not revise retroactively in PART 4; record the revision here as a calibration artifact.

This closes a feedback loop that no prior criterion addresses: C-42 requires the application sentence as a pre-commitment; C-25 requires calibration scoring. V-05 links them — when calibration fails, the *cause* is the misaligned application sentence, and the *output* is a revised sentence as a PART 5 record. The sentence is no longer only a detection-phase artifact; it becomes a **learning artifact** that is feedback-eligible after the sweep. The calibration record is produced after the sweep closes but before the session ends, so the revision is attributable to the specific domain scenario used, enabling session-to-session sentence improvement without retroactive manipulation of the detection record.
