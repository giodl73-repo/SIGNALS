### Excellence Signals (v5 Candidates)

**E-01: Paper type as planning parameter, not Claim mechanism** (V-01 axis)
C-18 requires paper type declared "before or during Claim drafting." V-01 moves the declaration to Phase 0, before signal acquisition. The introductory description makes the scope explicit: "Paper type declared in Phase 0 governs tense, register, and framing for the entire abstract." Background register (domain condition vs. conceptual tension), Gap framing strength (unmeasured vs. unresolved construct), Method scope framing, and Implication form (practitioner decision vs. tractability statement) all differ by type. C-18 captures only the Claim tense mechanism. v5 candidate: **paper-type-as-planning-parameter** — type governs all six sections, not just Claim tense. Distinguish from C-18 via Background and Implication section differences between empirical and theoretical papers.

**E-02: Semantic bridge type labeling with constrained vocabulary** (V-02 axis)
C-12 requires bridges "planned before merge" with non-stock phrases. V-02 adds a Type column with constrained five-term vocabulary (contrast / causation / resolution / escalation / application) and a consecutive-type constraint ("Do not use the same type for more than two consecutive boundaries"). The type-first instruction forces relationship identification before phrase composition — preventing syntactically original but semantically generic phrases. The consecutive-type constraint is a second-order mechanism: it prevents formulaic all-causation labeling without requiring the model to independently judge which type is correct. This is the C-19 candidate referenced in the v4 rubric note: "bridge table with labeled boundary pairs." v5 candidate: **semantic-bridge-type-labeled-vocabulary** — Type column with constrained terms + consecutive-type constraint as the C-19 mechanism.

**E-03: Section coupling verification before bridge composition** (V-03 axis)
No existing criterion captures post-draft, pre-bridge structural verification. Phase 2c's COUPLED / WEAK / REVISE checklist operates at a different lifecycle position than any current mechanism: after all six sections are drafted (so it can verify cross-section semantic fit), but before bridges are composed (so revision is still section-level, not post-merge prose editing). Gap-first order (C-14) prevents Background drift but cannot prevent downstream drift — Method using a different approach than what Claim asserts, or Implication naming a known insight rather than what this Result enables. Amendment 4 (C-16) catches prose flow after merge but cannot revise section content. Phase 2c catches structural misalignment before it propagates into the merged paragraph. v5 candidate: **section-coupling-verification-pre-bridge** — COUPLED/WEAK/REVISE at each of the five section boundaries, positioned post-draft/pre-bridge.

**E-04: All three R4 axes are additive at the ceiling** (V-05 confirmation)
