## Tiebreaker: V-05 over V-03

Both achieve the same composite. V-05 is the design recommendation because:

- **C-02 robustness**: role execution sequence converts C-02 from a post-hoc content audit to a pre-prose structural constraint. If a PM speaks before the sequence position allows it, the violation is detectable before any prose is written.
- **C-10 robustness**: charter constraints surface through two independent mechanisms (phase gates + table columns). V-03 has only one (phase gates). A weak phase gate implementation still leaves C-10 exposed in V-03; V-05 has a table-format backstop.
- **C-12 implementation quality**: phase 0 attendee list expressed as a table with a status column is a cleaner provisional annotation mechanism than inline prose notes.

## Excellence Signals (from V-05 / V-03 top tier)

### 1. Phase-gated charter constraint surfacing (V-03, V-05)

Phase entry conditions enforce C-10 without requiring explicit charter-annotation instructions. When a skill requires Phase 1 entry to confirm quorum and Phase 0 to declare scope, the two most common charter constraints (quorum + scope) surface structurally. The model cannot skip them to reach the simulation — they are prerequisites, not reminders. This is a fundamentally different mechanism than a fill-rule annotation.

**Implication for skill design**: C-10 is most robustly satisfied by phase gates, not by instructing the model to "include charter references." Gate-based enforcement is not captured anywhere in the current rubric as a design pattern.

### 2. Role execution sequencing as positional C-02 enforcement (V-05)

Declaring `ROB: PM → Eng Lead → SRE → Security → Architect` and `ARCH-BOARD: Architect → Eng Lead → Security → SRE → PM` converts C-02 from "does each participant's voice match their role?" (a semantic check requiring content review) to "does each row in the declared sequence match the expected role voice?" (a structural check verifiable by position). A PM in position 1 of a ROB is permitted; an SRE in position 1 of a ROB is a sequence violation before any content is evaluated.

**Implication for skill design**: Sequence tables create a pre-prose compliance gate that makes C-02 violations detectable earlier in output review — and makes the skill harder for the model to satisfy incorrectly without deliberate effort.

### 3. Why all variations hit 98+ (the R2 win propagation effect)

The shared C-13/C-14 wins from R2 raised the R3 floor significantly. Because all variations include the BEFORE YOU START block with per-type fail conditions, all variations prevent the most common C-07 and C-09-adjacent failure modes. The differentiation in R3 has narrowed to aspirational criteria (C-09 outside-in, C-10 charter fidelity) — exactly where the next rubric version should focus.
