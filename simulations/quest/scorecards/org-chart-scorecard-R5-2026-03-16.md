## Excellence Signals from V-05

**1. Gate-sequenced architecture as dependency graph.**
GATE 0 (classify) → GATE 1 (inertia) → structural gates creates a strict prerequisite chain. Role types become inputs to structural decisions, not byproducts. This is what C-20 required but also what makes every downstream assertion traceable.

**2. Checkbox-gated ANNOTATION-CHECK loop.**
The combination of C-21 (checkbox format) + C-22 (blocking verification loop) eliminates the single most common bypass: a model that internally asserts "all conditions met" without enumerating them. The loop holds the gate open structurally — it cannot pass by assumption.

**3. ABSENT/STOP as a structural halt, not a label.**
V-01's ABSENT labeling (C-15) + ARTIFACT END (C-18) converts flat-verdict handling from an annotation into a termination directive. Every other variation leaves a logical path where a model labels a section ABSENT and continues. The two-part pattern — label what's missing, halt the artifact — closes that path.

**4. Interleaving enforced at two levels.**
C-17 (rhythm-charter pairs) + C-19 (post-interleave count gate) closes both the ordering bypass and the silent-drop bypass. A model can interleave correctly for N−1 pairs and silently drop the last charter while satisfying C-17's ordering rule. C-19's count check makes this impossible.

**5. Two-site constraints generalize beyond roles.**
V-01 introduced two-site enforcement for roles. V-05 extends the pattern to org diagram, rhythm-charter coupling, and all critical constraints. The principle: one site permits silent bypass, two sites close it.

**6. Inline examples suppress format drift.**
V-02's contribution — complete example rows in every schema — prevents the most common interpretation gap: a model reads a schema as a field-label template and produces structurally plausible but format-non-compliant output. Worked examples set the bar at the exact expected format.
