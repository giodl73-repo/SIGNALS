## Excellence Signals from V-05

**What made V-05 the top scorer:**

1. **Document-head format contract with section-numbered citations (C-33 PASS + C-35 PARTIAL)** — SECTION 0 declares gate block format (0.1), evidence schema (0.2), moment-label format (0.3), and subject schema (0.4) as named authority. Downstream gates cite "per 0.3 contract" rather than re-stating requirements locally. This collapses per-section format overhead into a single auditable source. V-01 through V-04 all require the executor to remember format rules per section.

2. **`STATUS: OPEN / PASSED` field in every gate block** — gate completion becomes an explicit declared state, not inferred from whether items are checked. The master gate instruction "copy each sub-gate status directly" makes closure mechanical: the PRE-INTERVIEW MASTER GATE items are literally populated by copying sub-gate STATUS fields. No other variation provides this mechanism.

3. **PROHIBITED clauses on Phase 1 per-subject gates** — V-05's Phase 1 gate opens with `PROHIBITED: Cross-subject synthesis, comparison claims`, enforcing scope boundary at gate-read time. V-01 and V-04 have no PROHIBITED clauses on Phase 1 gates, meaning cross-subject material could drift in without tripping any gate condition.

## New Patterns Not Yet in C-01 through C-43

**Pattern 1 — Completion-state field in gate blocks:**
V-05 adds `STATUS: OPEN / PASSED` to each gate block and instructs "copy each sub-gate status directly" in the master gate. This transforms gates from checklists into state machines with declared transitions. C-32 requires named checklist blocks; C-34 requires sub-gate cross-references in the master gate. Neither requires an explicit STATUS field that propagates. The STATUS field makes gate state machine-readable without parsing item content — a structural audit point parallel to C-42's "row count, not cell values" logic.

**Pattern 2 — Point-of-use format compliance cited to contract section, not master gate:**
V-05's point-of-use confirmations cite "per 0.3 contract" or "per 0.2 schema." C-35 requires point-of-use confirmations to cite the **master gate** as authorization authority. V-05 achieves a parallel pattern via the **format contract**: the Section 0 authority is cited instead. C-35 only covers master gate citations; a new criterion would cover format contract citations — "when a point-of-use gate item enforces a format requirement, it names the Section 0 contract section by number as the compliance source rather than restating the requirement locally."
