### Excellence Signals — V-05 (top variation)

**1. Cross-block ordering integrity via column-value propagation.**
BLOCK 0's Amendment Cost column (High/Medium/Low) creates a downstream ordering contract with BLOCK 5's Priority Rank. The contract is: Priority Rank ordering MUST reflect Amendment Cost ordering from BLOCK 0 for corresponding signals. No F-ID currently enforces this. This is a structurally new enforcement class — not count-parity (F-12), not identity matching (F-10), not content-completeness (F-21 through F-26) — but *value-to-ordering continuity*. First instance of a cross-block constraint where the output column ordering is dictated by a source column value in an upstream block.

**2. Inertia framing as a structural enforcement constraint.**
"The design is committed and in-flight; it can only be amended, not replaced." This is not merely stylistic. It constrains the action vocabulary in BLOCK 5: any action row recommending "redesign X" or "replace Y" is non-conformant under inertia framing. The framing itself is instrumentally enforceable — a named F-ID could fire when any Action cell uses replacement vocabulary rather than amendment vocabulary. This extends the enforcement surface from structural shape (table form, column count, cell presence) into *semantic vocabulary compliance*.

**3. Priority Rank uniqueness as a cross-row constraint class.**
The requirement "no two rows SHALL share the same Priority Rank" cannot be verified by inspecting a single cell. It requires comparing a cell's value against all other cells in the same column — a comparative check, not a presence or content-validity check. This is algorithmically distinct from every previous named halt (all of which fire on individual cell inspection). Priority Rank uniqueness is the first constraint in this instrument that requires cross-row state.
