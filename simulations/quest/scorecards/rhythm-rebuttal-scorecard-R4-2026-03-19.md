### Excellence Signals (new R4 patterns from V-05)

**Pattern 1 — Rule violation consequence as pre-generation enforcement**: Adding "Violation: [specific failure mode]" to each RULE-N clause transforms a behavioral instruction into a structural constraint with named stakes. A rule that states "Violation: any Phase 5 block missing REVIEWER SAID / AUTHOR RESPONDS / MANUSCRIPT OUTCOME is incomplete — rewrite before submission" is not additive to the format prescription — it shifts when the format's importance is recognized from post-hoc review to pre-generation reading. The consequence makes non-compliance visible before the first exchange is written, not detectable only after the output is produced. This is distinct from R3's "format-as-rule architectural convergence," which was about placing the format rule in the pre-workflow block; R4 adds the cost of violation to that rule.

**Pattern 2 — Pre-classification type audit as upstream error gate**: Phase 3.5 requires one explicit justification sentence per scope, framing, or methodological assignment before the forecast is written. The sentence must state the specific distinguishing reason for this concern (not a restatement of the type definition). This is structurally different from providing a misclassification guide and relying on the author to apply it — the audit forces committed justification output per ambiguous assignment, making a wrong type assignment visible before it propagates to the forecast table, the exchange, or the cover letter. The three most confusion-prone types are caught at the earliest possible gate.

**Pattern 3 — Forecast flag carry-forward as live exchange annotation**: Phase 4 forecast rows targeting specific R-IDs are carried into Phase 5 exchange headers as FLAGGED annotations: "FLAGGED: [F-0N] [predicted failure mode — resist it]." The author cannot write the exchange without seeing the predicted failure mode. Phase 6b then validates FLAGGED exchanges first (did the predicted failure occur?) before checking unflagged exchanges (did any unpredicted failure occur?), creating two structurally distinct accuracy checks. This eliminates the isolation between the forecast step and the drafting step — the forecast is no longer a pre-generation memo but an active label that travels with the concern through Phase 5 and into the amendment pass.
