### Excellence Signals

Three patterns emerged from R3 beyond the two mandated structural fixes. They appear in one or two variations and represent the next generation of discrimination candidates:

**From V-05 — Explicit substitution mappings in dimension rule**
Rather than prohibiting unlisted names, V-05 provides canonical redirects: "Reliability → use Correctness. Performance → use Scalability. Discoverability → use Usability." A model encountering an ambiguous finding gets a resolution path rather than a format error. This is a stronger C-06 enforcement mechanism than a prohibition list alone.

**From V-05 — Role Architecture diagram as pre-execution halt declaration**
Declaring `COMMIT-STAGE-3-FLAGGED (halt all)` in the Role Architecture diagram — before any stage specification runs — makes the stop path visible at the structural level. A model parsing the architecture understands the halt possibility before it encounters Stage 3.5. This is qualitatively different from having the halt described only at the point of decision.

**From V-04 — Worked examples as positive modeling for Signal Source / Design Impact**
V-04's calibration examples ("trace-state-signal-scan-2026-03-10, trace namespace, dated 2026-03-10" and the full Design Impact example) demonstrate correct format by showing rather than prohibiting. The hypothesis is that models comply more reliably with demonstrated patterns than with prohibition-only language. This was untested in prior rounds.

**From V-03 — Gate Sequence Overview table surfacing halt behavior at the top**
Making Stage 3.5's stop condition visible in a table at the top of the skill — with "Stage 3.5 is a stop gate" called out explicitly — means the halt behavior appears in the skill's overview before the model reads any stage instructions. This structural visibility may reduce the probability of the model treating Stage 3.5 as an optional advisory step.
