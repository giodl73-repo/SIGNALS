### Excellence Signals from V-04/V-05

**E-01 — Additive synthesis without structural conflict (V-04):** C-44 (all-transition rationale) and C-45 (source-gate declarations) occupy orthogonal locations in Protocol II — the rationale block answers *why* each boundary is a gate; the inline `*[Source: FROM Phase X]*` annotations answer *where* the required input comes from. Both can coexist in a numbered prose list without competing for the same structural slot. The synthesis requires only appending the "anti-bundling rationale for all transitions" block after the phase list and appending `*[Source:...]*` to each output-dependent completion condition — no restructuring of the base format.

**E-02 — Protocol V pre-committed dependency graph with temporal separation (V-05):** The temporal-separation commitment device (established in Protocol III for self-diagnosis, in C-29 for bridge format) can be applied to the full inter-gate dependency structure. A pre-committed execution dependency graph positioned before Protocol II closes a gap that inline source-gate declarations (C-45) leave open: inline declarations in Protocol II are co-temporal with the lock's construction, making it impossible to verify they were not added while reading forward. Protocol V, positioned before Protocol I/II/III, makes the complete dependency structure independently auditable before any execution preamble exists. The gate: Protocol V is declared before Phase 0, before signals exist, before any paragraph exists — the dependency structure cannot be a post-hoc rationalization of the lock.
