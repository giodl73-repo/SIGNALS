### Excellence Signals from V-05

**Pattern 1 — First-position belief anchoring**
`**What We Expected (B-[#]):**` is the first labeled sub-field, before Signal Source, Design Impact, and Build Risk. Position matters: when B-# is first, the model commits to a specific prior belief before constructing any analytical field. When B-# is second or later, the analytical fields are already partially formed and the B-# selection becomes post-hoc rationalization. First position forces forward-commitment to the belief; later positions allow backward selection of whichever belief the analysis happens to match.

**Pattern 2 — Three-point convergent apparatus for C-33**
V-05 satisfies all three C-33 apparatus elements simultaneously:
- (a) Sub-field in Field Reference block — field presence enforced before entry loop
- (b) Surprise 0 demonstrates `**What We Expected (B-2):**` with a full sentence — imitation floor
- (c) Per-entry EXIT gate states "full sentence referencing a specific B-# from Stage 1 — not a generic 'we thought it would work'" — binary gate with named failure mode

V-01–V-04 satisfy (a) and (b) but their EXIT gates enforce B-# presence structurally through surrounding design (deviation chain: Stage 2 B-# → GATE-CONFIRMED-[N] → entry N) rather than naming it as an independent gate condition. The chain can be intact and still permit a generic sentence if the model loses the link between entry index N and B-# label.

**Pattern 3 — Failure-mode naming at EXIT gate**
The prohibited phrase "not a generic 'we thought it would work'" names the exact failure pattern the model reaches for when no specific prior belief is available. This is the same structural move as C-32 (naming "multiple sources," "the signals," "various artifacts") and C-17 (naming "Reliability," "Performance"): converting a positive-only requirement into a named-antipattern exclusion at the point of emission. The named failure pattern is harder to accidentally produce than an unnamed one.
