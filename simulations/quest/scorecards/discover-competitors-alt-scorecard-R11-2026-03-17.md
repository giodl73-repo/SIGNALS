## Excellence Signals — V-05

**1. Write-token pattern generalized to control-flow gates (→ C-43)**
GATE 4's write row (which established INERTIA-REF as the data-dependency root) is structurally identical to GATE 0's write row in V-05. The pattern: a gate that resolves a consequential binary state should produce a named output-contract token — not just set control flow. FOCUS-STATE as slot 0 makes the path-selection event machine-readable as a slot dependency in the "Required by" column of every downstream slot that branches on it (slot 5). This is what fixes C-34 in V-01/V-04/V-05.

**2. Vacuous inactive path encoded as format token, not prose permission (→ C-44)**
Non-V-02 variations use "vacuous satisfaction when slot 0 = inactive" — prose that requires interpretation at evaluation time. V-02/V-04/V-05 replace this with `VACUOUS-5: focus-inactive`. Both the active path (`REDUCTION-1 NO: [...] | REDUCTION-2 NO: [...] | THEREFORE: [...]`) and the inactive path (`VACUOUS-5: focus-inactive`) are now checkable by token presence without parsing prose. This closes the format gap between the two branches of a conditional output slot.

**3. Pre-DAG step entry in EXECUTION DEPENDENCY (→ C-45)**
V-03/V-05 add GATE 0 as Step 0 with `-- (pre-DAG)` in the Prerequisite steps column. This distinguishes two structurally different roots: GATE 4 is the *data-dependency root* (`None (root)` — no upstream slots required) and GATE 0 is the *pre-DAG root* (`-- (pre-DAG)` — runs before the dependency graph is entered). The full execution sequence — including pre-root control-flow gates — is now readable from the single EXECUTION DEPENDENCY table without reading gate narrative.
