Patterns from the R3 variation set that made all 5 reach 100/100:

**1. Named-gate text that identifies the blocked step by role name** (C-13)
All 5 include text like `"Role N may not begin until this block appears in output."` The gate names both the trigger condition and the blocked step. This is structurally stronger than "proceed to next role" because the blocked role is named — a model cannot accidentally skip it by reordering unless it removes explicit text.

**2. N/A prohibition with field identification** (C-14)
All 5 include `"No action may omit [Field N / the Breaks field]. 'N/A' is not valid."` The prohibition closes the escape hatch by name. Variations without this could omit the pattern tie-back field by writing `N/A` or leaving it blank without a visible violation.

**3. Quantified inertia with urgency tiers** (V-02 innovation)
The 0–5 health score converts pattern detection from a binary "present/absent" finding into a scored severity metric. The urgency tiers (Healthy/Monitor/Alert/Critical) make the action priority derivable from the score, not from judgment. This is a richer C-10 implementation than categorical naming alone.

**4. Section-boundary gate as structural constraint** (V-05 innovation)
`"ASSESSMENT must close before FINDINGS opens. No exceptions."` encodes C-11 as a template constraint rather than a role pipeline instruction. Violating this gate requires reordering named sections, which is more visible than skipping a role. Format-level constraints are harder to accidentally violate than procedural ones.

**5. Evidence-first leaderboard before named pattern** (V-04 innovation)
Running the leaderboard before the inertia diagnosis means the reader sees the contributor distribution before the pattern is named. A Lone Wolf pattern visible in the leaderboard data is _confirmed_ by the diagnosis rather than _announced_ by it. This creates a more credible inertia claim for the reader.
