## Excellence Signals

From top-scoring variations (V-03, V-04, V-05) that were absent in V-01/V-02:

**Signal 1 — Trigger-gated cascade rule pattern**
RULE IB-URGENCY-CASCADE introduces a "field value → mandatory section behavior" architecture. The IB-02 Urgency Gradient field is the trigger; the named rule fires when the trigger is declared HIGH. The cascade is conditional on an explicit baseline attribute value, not always-on. This pattern separates baseline characterization (what the urgency is) from downstream compliance (what sections must do about it).

**Signal 2 — Three-constraint cascade enumeration with per-section behavioral specificity**
The cascade rule names three downstream sections with distinct required behaviors (cite IB-02 / include delay-compounding entry / name compounding path). Each constraint specifies what the section must do with IB-02, not just that it must mention IB-02. This mirrors the three-condition audit schema pattern (C-28): named schema entries with labeled functional requirements.

**Signal 3 — Incidental-compliance anti-pattern closes the cascade loophole**
"Citing IB-02 in downstream sections without a declared Urgency Gradient value... does not satisfy the cascade constraint." Without this anti-pattern, a generator could cite IB-02 incidentally (e.g., in a risk register for unrelated reasons) and claim C-39 satisfaction. The anti-pattern requires the Urgency Gradient field to be populated HIGH in IB-02 before any downstream citation counts toward the cascade. The anti-pattern makes the cascade's prerequisite explicit rather than implicit.
