## Excellence Signals from Top-Scoring Variations (V-04 and V-05)

**1. Schema citation embedded in HALT header creates gate-inseparable recall (C-35)**
Moving the schema reference from the `BEFORE PRODUCING -- re-read FORMAT CONTRACT schema (N):` label line above the HALT into the HALT line itself (`HALT. Re-read FORMAT CONTRACT schema (N) -- [name].`) makes recall structurally inseparable from the gate trigger. The model encounters the schema reference at the moment it encounters the HALT — not as a subsequent checklist item that can be processed after the gate fires.

**2. ROLE-NAME LOCK as co-located per-site instruction document (C-36)**
Adding full procedural per-cell notes immediately after the role-name list converts the lock declaration from a passive name list into a co-located instruction document for sites 2 and 5. The enforcement procedure travels with the declaration: the model encounters the per-cell directive at declaration time (STEP 3) rather than requiring recall from a downstream PRODUCE-command section hundreds of tokens later. Declaration time is the earliest possible enforcement point.

**3. Declaration-time + produce-time dual per-cell enforcement**
V-04 and V-05 achieve two independent per-cell enforcement checkpoints separated by maximum token distance: (a) ROLE-NAME LOCK per-cell notes at declaration time (before any content-producing step), and (b) per-cell sequencing directives at the PRODUCE-command CHECK sites. The failure mode that either checkpoint covers is caught by the other — the dual enforcement eliminates the single-point-of-recall dependency.

**4. Cross-reference from sites enumeration to adjacent notes (V-05 architectural refinement)**
V-05's sites enumeration says "per-cell directive above governs site 2 from declaration time forward" — an explicit forward reference connecting the adjacent declaration-time note to the downstream site enumeration. This creates a navigable instruction traversal path that makes the co-location benefit explicit: the model is told not only that the note exists but that it governs from declaration time forward.

**5. C-35 + C-36 stacking is non-interfering**
Both closing mechanisms operate on different structural positions (HALT header line vs. ROLE-NAME LOCK block body) and different temporal positions (produce-time gate vs. declaration-time instruction). They do not compete for the same instruction sites and create no register split — adding both simultaneously does not require any compensating adjustment to other criteria.
