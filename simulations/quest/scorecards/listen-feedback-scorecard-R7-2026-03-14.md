### Excellence Signals from V-05

1. **Combined axes are mutually reinforcing** — UX-first primes inertia-aware craft critique; mandatory `Current approach:` fields give PM READ concrete named patterns to synthesize; ascending card order puts the revision work before the confidence-boosting promoter cards. Each axis amplifies the others.

2. **NPS sensitivity quantified** — "by approximately how much (e.g., moving C-03 from 4 to 7 shifts mean by +0.25')" forces the model to compute, not estimate, leverage.

3. **UX READ creates a verifiable prediction list** — framing UX concerns as "switching-cost amplifiers" produces named predictions that the theme matrix can explicitly verify or refute. This closes a feedback loop no single-axis variation attempted.

4. **Inertia-framed band definitions** — defined at the top of the prompt, non-negotiable, so every NPS score and band label in the output is anchored to the same standard.

### New Pattern

**A-14 — UX-to-matrix verification loop** (format/depth): The UX READ section produces a named prediction checklist of expected persona friction points (framed as switching-cost amplifiers). The Cross-Persona Theme Matrix section explicitly closes the loop: states which predictions materialized as themes, which did not, and what the persona cards surfaced that the UX READ missed. The verification note is a subsection of the theme matrix, not a separate section (which would violate A-13). Pass condition: UX READ contains a named bulleted prediction list; theme matrix section contains an explicit "Verification against UX READ predictions" passage accounting for all predictions; the verification is inside the terminal theme matrix section, not after it.
