## Excellence Signals from V-01

1. **Dual evidence taxonomy as orthogonal columns (C-18 + C-27)**: V-01 declares Experience-Proximity (GROUNDED/CONDITIONAL/INFERRED) and Origin-of-Claim (verbatim/inferred/corroborated) as separate columns with an explicit "do not collapse" instruction. Every other variation omits Origin-of-Claim entirely. This means V-01 outputs are auditable on two independent axes simultaneously.

2. **Explicit sequence rationale paragraph (C-19)**: V-01 has a named `SEQUENCE DECLARATION — INCUMBENT FIRST` block with a multi-sentence rationale explaining *why* the order was chosen and what contrast is lost if reversed. Other variations embed sequence logic in adjacency blocks but never surface it as a named declaration.

3. **Experience-Proximity activating synthesis weight adjustment (C-24)**: V-01's Experience-Proximity column feeds directly into the EPISTEMIC RE-WEIGHTING section, creating a complete pipeline from evidence tagging to synthesis weighting. Other variations re-weight only by blind spots.

**What V-05 achieves that V-01 misses:**
- Phase 4 exit gate (C-20 PASS vs PARTIAL): V-05's gate forces synthesis section completeness to be structurally auditable at gate-check time.

## New Patterns Not Captured by C-01 through C-41

Two structural patterns emerge from V-01 and V-05 that have no direct criterion in v12:

**Pattern A — Named-anchor verbatim propagation chain**: A named structural output (the Phase 0A inertia anchor) must appear with exact textual match — explicitly "not a paraphrase" — as it travels from Phase 0C declaration → Phase 1 subject card field → Phase 2 question text. C-30 requires confirmation that a prerequisite is satisfied before a dependent mechanism deploys; it does not require verbatim named-output matching. This pattern makes anchor drift structurally detectable without content inspection: a Phase 2 gate item that checks "question names Phase 1 inertia anchor directly (not a paraphrase)" will fail if the anchor was restated rather than reproduced.

**Pattern B — Vocabulary contract extended to Phase 4 accounting closure**: V-05 declares the PROFILE RELEVANCE vocabulary at Phase 0A exit gate (C-41) and then explicitly states in the Phase 0A gate item that "Phase 4 INERTIA PROFILE ACCOUNTING must address each tag category represented." C-41 covers the entry declaration; no criterion covers the exit-closure requirement that every declared vocabulary category is explicitly accounted for in synthesis. This creates a closed-loop vocabulary contract: entry gate declares the set, synthesis gate confirms the set was fully consumed.
