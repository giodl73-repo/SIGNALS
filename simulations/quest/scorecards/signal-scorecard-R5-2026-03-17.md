### Excellence Signals

The three R5 axes each close a real failure mode not addressed by v5 criteria. They are observable in the prompt structures and warrant rubric extraction:

**Pattern 1 — Filter-mode completeness gate (C-20 candidate)**
V-01, V-04, V-05 add a per-namespace count check to the FILTER compliance gate, closing the gap where a model could omit a skill silently but pass the scope check. Structurally parallel to C-18 for bare mode. The symmetry argument is strong: if bare mode needs a count gate, filter mode has the same omission risk.

**Pattern 2 — Bare-mode order anchor with boundary markers (C-21 candidate)**
V-02, V-04, V-05 add "Begin with scout-competitors. End with org-committee." to BARE MODE. This converts "in namespace order" from an implicit instruction (requires model to read catalog top-to-bottom) to a machine-verifiable contract (first and last lines are explicitly specified). A 61-line correct-format output with wrong first or last line is caught by this anchor; C-18's count gate would pass it.

**Pattern 3 — Catalog transcription gate (C-22 candidate)**
V-03, V-05 upgrade the NAMESPACE CATALOG header from passive "emit verbatim" to active "confirm you have read every entry... Any deviation from the catalog text is an output failure, not an acceptable approximation." This shifts the model's instruction from a style preference to a commitment contract, closing the semantic-reconstruction failure mode where the model reads the catalog for comprehension then reconstructs from internal summary.
