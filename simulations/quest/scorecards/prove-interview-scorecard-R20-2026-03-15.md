## Excellence Signals from V-05

**1. Bidirectional contract closure (extends C-51)**
V-05 doesn't only declare the contract source at Phase 0A — downstream gates (Phase 1, Phase 2, Phase 3) each explicitly cite the source gate as the compliance authority. Phase 3 gate item 4 states "a tag list reproduced here without citing Phase 0A as contract source creates a parallel contract and does not satisfy this condition." This creates a closed loop: source names itself → receivers name the source → no intermediate gate can silently reproduce content without attribution. C-51 tests the source-side declaration; V-05 adds the receiver-side citation as a structural pattern.

**2. Gate-block-first rationale declares joint purpose (C-49 × C-51 interaction)**
In V-05 Phase 0A, the gate-block-first rationale reads: "gate-block-first placement makes vocabulary compliance **and contract authority** immediately auditable at gate entry." The rationale explicitly names two purposes served by first-position placement — compliance AND contract authority — rather than one. This is a new combined property: gate ordering performs double duty when the chain-critical item is also the contract source.

**3. Propagation + first-position + source citation collocated in one gate item (C-49 × C-50 × C-51)**
In V-05 Phase 0C, the first gate item contains: the INCUMBENT anchor (chain-critical, C-49), the gate-block-first rationale (C-49), the downstream propagation obligations block (C-50), and the propagation rationale sentence ("declared at the source so any chain break is auditable at the phase that names its predecessor"). V-04 has the same item minus the implicit C-51 context; V-05's Phase 2 adds "compliance is with the Phase 0C exit gate propagation obligation, not with any generalized instruction." The three criteria collapse into a single auditable gate item — scanning one gate item confirms anchor, obligations, and compliance authority simultaneously.
