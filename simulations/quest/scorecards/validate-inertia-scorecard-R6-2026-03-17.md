### Excellence Signals from V-30

**1. Self-labeling boundary markers as produced artifacts**
The "SCORING INFRASTRUCTURE DECLARED." statement is not a directive — it is a required *output* the model must write. This converts Phase 1's implicit role as the C-17 citation source into an explicit artifact with a name. Phase 4 then cites the artifact by that name, creating a traceable reference chain rather than a paraphrase chain.

**2. Gate = explicit advance prohibition, not format instruction**
"Do not write Phase 3 until every persona's inventory passes all three checks" and "Do not write Part (4) until the falsifiability test produces a specific observable condition" are structural prohibitions. The difference from format instruction ("you should...") is that a prohibition makes non-compliance structurally visible — the model would have to skip a labeled gate to proceed, which is harder to do silently than to produce weak content inside a required column.

**3. Cross-phase citation chain: Phase 1 → Phase 4 → Phase 5 → AMEND**
V-30's named infrastructure anchor creates a chain: Phase 1 produces "SCORING INFRASTRUCTURE DECLARED" → Phase 4 cites it by name → Phase 5(3) falsifiability test produces a named lever point → AMEND (d) inherits that lever point as its observable condition. Each downstream phase is anchored to a named upstream artifact, making the chain traceable and making failures (where the chain breaks) structurally detectable.

**4. Upstream inventory closes downstream construction pressure**
Building the competitive strength inventory in Phase 2 (completeness gate enforced) means Phase 5(2) cites a pre-named durability property rather than constructing competitive framing under analytical pressure. This is the mechanism behind C-22's design: the upstream asset exists before the high-stakes analysis, so "familiarity" Durability cannot be constructed on demand at Phase 5 without contradicting a Phase 2 entry.

**5. Falsifiability test as self-rejection mechanism**
Requiring the model to write "At T=6mo, the absence of this lever would be observable as: ___" between Part (3) and Part (4) forces self-evaluation at mechanism-specification time. The rejection clause ("Replace Part (3) if you cannot complete this") is a self-referential gate — the model rejects its own output, not the rubric. This is stronger than a rubric that evaluates specificity after the fact.
