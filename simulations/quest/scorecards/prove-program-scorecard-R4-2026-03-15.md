## Excellence Signals from V-05

**1. EXPERIMENT N COMPLETE tokens with consumed-by routing close C-12**
V-01–V-04 all earn C-12 PARTIAL because GATE-δN / GATE-EN PASS conditions confirm execution but don't produce a named handoff token. V-05 adds explicit `EXPERIMENT N COMPLETE / Produced: … → Consumed by:` per block — a discrete artifact marking experiment closure and its downstream destination. This is what C-12 tests.

**2. Role anchoring of the verbatim contract**
V-01 and V-04 place the blockquote requirement in a Phase 2 header block. V-05 places it inside the EXPERIMENTER role body itself, making it a role property rather than a phase header instruction. When the EXPERIMENTER role starts, the verbatim contract is already in scope — it doesn't require the model to re-read a preamble.

**3. Three-layer enforcement stack working in concert**
V-04 has two layers (GATE tokens + blockquote prohibition). V-05 adds FAIL labels at every enforcement point. The convergence — named gate (don't advance), COMPLETE token (acknowledge handoff), FAIL label (name the failure mode) — creates redundant enforcement paths so any one mechanism catching a violation is sufficient.

**4. Role transition contracts as explicit handoff surfaces**
`PLANNER COMPLETE → EXPERIMENTER receives: [list]` and `EXPERIMENTER COMPLETE → SYNTHESIZER receives: [list]` create scope-limiting transition contracts. Each role knows exactly what it received and what it may not touch — this prevents retroactive hypothesis changes (blocked by SYNTHESIZER rule) and premature synthesis (blocked by EXPERIMENTER rule).
