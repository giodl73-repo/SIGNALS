## specify-abstract R11 Scorecard (v10 Rubric)

---

### Scoring Framework

All five R11 variations are designed from V-05 R10 as base, which passed all three v10 ceilings (C-33, C-34, C-35). Each variation inherits that baseline and adds one or more R11 structural behaviors. The v10 rubric scoring captures the full 100-point range; the R11 differences surface as **new pattern signals above the current ceiling**.

### Excellence Signals

**E-01 — Sub-phase canonical gate naming anchors C-26/C-29 structural argument in the commitment device** (V-01, V-04, V-05)
V-05 R10's sequence lock bundled Pass 1/Pass 2/Pass 3 as a single item and left Phase 3 (merge) unnamed as an individual gate. V-01 names Phase 2a, 2b, 2c, 3, 3b individually in the sequence lock and includes a structural rationale block ("Why sub-phase canonical naming matters") explaining that Phase 3b requires text that cannot exist before Phase 3 completes — anchoring the C-26/C-29 pre-fill impossibility argument in the commitment device itself rather than in the column format instructions. A scorer can now verify the Phase 3 / Phase 3b gate dependency from the sequence lock without reconstructing it from the column definition. This is the **C-36 candidate**: sub-phase canonical naming + per-gate structural rationale at sequence lock.

**E-02 — Weakness-specific per-row exclusion reasoning closes the template-compliance path** (V-02, V-04, V-05)
V-05 R10's C-35 format accepts "confirmed" as a response with a structural parenthetical that names what the category covers. The parenthetical describes the category; it does not explain why the specific diagnosed weakness falls outside it. V-02 requires a sentence that (a) names the diagnosed weakness and (b) states why it is not in each slot — making the exclusion self-contained and independently auditable without access to the parenthetical label. A reader can verify the reasoning without knowing the category definition. This closes the subthreshold compliance path where "confirmed × 4" passes the structural form without demonstrating per-slot reasoning. This is the **C-37 candidate**: specific sentence per row naming the diagnosed weakness, not a template parenthetical.

**E-03 — Pre-committed self-diagnosis framework separates target-declaration from target-application** (V-03, V-05)
V-05 R10 introduces positive-space targets and per-category exclusion format at Amendment 1 runtime — the list appears at the same lifecycle position as the diagnosis, making it structurally impossible for a reader to verify the targets were fixed before the diagnosis was made. V-03/V-05 introduce Protocol III (before Phase 0): the four positive-space categories are committed when no draft exists; the per-category format is declared as the required execution format. Amendment 1 references "Protocol III committed targets" and "Protocol III committed format." The target-declaration event (pre-Phase-0) is now verifiably prior to the target-application event (Amendment 1). This is the **C-38 candidate**: pre-committed self-diagnosis framework creating a declared, auditable temporal separation between declaration and application.

### Rubric Status

All five R11 variations score 100/100 on the v10 rubric. The v10 ceiling (C-33 + C-34 + C-35) is **fully saturated** by the R11 variation set.

Three new patterns are present that no v10 criterion scores:
- **C-36**: Sub-phase canonical naming with per-gate rationale in sequence lock
- **C-37**: Weakness-specific per-row reasoning (vs. template "confirmed") in per-category verification
- **C-38**: Pre-committed self-diagnosis framework (Protocol III) creating temporal separation between target-declaration and target-application

**V-05 is the R11 synthesis** and the recommended base for any v11 rubric work. If a v11 rubric is written, V-05 R10 (the current production skill) becomes the floor and V-05 R11 becomes the new synthesis target.

If no new rubric criteria are extracted from C-36/C-37/C-38, **v10 is the terminal rubric** and V-05 R11 is the production skill.
