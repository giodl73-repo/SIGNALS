### Excellence Signals

**From V-02 — Phrasing register as C-25 enforcement pathway:**
Promoting the RESOLUTION PROTOCOL from inline gate text to a standalone ALL-CAPS headed declaration block — peer-register with GATEKEEPER FUNCTION DECLARATION — makes the named-repair-per-flag-type requirement visually sovereign. A reviewer cannot scan past the gate specification without encountering the repair structure. The visual distinctness converts C-25 from a property embedded in gate logic into an independent structural unit, making it impossible to satisfy C-22 (CHAIN-FLAG blocks production) without simultaneously encountering C-25's repair routing. Phrasing register is a zero-overhead enforcement mechanism.

**From V-03 — Role-boundary protection as C-26 enforcement pathway:**
The Enforcement Framer (EF) role's Step 0 placement — before any signal artifacts are read — makes the NO-ECHO COST declaration structurally prospective. Because the role scope excludes signal entry content, the cost cannot be reverse-constructed from what the signals say. In inline C-26 implementations, a writer could observe the signal content first, then write a cost declaration shaped by what was found. The EF role eliminates this path: the cost is declared from pre-investigation materials only, ensuring the declaration is genuinely forward-looking. This is the strongest semantic realization of C-26's requirement that the enforcement section carry stakes before entries rather than summarize them after.

**From V-04/V-05 — RULES-ANCHORED-COMPLETE as auditable distillation closure:**
The token mirrors C-19/CHAIN-COMPLETE exactly — same function, different phase. C-23 blocks production when RULES-ANCHORED check fails; RULES-ANCHORED-COMPLETE confirms when it succeeds. The token makes distillation-phase exit observable from output: a downstream reviewer can determine whether distillation completed successfully without re-running the audit. The C-19/CHAIN-COMPLETE pattern was already a top-performing pattern in v7; RULES-ANCHORED-COMPLETE extends the same auditable-closure discipline to the second major audit phase. This seeds C-28 for v9: the v9 rubric can require this token the same way v7 added C-19.

**From V-01 — Intentional C-26 omission as boundary confirmation:**
V-01 reaching 180 (not 185) despite adding the R8 isolated target confirms that RULES-ANCHORED-COMPLETE carries no v8 scoring weight. V-01 is a clean experiment: it proves the token's role is seeding-only in v8, and that the composite ceiling requires all three of C-25+C-26+C-27 — the token does not substitute for any of them. This boundary confirmation is useful for rubric integrity.
