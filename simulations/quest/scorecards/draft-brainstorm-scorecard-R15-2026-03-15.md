The patterns that distinguish V-02 and V-04 from V-03 (the baseline) under hypothetical v15:

**Signal 1 — Phase 1 Lifecycle Opening Tag (C-41 candidate)**
V-01, V-02, V-04, V-05 all add an explicit lifecycle signal immediately after the Phase 1 requires-line. In formal register: `> **Phase 1 opens now.** Phase 0 complete. Do Nothing established.` In conversational register: `> **Phase 1 is live.** Phase 0 done -- the inertia baseline is set.` In prose format: `*Phase 1 opens now. Phase 0 complete. Do Nothing established.*` Phase 1 already carries a closing blockquote (`> **Phase 1 complete.**...`). The opening tag completes the dual-anchor. The plain italic requires-line (`*Phase 1 requires: Phase 0 complete...*`) already present in all variations is a precondition statement, not a lifecycle opening signal — it does not satisfy C-41. The new tag is structurally distinct: it announces Phase 1 is **now active**, not just what Phase 1 requires.

**Signal 2 — Phase 2 Lifecycle Opening Tag (C-42 candidate)**
V-02, V-04, V-05 add an explicit lifecycle signal immediately after the Phase 2 requires-line. In formal register: `> **Phase 2 opens now.** Phase 1 complete. Full pool written, AMEND drafted.` In conversational register: `> **Phase 2 is live.** Phase 1 done -- pool built, AMEND draft written.` Phase 2 already carries a closing blockquote (`> **Phase 2 complete** when:...`). With both C-41 and C-42 present, every phase in the skill that carries a closing lifecycle blockquote (Pre-Phase, Phase 0, Phase 1, Phase 2) now also carries an opening lifecycle blockquote.
