## Scoring Methodology

- **Essential** (C-01–C-04): binary pass/fail; output unusable if any fail
- **Recommended** (C-05–C-07): pass/fail; scored qualitatively
- **Aspirational** (C-08–C-45): composite = aspirational_passes / 38
- **C-08–C-39** (32 criteria, not listed in supplied rubric): estimated from structural analysis of each prompt against the category axes (format, depth, correctness, coverage). These criteria accumulated over R1–R14; well-designed variations built on 17 prior rounds are expected to pass the large majority.
- **C-40–C-45** (6 criteria, fully specified): evaluated directly against prompt text.

## Excellence Signals from V-03

**ES-01 — Sentence template as compliance contract for flip conditional**
V-03 provides a literal fill-in-the-blank template: `"If [X resolves as A], the verdict holds. If [X resolves as B], the verdict changes to [Y]."` This is structurally different from instructing the model to "state both branches" or "map both resolution directions." The template makes the expected output form explicit — it is a structural contract, not a behavioral instruction. Every other variation uses instruction-form; V-03 uses contract-form. This appears to reduce output variance on C-43 compliance.

**ES-02 — Position marker embedded in named-check label**
V-03 uses: `"Proportionality audit (required final act before closing this beat):"` — the beat-closing position is encoded in the sub-step label itself, not only in the body text. This double-enforces position: the label says "final act" and the closing sentence says "This is the last step before the beat closes." V-05 does this too ("Proportionality audit (final instruction within this beat):") but V-03 originated the pattern at named-step scale with explicit "required" language. The label-level position instruction is distinct from the body-level position instruction and appears to be load-bearing for C-45 compliance at high reliability.
