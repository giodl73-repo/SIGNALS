## R11 New Territory vs R10

R10 confirmed **C-33** (self-describing form names satisfy C-31 without parenthetical elaboration when the name encodes the failure mode). R11 probes two open questions from that finding:

1. **V-01** tests whether C-33's principle extends up one level — to *column names in output schemas* rather than *form names in invalid-form catalogs*. If "Surface" and "Fix" are self-describing enough to carry C-31, the column constraint text can be omitted.

2. **V-05** tests whether C-33's precision can be applied to divergence explanations: `PerspectiveLabel` and `CodeProperty` are form names that encode exactly the failure mode each represents — the first encodes "role-identity attribution", the second encodes "surface code property without architectural layer". Both names are self-describing in the C-33 sense, and neither requires a parenthetical to establish the governance function.
