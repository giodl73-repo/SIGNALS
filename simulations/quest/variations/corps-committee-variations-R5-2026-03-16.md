## corps-committee -- R5 Variations

| ID | Axis | Mechanism |
|----|------|-----------|
| V-01 | Lifecycle emphasis | `## COMMITTEE TYPE GATE` — skeleton block between Participants and PHASE-0-COMMIT; 2 checkboxes (vocabulary + specific subject); halts Phase 0 if unticked |
| V-02 | Inertia framing | Structured `## INERTIA CONTINUITY BRIDGE` — expands single prose line to multi-field skeleton block with Inspection, Decision, OWNER RECORD or INJECTION RECORD, and BRIDGE RESULT halt clause |
| V-03 | Output format | `*** TALLY LEDGER PROTOCOL ***` — named Phase 4 protocol enumerating each STANCE MANIFEST entry, per-token category assignment, independent sums, participant-count cross-check with self-referencing restart on discrepancy |
| V-04 | V-01 + V-02 | Phase 0 Gate + Structured Bridge — additive coverage at Phase 0 exit and Phase 3 entry, both using the same skeleton-gating pattern |
| V-05 | Full R5 integration | All three R5 mechanisms on the complete R4 V-05 foundation — extends skeleton gating to Phase 0, Bridge, and Phase 4 |

**Single-axis choices:** V-01 (lifecycle), V-02 (inertia framing), V-03 (output format).

**Candidate criteria for rubric v6:**
- C-20 — Phase 0 Vocabulary Gate (from V-01)
- C-21 — Bridge Completeness Record (from V-02)
- C-22 — Ledger-Verified TALLY (from V-03)

V-05 absorbs no rubric v5 penalty (it inherits all C-17/18/19 from R4 V-05) and is the candidate to score against v5 and then surface the v6 rubric upgrade.
