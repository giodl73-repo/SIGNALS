**V-01 — Single-axis: Lifecycle Emphasis (C-26)**

Hypothesis: when phases are load-bearing structure rather than post-hoc labels, retroactive softening of prior commitments is a visible phase-boundary violation.

V-02 makes it a required discrete block with fixed test condition and declared result. Hypothesis: correct tier ordering without SORT-CHECK is a visible missing field — the gate is the enforcement mechanism, not the ordering outcome.

Hypothesis: the switching-cost framing raises the bar for GATE-N: YES and makes C-21's `RESPONDS-TO:` requirement structurally natural — advocates must name a tag, ruling out generic acknowledgment.

Phase structure makes gate positions unambiguous (each gate is a specific phase's exit condition); SORT-CHECK two-field format makes the Phase 2 exit condition a discrete labeled gate. Each criterion locks into a phase position.

**V-05 — Combined: All Three Axes as Pre-Printed Template (C-24 + C-25 + C-26)**
Golden synthesis. All three new criteria become pre-printed template slots: `INVESTIGATION-ATTEMPT-N` blocks with incrementing labels (C-24), `SORT-CHECK:` block with `Test:` + `Result:` fields (C-25), Phase 0–5 skeleton headers (C-26). Compliance becomes transcription — a missing sequential label, missing gate field, or missing phase header is a visible gap.
