## Round 17 Variations — Summary

**Two new criteria probed (C-50, C-51):**

| | What it is | Single encounter failure it closes |
|---|---|---|
| **C-50** | Pre-table FM-ID Protocol block before the SQ Declaration | D-16: author skips to cells after reading section header; Protocol block intercepts them |
| **C-51** | Compact two-clause token (`does not count; Mandatory`) as leading clause in every SQ field cell | D-15: trailing token hidden by column truncation in wide tables |

**Variation matrix:**

| | C-50 | C-51 | Axis |
|---|:---:|:---:|---|
| **V-01** | FAIL | **TARGET** | Output format: leading-clause truncation-safe placement |
| **V-02** | **TARGET** | FAIL | Inertia framing: pre-table section-entry Protocol block |
| **V-03** | FAIL | FAIL | Phrasing register: descriptive prose dissolves enforcement structure |
| **V-04** | PARTIAL | PARTIAL | Role sequence + lifecycle emphasis: phase-first framing dilutes SQ block |
| **V-05** | **TARGET** | **TARGET** | Inertia framing + output format: full dual-encounter defense-in-depth |

**V-05 is the R17 ceiling** — the only variation that structurally blocks both D-15 and D-16 simultaneously. V-01 blocks D-15 only; V-02 blocks D-16 only; V-03 blocks neither; V-04 partially closes both without fully satisfying either check.

The key discriminator: the two encounter points are independent. An author who reads the Protocol block but skips to fill still encounters the leading-clause token. An author who reads only the leading clause without the Protocol block still has section-entry guidance waiting. Only when both are absent (V-03) or incomplete (V-04) do the gaps emerge.
