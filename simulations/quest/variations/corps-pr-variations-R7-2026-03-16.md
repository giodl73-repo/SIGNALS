## R7 — corps-pr: 5 Variations

| Variation | Axis | R7 Innovation |
|-----------|------|---------------|
| **V-01** | Inertia framing | C-21: Budget verdict is a 3-clause formula — `WORSE rows:` / `BETTER rows:` / `Conclusion:` as named lines. A verdict that reaches the correct direction without naming the rows fails C-21 on string-match, not interpretation. |
| **V-02** | Lifecycle emphasis | C-22: Phase C entry has two symmetric output blocks — C1 produces `C1 RESULT: ALL CLEAR / BLOCKED`; C2 per-role produces `C2 RESULT: SATISFIED / UNSATISFIED`. Both failure modes are visible output gaps, not just pipeline violations. |
| **V-03** | Role sequence | C-23: READ RECEIPT is pre-commitment — written before the PR diff is re-read for findings. The IA cost anchor shows a required before/after delta: `CONFIRMED / REVISED` with a named code artifact. Post-hoc receipts produce rationalized initial positions; the timing constraint closes that. |
| **V-04** | Inertia framing + Lifecycle emphasis | C-21 + C-22 chain: B3 checklist item is specifically "3-clause structure present"; C2 compliance block cites the verdict's `WORSE rows:` / `BETTER rows:` lines, not just a raw ledger row. Citing the verdict's row lists proves both that Phase B was read and that the derivation structure was recognized. |
| **V-05** | All four R7 axes | C-21 + C-22 + C-23 + C-24: the IA findings table uses `IA-gate` column; all technical role findings tables use `Domain-Lens` column. Universal table schema means every findings section is format-auditable by the same column-check regardless of role. |

**What each variation advances over R6:**
- R6 V-01 said "verdict derived from row directions." R7 V-01 enforces `WORSE rows: [names]` — checkable by string match.
- R6 V-02 had C1/C2 structurally. R7 V-02 makes both produce output result lines with explicit PASS/BLOCK values.
- R6 V-03 had the receipt. R7 V-03 makes the initial position pre-commitment with a post-findings delta as required output.
- R7 V-04/V-05 chain the mechanisms so each criterion is verifiable from the prior phase's named fields, not from re-reading prose.
