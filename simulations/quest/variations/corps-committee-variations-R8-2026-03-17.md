## corps-committee R8 -- Variation Summary

| ID | Axis | New Criteria Targeted |
|----|------|-----------------------|
| V-01 | **Outcome vocabulary contract** | C-24 only |
| V-02 | **Tier-sequenced voice protocol** | C-25 only |

---

### Key structural innovations per variation

**V-01 (C-24):** Adds a `VOCABULARY LOCK:` line before Phase 0. Every STANCE, TALLY, and Verdict token is constrained to the per-type committed set. Wrong token = observable type mismatch, not prose drift.

**V-02 (C-25):** Adds a `TIER SEQUENCE PROTOCOL` block in BEFORE YOU START. Each participant gets a pre-assigned tier with rationale. Discussion renders as T1 → T2 → T3 with explicit `TIER BOUNDARY` gates. An Advocate speaking before a Challenger is a named violation.

**V-03 (C-27):** Adds an `INERTIA FINDING INVENTORY` block (INERTIA-FINDING-1..N). Every CONDITION/BLOCK stance and every dissent row carries a mandatory `Inertia Token` column. "No dissent" rows must state which finding was resolved and by what evidence — no blank cell accepted.

**V-04 (C-26):** Splits the discussion into three separate tier grids, each with a distinct required column schema. T1 needs `Inertia Citation`; T2 needs `Conditions + Condition Basis`; T3 needs `CITE + RESPONDS-TO`. Column mismatch is structurally visible without reading prose.

**V-05 (full integration):** Combines all four registries (vocabulary lock, tier protocol, inertia inventory, Challenger designation) in a four-block BEFORE YOU START, then threads every token constraint through Phase 0/1/2. The PRE-MORTEM CHAIN CHECK gate from C-19 anchors CHAIN-3 to Phase 2's pre-mortem risk, making the inertia → pre-mortem → dissent chain end-to-end traceable.
