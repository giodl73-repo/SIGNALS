## Round 5 Design Summary

**Context**: R4 achieved 100/100/100/100/100 on rubric v2 — all structural paths saturated. R5 explores axes not yet exercised.

### Variation Table

| Variation | Axis | Hypothesis |
|-----------|------|------------|
| **V-01** | Scenario decomposition gate | Force explicit event tuple (entity + message + field deltas) before any candidate is named — closes scope structurally, not narratively |
| **V-02** | Counterfactual pairing | Every entry (firing or not) carries a `Flip to NOT FIRE` / `Flip to FIRE` pair — makes C-07 and C-12 bilateral obligations rather than separate checklists |
| **V-03** | Non-firer-first enumeration | Block 1 disposes of all non-firers with gap tokens before Block 2 fires anyone — C-12 becomes structurally prior to C-01 |
| **V-04** | Witness citation per entry | Every entry must name the registered artifact (solution, step, flow name) that proves it fires — C-05 becomes per-row not per-document |
| **V-05** | Combined: decomposition + counterfactual pairs + R4 authority contracts | Three new mechanisms layered on R4's strongest structural foundation |

### Key structural innovations over R4

- **Decomposition gate** (V-01, V-05): closes candidate scope to an explicit event tuple — first time scope is mechanically defined before the scan rather than inferred from scenario prose
- **Counterfactual pairs** (V-02, V-05): bilateral condition reasoning required for every row; C-07 and C-12 collapse into one unified obligation
- **Non-firer-first** (V-03): reverses enumeration order so gap disposal is the opening obligation, not a trailing cleanup
- **Witness citation** (V-04): grounds each entry in a named registered artifact; unwitnessed entries are flagged rather than silently accepted
