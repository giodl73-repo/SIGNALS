# program-plan R16 Variate Prompt Bodies (V-01 through V-05)

Rubric: v16 (C-01 through C-51, 310 pts). New targets: **C-50** (4-column table index) and **C-51** (protocol-first placement).

---

## V-01 — Output Format Axis (SCAN PROTOCOL — 4-Column Table Index)

**Hypothesis**: A tabular C-41 index promotes each data type to an independently scannable column. The table header names the column semantics (field type | criterion | tag string + full Why: | C-42 back-ref), making C-44/C-47/C-48 simultaneously visible per row. C-50 satisfied.

**C-50 form**: BAD PLAN C1 header is a 4-column pipe table. Column 3 carries the full `Why: execution-history check, not artifact-verifiable` text for the gate_fail: row.

**Anticipated**: 310/310.

---

## V-02 — Role Sequence Axis (BOUNDED BLOCK PROTOCOL — First Section)

**Hypothesis**: Positioning the protocol as the document's first major section means every subsequent section — three-class table, lifecycle zones, catalog, construction steps — is read through the protocol frame. C-51 satisfied. A model encountering the BAD PLAN block already knows its three-component structure by name.

**C-51 form**: BOUNDED BLOCK PROTOCOL section precedes all implementation content. Only a single opening sentence precedes it.

**Anticipated**: 310/310.

---

## V-03 — Lifecycle Emphasis Axis (Zone Decision Tables)

**Hypothesis**: Formalizing each zone's advancement criteria as a 3-row table (Commit Trigger / Artifact Floor / Gate Pattern Class) makes zone commitment independently auditable. A model building gates checks its output against the Zone Decision Table column, not prose paragraphs. **Anticipated**: 300/310 (C-50 and C-51 not targeted).

**New structural element**: Three Zone Decision Tables declared before the catalog opens. Protocol name: **BLOCK AUDIT PROTOCOL**.

---

## V-04 — Inertia Framing Axis (Per-Step AVOID Blocks)

**Hypothesis**: The F-1 inertia behavior (catalog-first) is most effectively countered at the exact step where a model would otherwise default to it. Every construction step header carries an explicit `**AVOID (F-1):**` block naming the concrete status-quo behavior displaced. **Anticipated**: 300/310 (C-50 and C-51 not targeted).

**New structural element**: Per-step AVOID blocks at Steps 1–9 headers. Protocol name: **ANNOTATION VERIFICATION PROTOCOL**.

---

## V-05 — Combined Axis (C-50 + C-51: Table Index + Protocol First)

**Hypothesis**: Combining principle-first placement (C-51) with a tabular C-41 index (C-50) under the same protocol (BOUNDED BLOCK PROTOCOL) tests whether the two patterns coexist and reinforce. The Component 1 spec in the protocol section names the 4-column table structure before the BAD PLAN header is encountered. Both C-50 and C-51 pass. **Anticipated**: 310/310.

**Structural design**: BOUNDED BLOCK PROTOCOL first (C-51), Component 1 spec describes the 4-column table format, BAD PLAN header implements that table (C-50).
