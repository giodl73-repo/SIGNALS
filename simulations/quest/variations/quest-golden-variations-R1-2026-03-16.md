## V-01 — Role Sequence

**Axis:** 6 named roles in fixed execution order per round (RUBRIC KEEPER → VARIATOR → SCORER → ANALYST → CRITERION AUTHOR → CONVERGENCE JUDGE)

**Hypothesis:** Structural sequencing enforces C-04 as a property of role order rather than a compliance instruction — ANALYST must complete before CRITERION AUTHOR can run, making QU2-before-QU3 unbypassable. RUBRIC KEEPER as the first role structurally enforces C-05.

The skill body names each role with a mandate, separates ANALYST (QU2) and CRITERION AUTHOR (QU3) as distinct sequential roles, and keeps the CONVERGENCE JUDGE as a final distinct step that checks both gates simultaneously.

---

## V-02 — Output Format (Tables Throughout)

**Axis:** All round records, scoring, and criterion proposals expressed as tables with fixed columns.

**Hypothesis:** Table 1 (variations), Table 2 (scoreboard with one column per criterion), Table 3 (excellence signal + criterion proposal with fixed rows) map directly to the C-06 fields, making convergence checkable by column inspection. The contrast column in Table 3 structurally enforces C-07 (isolation, not description). The criterion proposal table enforces all three C-08 fields.

The skill body replaces prose-scored evidence with a full P/F matrix table, adds a convergence check table, and enforces the round log as a 4-column table.

---

## V-03 — Lifecycle Emphasis (Hard Phase Gates)

**Axis:** 4 explicit phases per round (PHASE 0 INITIALIZATION → PHASE 1 VARIATION → PHASE 2 SCORING → PHASE 3 ANALYSIS → PHASE 4 CONVERGENCE CHECK) each with ENTRY CONDITION, PROCEDURE, EXIT CONDITION.

**Hypothesis:** Hard entry/exit conditions prevent premature convergence declaration (C-01) because convergence requires completing PHASE 3, which requires the PHASE 2 artifact, making artifact dependency explicit. QU2 and QU3 are named steps within PHASE 3 with stated order, strengthening C-04.

The skill body reads as a protocol document with gates; dual convergence requires successfully passing through PHASE 4's two-condition table.

---

## V-04 — Phrasing Register (Conversational / Intent-First)

**Axis:** Describes the WHY behind each step ("here's what you're looking for...") rather than issuing imperatives.

**Hypothesis:** Intent-first framing embeds C-04 compliance by explaining that QU2 must precede QU3 because "a criterion not grounded in an observed output gap is guesswork, not discovery" — the model reasons from the goal, not the instruction. Predicted to improve C-09 (mechanism narrative) because the explanatory style invites the model to explain what it discovered. Risk: softer language may weaken C-01 and C-05 gate precision.

The skill body flows as continuous prose with bold lead-ins per step; convergence conditions are stated as two bullet requirements.

---

## V-05 — Combination: Inertia Framing + Lifecycle Emphasis

**Axis:** Inertia baseline as named primary competitor + explicit phase gates.

**Hypothesis:** PHASE 0 names the inertia prompt (unconstrained output) before any variations are written. PHASE 3 QU2 compares the top variation against both the runner-up AND the inertia baseline — the inertia contrast is declared the more important of the two. This produces sharper C-07 signal isolation because the contrast reveals what the loop adds, not just what one variation does better. Phase gates anchor the inertia comparison inside PHASE 3 so it cannot be skipped. "What Made It Golden" naturally becomes "what inertia missed," strengthening C-09.
