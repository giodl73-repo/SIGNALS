## Round 15 — Variation Map

| ID | Axis | Hypothesis |
|----|------|------------|
| **V-01** | Schema-as-sole-truth | Phases contain zero inline prose constraints — every rule lives in FIELD SCHEMA, COVERAGE SCHEMA, or STAKEHOLDER SCHEMA rows; phases reference only schema IDs. Closes the **instruction-drift vector**: prose constraints in phases can diverge from schema rows on revision; schema-only phases cannot drift. |
| **V-02** | Inertia vulnerability linkage | INERTIA REGISTER gains a "Stakeholder Most Harmed" column; per-phase override directives and exit gate verbatim reproductions include that role name. Closes the **abstract-default vector**: a model rationalizes past an abstract default more easily than a named-role consequence. |
| **V-03** | Failure cascade registry | Pre-pipeline FAILURE CASCADE REGISTRY (FC-NN IDs) collects execution-time model failures separately from team inertia defaults. Pipeline cites FC-NN; phases reproduce FC-NN text at exits. Mirrors the C-42 architecture improvement, applied to model-execution failures. |
| **V-04** | Inertia vulnerability + Commitment block gate | V-02's vulnerable-role INERTIA REGISTER + COMMITMENT BLOCK gate topology (fill-in structured templates with Status: PASS/FAIL fields instead of checkbox lists). Active authorship of each gate field produces stronger commitment than passive checkbox ticking. |
| **V-05** | Failure cascade registry + Schema-as-sole-truth | Both registries (INERTIA REGISTER + FAILURE CASCADE REGISTRY) with zero-prose phases (schema citations only). Maximum ID-citation density; all constraint text in named registries; nothing can drift. The most machine-readable form of the prompt. |
