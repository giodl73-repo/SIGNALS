## Variations: listen-adoption — Round 1

**Rubric:** v1 | **Axes picked:** Output format · Lifecycle emphasis · Inertia framing

---

### Axis selection rationale

| Single-axis | Axis | What it tests |
|---|---|---|
| V-01 | Output format (table-primary) | Does mandatory table structure force C-01/C-08 role completeness and C-04 delta ordering? |
| V-02 | Lifecycle emphasis (explicit month blocks) | Do named phase headers make C-02 and C-10 structural rather than discretionary? |
| V-03 | Inertia framing (incumbent named) | Does naming THE INCUMBENT force causal specificity in C-03, C-04, and C-09? |
| V-04 | Lifecycle + Output format | Do phase blocks + table columns together close both temporal and role-coverage gaps? |
| V-05 | All three combined | Does the causal layer (inertia) fill in *why* each time-step stalls, completing C-09? |

---

### V-01 — Output Format (Table-Primary)

Replaces prose narrative with seven mandatory tables (archetype map, simulation timeline, champion network, interventions, churn register, S-curve). Prose confined to causal analysis sections.

**Critical rubric bets:**
- C-01/C-08: 16 named rows in SECTION 1 + every role must appear in the timeline table — omission is a visible empty cell
- C-04: "Adoption delta" is a required column, so ordering is mechanical
- C-07: SECTION 7 S-curve table forces numeric phase-close counts against the 16-role total
- **Risk:** Tables compress per-role mechanism detail; C-09 influence dynamics may reduce to terse column entries

---

### V-02 — Lifecycle Emphasis (Explicit Month Blocks)

Five named phases (Innovator Ignition → EA Spread → Chasm → EM Uptake → LM Close), each subdivided by month with fill-contract fields. Phase-close state declarations at each boundary.

**Critical rubric bets:**
- C-02: Every month block requires "who + why + cumulative" — no telescoping
- C-03: PHASE 3 is a mandatory structural stop with CHASM-A/B/C subsections
- C-10: "Deploy before: Month M" field on every intervention in PART C
- **Risk:** Without forced table columns, individual roles can still disappear inside "Early Adopter cohort" labels in month rows

---

### V-03 — Inertia Framing (Incumbent Named)

THE INCUMBENT is declared in a PREAMBLE block before any simulation. Every chasm cause, every churn risk, and every intervention must reference what THE INCUMBENT still offers or what behavior it displaces.

**Critical rubric bets:**
- C-03: Chasm cause must name the specific incumbent advantage, not generic "change resistance"
- C-09: Influence pathway must answer "what incumbent argument does this champion counter?"
- C-04: Intervention carries "Displaces from incumbent: [specific habit]" — forces concrete targeting
- **Risk:** Adding incumbent framing to every field may overwhelm prompts where the incumbent is diffuse (e.g., "no current process")

---

### V-04 — Combined: Lifecycle + Output Format

Phase headers + per-month tables with required columns. Phase-close state declarations persist. Each month block has a `| Role | Archetype | Trigger |` table forcing named rows.

**Complementarity:** V-02 enforces temporal completeness; V-01 enforces role completeness. Together they make C-08 (all 16 traceable in body) pass structurally — each role must appear as a row in at least one month table.

---

### V-05 — Full: Lifecycle + Output Format + Inertia Framing

All three axes active. Month tables gain an "Incumbent pull" column. Phase-close blocks gain an "Incumbent position" field. Intervention table adds "Displaces from incumbent" column. S-curve summary adds "Incumbent position" column.

**Full rubric coverage target:** All 10 criteria have a structural home. C-09 is the hardest — it requires an explicit influence pathway from early to late cohort; the "Incumbent argument countered" column in the champion table forces this to be causal rather than descriptive.
