## R3 Variate Summary

Five complete variations targeting **C-15** (person-portrait quality) and **C-16** (resistance profile extraction), building on R2's established mechanisms:

| Var | Axis | Key Mechanism | Targets |
|-----|------|---------------|---------|
| **V-01** | Inertia framing | Portrait standard + per-file transplant-test annotation | C-15 |
| **V-02** | Lifecycle emphasis | 3-field PROFILE phase (`defended-artifact` + `change-pressure` + derived `lens-phrase`) as hard gate before any IA write | C-16 |
| **V-03** | Phrasing register | Output-forward derivation chain showing how profile fields flow to portrait fields, with annotated strong/weak examples at each stage | C-15, C-16 |
| **V-04** | Inertia + Lifecycle | Explicit derivation table (profile field -> IA file field) + portrait self-check gate; makes C-15/C-16 one pipeline | C-15, C-16 |
| **V-05** | All axes | All 8 aspirational criteria in linear sequence; VALIDATE -> CONTRACT TABLE -> PROFILE -> IA PORTRAITS -> ROLES -> CHART -> CROSS-REF | C-09 through C-16 |

**What's new in R3 vs R2:**
- R2 V-03 showed a strong/weak IA example but didn't require the transplant test -- V-01 makes it an inline gate
- R2 V-04 had resistance profiles with "credibility source" + "failure mode" fields -- V-02 tightens to the exact C-16 three fields (`defended-artifact`, `change-pressure`, `lens-phrase`) and requires explicit derivation
- V-03 introduces the derivation chain concept (showing the inference rather than requiring it) -- no R2 precedent
- V-04 adds the derivation table as a build contract for the IA phase -- makes the profile→portrait connection structural rather than inferred
- V-05 composes all 8 aspirational criteria into the R2 V-05 skeleton, inserting the PROFILE phase between CONTRACT and IA-WRITE, and adding the portrait self-check inside the IA phase
