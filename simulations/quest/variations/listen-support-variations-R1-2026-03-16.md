## Round 1 Variations — listen-support (v1 rubric)

**Axes selected for this round** (prior R1/2026-03-15 already covered role-sequence, output-format, lifecycle-emphasis):

| Variation | Axis | Single/Combined | Primary probe |
|-----------|------|-----------------|---------------|
| **V-01** | Phrasing register — imperative/technical | Single | C-03: does a hard prohibition list ("do not write: the SRE, they") produce better first-person compliance than a soft invitation? |
| **V-02** | Phrasing register — performance/conversational | Single | C-03: does "I AM / MY SITUATION / MY FRUSTRATION" activation produce richer first-person voice — and does "I, as the SRE" bleed appear? |
| **V-03** | Inertia framing — front-loaded competitor | Single | C-09: does locking PRIOR TOOL before any ticket is written make competitor grounding structurally guaranteed without naming the criterion? |
| **V-04** | V-02 + V-03 | Combined | C-03 + C-09 simultaneously — does performance activation produce richer transition-friction bodies? |
| **V-05** | V-02 + V-03 + pre-commitment table | Full synthesis | C-03 + C-09 + C-10 together — does the metadata table flatten the voice, or does the activation block preserve it? |

**Key design choices:**

- V-01 uses a **prohibition list** rather than an affirmative instruction for first-person — testing which enforcement style is stronger for C-03
- V-03 requires the PRIOR TOOL name to appear in at least one body and includes a post-generation check (`[PRIOR TOOL] named in body: [T-NN list]`) that fires before gap analysis
- V-05 splits taxonomy decisions (Step 3 table) from prose execution (Step 5 bodies) with a TABLE CHECK gate — the same table/prohibition pattern that drove the R3→R20 golden progression
- All 5 variations include phase-severity guidance targeting C-07, and role coverage checks targeting C-08
