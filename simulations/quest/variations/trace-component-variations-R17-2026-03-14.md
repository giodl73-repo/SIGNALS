## R17 Variations Summary

| Variation | Axis | Target | Key Change from R16 |
|-----------|------|--------|---------------------|
| **V-01** | Role sequence | C-52 alone | Register headers use compound form: `**LABEL** -- descriptor`. Questions NHQ-1, NHQ-2 derive why. Schema Area 3 declares template. Plain close markers. C-51 manifest close baseline. |
| **V-02** | Output format | C-53 alone | Per-register close markers use compound form: `**LABEL CLOSE** -- [accomplishment]`. Questions NCQ-1, NCQ-2 derive why. Schema Area 3 declares template. Plain headers. C-51 manifest close baseline. |
| **V-03** | Lifecycle emphasis | C-54 alone | Manifest closes are two grammatically independent sentences. Questions DSQ-1–3 derive why one-sentence embedding fails. Schema Area 3 declares two-sentence template. Plain headers and close markers. |
| **V-04** | Role sequence + Lifecycle | C-52 + C-54 | Compound headers (NHQ-1, NHQ-2) + dual-sentence manifest closes (DSQ-1–3). No compound close markers. Tests whether C-52+C-54 together produce emergent C-53. |
| **V-05** | All axes + Phrasing | C-52 + C-53 + C-54 | All three compound/dual-assertion forms. Schema Area 3 split into 3a/3b/3c. Unifying principle made explicit: no boundary is a bare positional marker — every boundary is a double assertion. |

**Key structural distinctions across variations:**
- V-01 vs V-02: compound form applied to the header side vs the close side of the same register boundary
- V-03 is independent of V-01/V-02: manifest close sentences are orthogonal to causal phase register structure
- V-04 tests cross-domain combination: schema boundary style (C-52) + trace boundary style (C-54)
- V-05 uses `Area 3a/3b/3c` labeling to make the three templates individually referenceable in Role 2 instructions
