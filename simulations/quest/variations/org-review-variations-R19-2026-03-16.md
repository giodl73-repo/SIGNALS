# org-review -- Variate R19 (rubric v11)

**R19 design target**: All five R18-v11 variants predicted at 225/225 through three paths (pre-§§ schema block, dense behavioral §13, scope-anchored NH dimensions). R19 explores three fresh single axes to stress-test robustness:

| Variation | Axis | C-33 mechanism | C-34 mechanism | C-35 mechanism | Predicted |
|-----------|------|---------------|---------------|---------------|-----------|
| V-01 | Role sequence | LENS APPLICABILITY PRE-REGISTRATION step (step 2) before any reviewer runs | §14 behavioral + gap-check as step 11 | §3 NH table format | 225/225 |
| V-02 | Lifecycle emphasis | §14 behavioral Applicability field | DOMAIN COVERAGE GAP-CHECK as numbered step 10 in SECTION ORDER CONTRACT | §3 NH table format | 225/225 |
| V-03 | Phrasing register | TYPED FIELD: Applicability with 4 constraints + immutability | MANDATORY OPERATION with single exception class and named deviation case | PRIMACY DEVIATION CASE (table governs if prose diverges) | 225/225 |
| V-04 | Role sequence + Lifecycle emphasis | Pre-registration (step 2) | Numbered step 11 | §3 NH table | 225/225 |
| V-05 | All three | Pre-registration + typed field + deviation case | Numbered step + MANDATORY OPERATION + deviation case | Typed table + PRIMACY DEVIATION CASE | 225/225 |

**Key R19 innovations vs R18:**

- **V-01** shifts C-33 from "declare schema in preamble, enforce at §13" to "commit Applicability ratings before reviewers run" — the model has already produced the pre-registration table before it generates any finding, making retroactive omission structurally impossible
- **V-02** promotes C-34's gap-check from a behavioral mandate inside §13 to a numbered section in the SECTION ORDER CONTRACT — omitting it is now a section-order violation, not just a §13 protocol violation
- **V-03** introduces TYPED FIELD definitions and DEVIATION CASES — C-35 gets a **PRIMACY DEVIATION CASE** that handles the specific failure mode where prose narrative asserts a different g_null than the table formula; the table wins by name
- **V-04** combines the two structural-ordering axes (pre-registration + numbered gap-check) for dual structural mandate on C-33/C-34 without any behavioral text dependence
- **V-05** applies all three: structural pre-commitment (C-33), section-order mandate (C-34), and explicit deviation case (C-35) simultaneously
