## Design summary

**What R10 proves:**

| Variation | Axis | C-24 | C-25 | Key differentiator |
|-----------|------|------|------|-------------------|
| V-01 | Phrasing register | PASS | FAIL | CONSISTENCY GATE as Step 2 — named, discrete, numbered — with abstract taxonomy only. C-22 FAIL isolates C-24 as the single new axis. |
| V-02 | Output format | FAIL | PASS | EXAMPLE blocks embedded in pair template. Consistency is a parenthetical `(mechanism must be consistent with archetype)` — field annotation, not named step. C-22 FAIL isolates C-25. |
| V-03 | Role sequence | FAIL | FAIL | Three-field format: Archetype → Mechanism → Composability-claim. Archetype precedes mechanism, but no verification action between them. C-22 PASS confirms C-24/C-25 failure is orthogonal to provenance. |
| V-04 | Register + format | PASS | PASS | CONSISTENCY GATE + embedded EXAMPLES + gate provenance. Full chain: C-21+C-22+C-23+C-24+C-25. |
| V-05 | Lifecycle + inertia | PASS | PASS | V-04 + post-inspection distribution audit (macro-structural pipeline finding) + opening inertia context block. Potential C-26 signal. |

**Scoring math:** V-01 and V-02 both score 100 through the C-22 compensation path: 18 proven (−C-22) + 3 unproven (C-21+C-23+C-24 or C-21+C-23+C-25) = 21 pts at budget cap. V-03 takes the cleaner path: 20 proven + 2 unproven (C-21+C-23) = 22 → capped 21 → 100. V-04/V-05 earn 4 unproven pts against the same 21-pt cap.

**Critical boundary being established:** C-24's pass condition is a **named, discrete verification step** — V-02 demonstrates the boundary by having a parenthetical reminder that implies consistency but does not instruct the author to verify it as an action. V-03 demonstrates the same boundary from a different angle: field ordering (archetype before mechanism) does not satisfy C-24. Together, V-01/V-02/V-03 establish the full pass/fail geometry for both new criteria.
