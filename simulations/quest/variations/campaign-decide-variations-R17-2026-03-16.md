## R17 Summary

**Three single-axis variations, two combinations.**

| Var | Axis | Key Change vs. R16 V-05 |
|-----|------|------------------------|
| V-01 | **C-43 candidate — preamble temporal annotation** | SCHEMA PREAMBLE experiment schema columns get `(fill-now)` / `(defer-to-Phase-5)` in their header text at definition time — one source of truth instead of distributed declarations at gates |
| V-02 | **Inertia framing — Decision Stakes block** | A single-paragraph `**Decision Stakes:**` block appears before the SCHEMA PREAMBLE, making the inertia-displacement argument before any schema mechanics appear |
| V-03 | **Lifecycle emphasis — phase budget annotations** | Each phase header gains `[min: N]` bracket annotations (`[min: 3 experiments]`, `[min: 2 named rivals]`, etc.) — under-population becomes a header-level failure, not a body-prose reading |
| V-04 | **Combined: C-43 + phase budgets** | V-01 + V-03: orthogonal axes — fill discipline vs. evidence density — no interaction effects |
| V-05 | **Full integration targeting 107.0/107.0** | All three R17 axes over R16 V-05 baseline |

**Two new criterion candidates extracted:**

- **C-43** (V-01): temporal commit in SCHEMA PREAMBLE column headers at definition time — implies C-42, implies C-28; the chain extends to C-43 → C-42 → C-40 → C-38 → C-36
- **C-44 candidate** (V-03): per-phase minimum evidence count as bracket-notation gate annotation in section header — independent of C-01 through C-43, detectable by header scan only

**Design note on C-43 vs C-42:** C-42 declares fill-phase at closure-directive time (distributed across 6 phase gates). C-43 declares it at preamble-definition time (single location, propagates to all gates). The closure directives in V-01 become confirmations rather than declarations — same verifiability, higher structural authority.
