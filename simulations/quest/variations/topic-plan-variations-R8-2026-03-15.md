## Round 8 Design

**New criteria this round**: C-28 (WS blocks as first-class `##` section headers), C-29 (write-surface register table before Phase 0), C-30 (each WS block cites its register row number).

All five variations share these structural invariants:
- A `## WRITE SURFACE REGISTER` table as the first major section (C-29)
- `## WS-1`, `## WS-2`, `## WS-3` as top-level section headers — auditable by scanning titles (C-28)
- Each block opens with `WS-N MILESTONE — fulfills Register Row N` (C-30)
- Gate-0 inside WS-1 before any artifact opens (C-26)
- Sequential named gates referencing next phase (C-20)
- Defense register before signals (C-18)
- Two-tier sentinel vocabulary with violation enumeration (C-17, C-21)
- Banned forms listed verbatim with cell-level check instructions (C-23, C-24, C-25)
- Counted-total delta sentence with integer enforcement (C-15, C-19)
- Typed null declarations per absent change type (C-09)
- Conflict detection with typed null form (C-10)

## Key Structural Distinctions

**V-03 and V-05** rename the mandatory column to `Null Hypothesis Defeater [MANDATORY]` and `Null Hypothesis Override [MANDATORY]` respectively — stronger semantic forcing than `Inertia Defeated`.

**V-05** is the only variation where WS-1 fires *after* filename/date inventory but *before* content reading. This inverted sequence lets the defense register be calibrated to how many NEW artifacts exist, producing stronger defense arguments than a blind pre-inventory defense. The register declares this explicitly: `"WS-1 fires after inventory but before content reading."` This is a structural innovation that could seed a Round 9 criterion.
