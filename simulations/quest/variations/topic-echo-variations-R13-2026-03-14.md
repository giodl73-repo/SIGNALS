## Round 13 Variations — Design Summary

**Primary structural target:** Make C-30 the default, not the exception. All five R12 variations that had C-30 FAIL (V-01–V-04) are redesigned with template-complete baseline entries. If all five R13 variations pass C-30, R13 is the first confirmed round.

**Secondary structural target:** Second confirmation of C-29. All five R12 variations passed C-29 (first confirmed round). If R13 confirms uniformly, C-29 promotes to Proven in v13.

**Axis selection:**

| | Variation | Axis | C-29 | C-30 |
|---|---|---|---|---|
| V-01 | Single | **Phrasing register** — conversational imperative ("Fill in every field. Cut anything that would appear in a findings summary.") vs. the formal documentary register of prior rounds. | PASS | PASS |
| V-02 | Single | **Lifecycle emphasis** — explicit phase boundary headers (PHASE 1: PRE-WRITE / PHASE 2: PER-SURPRISE / PHASE 3: POST-WRITE / PHASE 4: OUTPUT) as the primary organizational frame, above and distinct from step numbers. | PASS | PASS |
| V-03 | Single | **Output format (numbered-list pairs)** — each pair inspection is a numbered list item `(1) Archetype / (2) GATE / (3) Mechanism / (4) Composability-claim`, distinct from the step-labeled field blocks (prior rounds) and the four-column table (V-02(R12)). | PASS | PASS |
| V-04 | Combined | **Phrasing register + Lifecycle emphasis** — tests whether conversational register and lifecycle phase headers compose without structural degradation. | PASS | PASS |
| V-05 | Combined | **Phrasing register + Lifecycle emphasis + Inertia framing** — three-axis composition: register + lifecycle + inertia preamble. Tests full surface-layer independence from the structural criteria layer. | PASS | PASS |

**Key structural invariant across all five:** The ★BASELINE entry carries all four fields including `Composability-claim`. This is the C-30 universalization: what V-05(R12) demonstrated as a single-variation finding becomes the baseline expectation across all variation families.
