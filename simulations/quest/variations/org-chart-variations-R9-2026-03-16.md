### What each variation adds

The existing CHECKPOINT INERTIA-CHECK (GATE 1) and CHECKPOINT DIAGRAM-CHECK (GATE 2) are unchanged. C-22 was already satisfied by two checkpoints; C-29 requires all four.

**V-02 (C-30 + C-31 — GATE 3 dual boundary):** Adds a `GATE 3 PROHIBITED CONTENT` contract listing: appendices, supplementary sections, editorial commentary after Anti-Pattern Watch, extra charters, any content outside Steps 3.1–3.5. The flat-verdict `STOP` (C-18) was already present; C-31 seals the other exit path symmetrically.

**V-03 (C-32 — complete charter example):** Replaces the field-label charter template with a complete filled example — `### Architecture Review Board` / `**Mission:** Evaluate and approve...` / `**Membership:** Principal Engineer (DECISION), Engineering Manager (GOVERNANCE), Staff Engineer (ADVISORY)` / `**Quorum:** 2 of 3 member roles` / `**Escalates:** VP Engineering`. The filled example teaches the Quorum fraction form and the `(TYPE)` annotation syntax that field labels alone cannot demonstrate. Upgrades C-11 from PARTIAL (3/4) to PASS (4/4).

**V-04 (C-29 + C-30 + C-31):** V-01 + V-02 combined. The two axes are structurally independent — gate verification checkpoints and GATE 3 boundary contracts do not interact.

**V-05 (C-29 + C-30 + C-31 + C-32):** Full combination. Predicted 32/32.
