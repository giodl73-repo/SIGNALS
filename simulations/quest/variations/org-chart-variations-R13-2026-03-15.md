## org-chart Variate — Round 13 Summary

**R13 target:** All five variations earn C-33 + C-34 + C-35 simultaneously → 27/27 (100.0).

**Three new criteria being combined:**
| Criterion | What it requires |
|-----------|-----------------|
| C-33 | FLAT-CASE-PRESSURE as directive-embedded inline backtick format string, with VERDICT referencing the rating |
| C-34 | Named ROLE-LOAD-ORDER block before ROLE-TYPE-CLASSIFICATION — pure imperative sequence constraint, no motivation sentence |
| C-35 | Quorum field as directive-enforced fifth charter element in inline backtick format string |

**Variations:**

| Variation | Axis | C-33 form | C-34 form | C-35 form | Expected |
|-----------|------|-----------|-----------|-----------|----------|
| **V-01** | Canonical synthesis | Standalone Sub-section 4 (R12 V-02 style) | Two-tier order (R12 V-03 style) | `[N roles]` format | 27/27 |

| **V-03** | Role sequence | Standalone Sub-section 4 | **Three-tier order: Engineering / Operations / all others** | `[N roles]` format | 27/27 |

**Key open questions for scoring:**
1. **C-33 / V-02**: Does "VERDICT carries the annotation" require embedding in VERDICT itself, or does a prior sub-section that VERDICT references satisfy the criterion? Both V-01 and V-02 should pass, but V-02 is the stricter interpretation.
2. **C-34 / V-03+V-04+V-05**: Does a three-tier ordering directive (Engineering / Operations / rest) satisfy C-34 equally with the original two-tier form? The criterion requires a "pure imperative sequence constraint" — tier count is not constrained.
3. **C-35 / V-05**: Does the rubric read C-35's format string as exact (`[N roles required...]`) or extensible? The extended `[N] of [M]` form is a richer format string that still satisfies the criterion's intent (directive-enforced fifth field) but changes the literal backtick template.
