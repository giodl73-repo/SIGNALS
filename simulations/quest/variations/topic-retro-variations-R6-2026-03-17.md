**R6 Variations — topic-retro**

Five complete variations targeting C-18 (echo-systemic-pattern-named) and C-19 (phase-completion-self-contained):

| ID | Axis | C-18 Mechanism | C-19 Mechanism |
|----|------|----------------|----------------|
| **V-01** | Output format | Pattern taxonomy column in Echo table; `[name] — [recurring failure description]` format; propagates to Phase 9 | Inherited from R5-V01 table structure |
| **V-02** | Lifecycle emphasis | Pattern entry required in Phase 1 Seal (`> **Pattern**: [name] — [desc]`) | Phase seal checklist per phase; exits blocked until all fields confirmed |
| **V-03** | Phrasing register | `[REQUIRED]` inline marker at Pattern field with format instruction | `[REQUIRED]` markers at every required field position |
| **V-04** | Role sequence | Dedicated Pattern Classifier role (Role 3) before Scorer; locked Pattern Classification as handoff artifact | Role handoff protocol — each role complete before next begins |
| **V-05** | Seals + inertia framing | Phase 0 seal requires Pattern entry; Phase 5 design propagates pattern | Seals per phase + baseline columns (two required outputs per row create double visibility) |

**Design decisions:**
- V-01 is the highest-confidence C-18 bet — column enforcement has the strongest R5 track record
- V-02 is the highest-confidence C-19 bet — seal checklists make incompleteness visible without requiring tabular content
- V-03 is the challenger: if it scores below V-01/V-02, that confirms structural enforcement outperforms inline markers
- V-04 tests whether role-boundary enforcement (R3-V04 pattern) extends cleanly to C-18
- V-05 tests whether combining two independent mechanisms outperforms either alone
