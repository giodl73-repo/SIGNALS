## corps-committee — R1 Variations

Five complete prompt bodies, single-axis first then combinations:

| ID | Axis | Core Bet |
|----|------|----------|
| **V-01** | Role sequence | Challenger-priority rule stated before all other instructions; Tier 1 voices get 3-4 sentences, advocates get 2. Tests whether foregrounding the ordering rule increases C-06 compliance. |
| **V-02** | Lifecycle emphasis | Each phase replaced with a compact `INPUT / OUTPUT / COMMIT` contract block instead of prose fill rules. Hypothesis: reduces content-after-commit failures (C-02) and INERTIA RECORD omissions (C-03). |
| **V-03** | Inertia framing | Phase 1 renamed "INERTIA INVESTIGATION," gains an opening "why inertia matters" paragraph, and the GATE check requires explicitly outside-in perspective. Bridge has a "Purpose" line. Hypothesis: increases C-03 finding quality (non-obvious D) and C-05 bridge accuracy. |
| **V-04** | Output format + phrasing register | STANCE MANIFEST and ACTION ITEMS become markdown tables; fill rules use conversational imperative ("you'll need to..."). Hypothesis: table format reduces TALLY-mismatch (C-07); conversational framing reduces missing action-item-field errors (C-04). |
| **V-05** | Role sequence + Inertia framing | Inertia-Advocate named as a "candidate participant" in Phase 0 roster note before the tier sort happens; injection announced as the default in a pre-execution RULE block. Targets the incorrect-YES affirmation failure on C-05 specifically. |

The key discriminator across variations is **C-05** (Inertia Continuity Bridge): V-01 and V-02 address it mechanically, V-03 and V-05 make it structural reasoning. V-04 is the neutral baseline on inertia but strongest on tally accuracy. Run all five against the same input and C-05 + C-07 are the likely differentiators.
