## Round 5 Design Summary

All 5 variations carry full C-01–C-20 coverage from Round 4. The discriminators are the three new v5 criteria:

| Variation | Axis | C-21 | C-22 | C-23 |
|-----------|------|------|------|------|
| **V-01** — All-Templates Declaration | Output format | ★ Every table | Delta step only | All 6 null outcomes |
| **V-02** — Active Checkpoint Cascade | Phrasing register | Proposals + diff + schema-commitment statements | Delta + proposals + diff (3 checkpoints) | All 6 null outcomes |
| **V-03** — Null Template Library | Lifecycle emphasis | Proposals + diff only | Delta step only | ★ All 6, individually labeled |
| **V-04** — Schema-First + Checkpoints | Role sequence + Output format | ★ Upfront schema + proposals | Delta + proposals | All 6 null outcomes |
| **V-05** — Full Enforcement Stack | Output format + Lifecycle + Inertia | ★ Every table | Delta step only | ★ All 6 null outcomes + inertia depth |

### Key structural bets

**V-01**: C-21 saturated — every table has the declaration, making partial compliance structurally visible by contrast.

**V-02**: The only variation with three independent auditable compliance checkpoints (delta, proposals, diff). A reviewer can confirm C-22 at three points without reading the full prompt.

**V-03**: The null templates are labeled individually under their own section headers ("NULL CASE — no ADD proposals:"), making them impossible to conflate. Weakest on C-21 by design — single-axis.

**V-04**: Schema-first priming separates the structural contract from execution. The model commits to all ★ columns before reading any artifact.

**V-05**: Full stack — same C-21/C-23 depth as V-01, plus inertia framing ("If unchanged") that forces causal tracing in Delta Finding entries, making C-20 entries substantive rather than mechanical ID fills.
