---

**What each variation isolates:**

**V-01** -- Removes only the reader-decision function framing from the non-substitutability declaration. Replaces `"the obligation tells you what to do; the existence condition tells you what you lose if you do not"` with activation/standing framing: `"one defines the activation precondition, the other is the consequence of failing to meet it."` Labeled sections and non-substitutability assertion stay intact (C-29 passes), but C-33's reader-function characterization fails.

**V-02** -- Keeps reader-function framing intact. Restructures ENFORCEMENT STACK NOTE from inline-label format (`Rule 1 (absence declaration) -- prevents silent omissions:`) to prose-sentence format (`Rule 1 addresses the absence-drift failure: outputs that appear complete...`). Failure classes are named (C-21 passes), but a reviewer must parse a sentence body to find the type, not scan a label (C-34 fails).

**V-03** -- Orthogonality test. Rule 3 heading changed from `RULES WITHOUT DECLARED SCOPE DO NOT EXIST` to `CONSTRAINTS ARE LOCATION-COMPLETE` (generic, C-28 fails). All body elements including reader-function framing are identical to R9 V-05. Confirms C-33 (body-level) and C-28 (heading-level) test different structural locations -- one can pass without the other in either direction.

**V-04** -- Both V-01 and V-02 changes applied simultaneously. Combined regression; confirms independent isolability.

**V-05** -- R9 V-05 verbatim. Already passes both C-33 (reader-function framing present) and C-34 (inline failure-class labels in ENFORCEMENT STACK NOTE). 26/26.

**Potential R11 signal:** the ENFORCEMENT STACK NOTE inline labels place failure-class names one level below the top-level rule headings. Candidate next criterion: failure class encoded directly in the top-level rule heading (e.g., `Rule 1 -- Absence Drift -- ABSENCE MUST BE DECLARED:`), making the diagnostic taxonomy recoverable from a top-level heading scan without entering the ENFORCEMENT STACK NOTE block.
: one can be satisfied without the other in either direction.

- **V-04** combines V-01 and V-02 changes simultaneously. The non-substitutability declaration uses the activation/standing framing (C-33 fails) and the ENFORCEMENT STACK NOTE uses prose-sentence failure classes without inline labels (C-34 fails). C-28 passes (ontological heading retained). Three prior single-axis slots are consumed by V-01/V-02/V-03; V-04 provides the combined regression confirming the two new criteria are independently isolable.

- **V-05** is R9 V-05 verbatim. The R9 ceiling already satisfies both C-33 (non-substitutability declaration with reader-function framing: "the obligation tells you what to do; the existence condition tells you what you lose if you do not") and C-34 (inline failure-class labels co-located with rule references in the ENFORCEMENT STACK NOTE: "Rule 1 (absence declaration) -- prevents silent omissions:", etc.). No new language needed. 26/26 under v10.

**Potential R11 signals from V-05:** the ENFORCEMENT STACK NOTE inline label structure places failure-class names one level removed from the top-level rule headings. A candidate next criterion: failure class encoded directly in the top-level rule heading (e.g., "Rule 1 -- Absence Drift -- ABSENCE MUST BE DECLARED:"), making the taxonomy recoverable from a top-level scan without entering the ENFORCEMENT STACK NOTE block. Also: whether the reader-function framing in the non-substitutability declaration should name the reader's position explicitly (e.g., "a committing engineer reading for what to fix" vs "a reviewer reading for what is already lost").

## V-01: C-33 FAIL -- Reader-function framing absent; non-substitutability declaration uses activation/standing framing

**Variation axis**: Reader-function framing (C-33)
**Hypothesis**: A Rule 3 that has "Obligation:" / "Existence condition:" labeled sections (C-29), a non-substitutability declaration that explains what each register governs in activation/standing terms (not reader-decision function terms), the ontological heading (C-28), step-up disclaimer (C-30), creation-time gate (C-31), forward-scope Applies-to (C-32), and inline failure-class labels in the ENFORCEMENT STACK NOTE (C-34) fails C-33 and only C-33. The declaration explains what each register does at the level of rule mechanics, not at the level of what the reader consults each for -- C-33's reader-decision function framing test fails. Scores 25/26 under v10. Confirms C-33 is an independent tightening of C-29: C-29's "explanation of what each distinctly conveys" is satisfiable without the specific reader-decision function mapping required by C-33.
