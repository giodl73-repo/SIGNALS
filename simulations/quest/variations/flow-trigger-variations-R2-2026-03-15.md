## Round 2 / Rubric v2 — 5 Variations

**What changed from the R2-v2 file**: The rubric's aspirational tier was revised — C-11 became "denominator declared before enumeration", C-12 became "gap tokens inline in sequence" (not vocabulary self-annotation), and N_aspirational grew to 5. These variations target the new C-11/C-12/C-13 specifically.

| # | Axis | Primary Criteria | Core Innovation |
|---|---|---|---|
| V-01 | Role sequence | C-11, C-13 | Scanner → Enumerator → Verdict Analyst. Two hard dependencies: Scanner locks denominator before Enumerator opens; Verdict Analyst contractually required to cite Role B entry numbers |
| V-02 | Output format | C-12, C-10 | Full-Roster Sequence: pre-allocates one numbered slot per candidate. Slots fill with firing entry or `[NOT FIRED — reason]` disposal token — silent omission is structurally impossible |
| V-03 | Phrasing register | C-13 | Evidence-Row-Verdict (ERV) three-part sub-structure: ROW RANGE → FINDING → VERDICT. A verdict without a preceding ROW RANGE is visibly incomplete; "all entries" without specific numbers is a weak pass |
| V-04 | Role sequence + output format | C-11, C-12 | C-ID-ordered sequence: Phase 1 declares denominator in Sync/Async groups; Phase 2 sequences by C-ID order producing exactly N primary slots, each filled or disposed |
| V-05 | Phrasing register + inertia framing | C-11, C-12, C-13 | Names Failure Modes 7/8/9 — Denominator Post-Declaration, Silent Candidate Omission, Citation-Free Verdict — each with a formal SHALL requirement and its inline detection tag |

**Key structural pattern across all five**: V-01/V-04 use role/phase separation to make the pre-scan a structural precondition for enumeration (C-11). V-02/V-04 make disposal tokens fill mandatory pre-allocated slots (C-12). V-01/V-03/V-05 make row-level citation a named required field rather than an instruction (C-13).
