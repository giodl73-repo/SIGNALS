## V-01: C-41 FAIL -- Count annotation completeness; NOTE drops ownership attribution

**Variation axis**: Consulting directory completeness (C-42) without NOTE ownership attribution (C-41)
**Hypothesis**: A STANDING RULES block where the consulting directory carries an explicit
count annotation -- "3 reader roles governed by this block:" -- making role coverage verifiable
from the directory alone (C-42 PASS), but where the ENFORCEMENT STACK NOTE is a pure
interdependency statement that omits the ownership-attribution phrase ("failure class encoded in
each rule's heading above") -- the NOTE states only that each rule addresses an independent failure
mode without attributing failure-mode ownership to the headings (C-41 FAIL). A reader scanning
only the NOTE encounters the interdependency claim but cannot determine from the NOTE alone why
no failure-mode description is present; they must read the headings first to infer it.
C-42 PASS, C-41 FAIL. Confirms C-41 is not discharged by C-39 alone: a pure interdependency
NOTE can omit the ownership attribution while still containing no "prevents..." prose.
Scores 33/34.
