### Key design decisions

**Why V-01 intentionally omits C-26:** Clean single-axis test for RULES-ANCHORED-COMPLETE. If it scores 180 (not 185), the R8 isolated target does not elevate the composite — it only seeds the v9 candidate. This confirms the token's role as a v9 criterion, not a v8 scoring factor.

**Why V-02 targets C-25 via phrasing register:** The RESOLUTION PROTOCOL is a named specification within PHASE 3B. Making it a standalone headed declaration block (same visual register as GATEKEEPER FUNCTION DECLARATION) tests whether visual distinctness is what makes C-25 reliable.

**Why V-03 targets C-26 via role sequence:** The EF role is role-scope-excluded from signal artifacts, making the NO-ECHO COST claim structurally prospective — not instructed. This is a stronger C-26 mechanism than inline addition because the role boundary prevents the cost declaration from being constructed retrospectively from entry content.

**RULES-ANCHORED-COMPLETE pattern:** Mirrors C-19/CHAIN-COMPLETE exactly. C-23 blocks on failure; RULES-ANCHORED-COMPLETE confirms success. The token makes distillation-phase closure auditable from output — seeding C-28 for v9.
