### Key structural differences across variations

**C-55 (PHASE markers)** — V-01, V-04, V-05:
Every block preceded by `PHASE A: SIGNAL CATALOGUE` through `PHASE I: AMEND PATHWAY`. Structurally distinct from SEALED gates (entry vs. exit). A block with PHASE and no SEALED fails C-46 and passes C-55; a block with SEALED and no PHASE passes C-46 and fails C-55.

**C-54 (INERTIA REGISTER table)** — V-02, V-04, V-05:
F-30's inertia note appears as a 4-column structural table in BLOCK 2.7: `F-Code | Prior Form | Current Scope | Why Both Non-Conformant`. The V-02/V-04 form is a single table row; V-05 uses a named INERTIA REGISTER table with a `SHALL NOT revert` modal clause. V-01 and V-03 retain inline prose (C-52 PASS, C-54 FAIL).

**Role sequence** — V-03, V-05:
BLOCK 2 generates domain expert finding blocks before stock disciplines. V-05 also reorders BLOCK 1.5 to list Domain reviewers first (matching the BLOCK 2 generation sequence).

**C-49 (CONSENSUS ELEVATION RULE full enumeration)** — V-05 only:
Three-level explicit algorithm with numbered rank ranges (ELEVATED tier: ranks 1–ELEVATED_count; BASELINE tier: ranks ELEVATED_count+1–P1_count), sort keys (Amendment Cost then reviewer count), tiebreaker (alphabetical Section), and explicit non-conformance prohibition ("re-assessing Consensus Status at BLOCK 5 generation time is non-conformant").
