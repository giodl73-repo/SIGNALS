## Round 2 — org-review Variations

**Axis coverage summary** — Round 1 exhausted: inertia-first sequence, strict table, null-hypothesis-universal, formal register + lifecycle, parallel blind + prose. Round 2 targets:

| | Axis | Key criterion targeted |
|--|------|----------------------|
| **V-01** | Output format: multi-dimension scorecard | C-03, C-09 (score-level divergence) |
| **V-02** | Lifecycle emphasis: four explicit gates | C-05, C-11, C-12 (gate as natural disposition aggregator) |
| **V-03** | Inertia framing: challenger-last / rebuttal | C-09, C-12 (targeted rebuttal vs. baseline-setting) |
| **V-04** | Combination: inertia bracket + per-role null verdict | C-12 structural (bracket forces it), C-09 (overturn conflict) |
| **V-05** | Combination: verdict cards + disposition matrix | C-11 structural (computed, not editorial), C-09 (matrix-visible conflicts) |

---

**Key design decisions:**

**V-01** separates `severity` into three scored dimensions — Decision-Readiness, Technical-Soundness, Inertia-Risk — each owned by a different archetype. Cross-role score divergence on the same dimension surfaces C-09 conflicts numerically before they appear in prose.

**V-02** uses a gate metaphor (Null Hypothesis → Domain Evidence → Commitment Readiness → Synthesis). Each reviewer is assigned to a lifecycle phase rather than just an artifact type. C-11 disposition = count of gate pass/fail; C-12 = mandatory Gate 1 output that every later gate must reference.

**V-03** tests the opposite of R1 V-01: inertia runs last as a rebuttal, not first as a baseline. Domain reviewers state a Build/Don't-Build stance; inertia then evaluates whether those stances collectively defeat the null hypothesis. Conflict detection (C-09) is structural — inertia's overturn of domain consensus is named explicitly.

**V-04** combines a bracket (inertia Round 1 → domain reviewers → inertia Round 2) with a required null hypothesis address from each domain reviewer. The closing bracket produces a DEFEATED / PARTIAL DEFEAT / HOLDS verdict, making C-12 the structural backbone of the entire review rather than a per-reviewer addition.

**V-05** replaces severity with a per-reviewer BUILD / CONDITIONAL / BLOCK verdict card. The disposition code (C-11) is computed from the cards — BLOCKED if any card is BLOCK, CONDITIONAL if any is CONDITIONAL and none are BLOCK, READY if all are BUILD. Conflicts appear in the matrix as opposing verdicts (C-09) rather than in prose, with a tiebreaker rule giving the inertia-advocate BLOCK precedence over domain BUILD.
