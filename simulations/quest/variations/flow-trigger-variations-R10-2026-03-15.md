## flow-trigger Round 10 — 5 Variations (Rubric v7, C-25 + C-26)

### Variation axes

| Variation | Axis | C-25 Mechanism | C-26 Mechanism |
|-----------|------|----------------|----------------|
| **V-01** | Role sequence | Phase 0 as a first-class sealed role before the Auditor; EXIT SIGNAL is an inter-role gate | INERTIA CONTRAST section in Auditor role — two mechanisms contrasted (ART-02 prose refs vs. ART-IDs; EOR global vs. per-entry) |
| **V-02** | Output format | Phase 0 Exit Condition Registry table — Status ∈ {SATISFIED \| PENDING \| BLOCKED}; EXIT SIGNAL is computed from table aggregate (5/5) | INERTIA CONTRAST table in Phase 0 — two mechanisms: EC Status enum vs. tick-boxes; ART-IDs vs. prose headings |
| **V-03** | Phrasing register | Phase 0 as governing law Articles 0.1–0.5 each with SHALL/MUST and a named refutation condition | INERTIA CONTRAST as dedicated table after Article set — govering articles cited; mechanism vs. alternative vs. failure mode |
| **V-04** | Role sequence + Lifecycle emphasis | Phase 0 sub-steps 0.1–0.5 each with explicit "What / Why / How" triad; EXIT SIGNAL after all five sub-step tokens | INERTIA CONTRAST is embedded in each sub-step "Why it is required (Risk Prevented)" field — C-26 satisfied as Phase 0 byproduct, no separate section |
| **V-05** | Output format + Inertia framing | Phase 0 Exit Condition Registry is a 6-column table adding `Inertia-Driven Alternative` and `Failure Mode` columns inline; EXIT SIGNAL cites "Inertia Contrast: 5/5 satisfied" | C-25 and C-26 collapse into a single artifact — the registry is simultaneously a lifecycle gate and an embedded rationale document |

### Key structural advances over R9

- **C-25** (Phase 0 gate): All five variations declare a named pre-enumeration lifecycle phase with typed exit conditions. V-02 and V-05 use a typed Status enum column; V-03 uses governing law article refutation conditions; V-04 uses per-sub-step exit tokens. Every variation issues an explicit `PHASE 0 EXIT SIGNAL` statement before enumeration.
- **C-26** (INERTIA CONTRAST): V-01–V-03 use a standalone named section; V-04 distributes rationale into Phase 0 sub-step prose ("Why it is required"); V-05 embeds rationale as columns in the Exit Condition Registry itself. V-05 is the most compressed form — the table is both the gate and the rationale.
