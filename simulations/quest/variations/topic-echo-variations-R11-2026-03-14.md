## Variation Summary

**V-01 — Single-axis: Phrasing register (C-26)**
- **Axis:** CONSISTENCY GATE names vocabulary constraint class per archetype — `PREREQUISITE → input-dependency`, `ESTABLISHES → property creation`, `AMPLIFIES → effectiveness scaling`, `PARALLEL → independent paths` — converting the gate from an author-judgment call to a binary vocabulary check.
- **Pair format:** Three steps (Archetype → Vocabulary-Gate → Mechanism). No Composability-claim (C-27 FAIL). No pre-populated entry (C-28 FAIL).
- **Hypothesis:** C-26 is a property of gate instruction text, not field structure. Naming constraint classes passes C-26 without requiring C-27 or C-28.

**V-02 — Single-axis: Output format (C-27)**
- **Axis:** Four-field pair inspection (Archetype → Gate → Mechanism → Composability-claim). `Mechanism` = analytical causal description. `Composability-claim` = synthetic composability assertion. Two independently evaluable fields; merged field hides inconsistency, separated field exposes it.
- **Gate:** CONSISTENCY GATE present but vocabulary-free — "confirm mechanism matches archetype" (C-26 FAIL). No pre-populated entry (C-28 FAIL).
- **Hypothesis:** C-27 is a property of field structure, not gate vocabulary or pre-population. Field separation passes C-27 without C-26 or C-28.

**V-03 — Single-axis: Lifecycle emphasis (C-28)**
- **Axis:** Manifest template carries a fully pre-completed NGT+Check B baseline entry — all fields pre-filled (archetype=PREREQUISITE, gate="input-dependency — PASS", mechanism=full causal statement). Author inherits it; does not re-derive. Subsequent pairs use blank templates.
- **Gate:** CONSISTENCY GATE present, no vocabulary (C-26 FAIL). Three-step output: Archetype + Gate + Mechanism; no Composability-claim (C-27 FAIL).
- **Hypothesis:** C-28 is a manifest-template property — what is present before the author writes — orthogonal to gate vocabulary and field structure.

**V-04 — Combined: Phrasing register + Output format (C-26 + C-27)**
- Vocabulary-specified gate (C-26) + four-field sequence with Composability-claim (C-27). No pre-populated entry (C-28 FAIL).
- Confirms C-26 and C-27 are jointly stable: vocabulary gate tests constraint class before mechanism is stated; Composability-claim builds on verified mechanism as a distinct synthetic layer.

**V-05 — Combined: Lifecycle emphasis + Inertia framing (C-26 + C-27 + C-28 + inertia)**
- V-04 baseline + pre-populated four-field NGT+Check B entry (C-28) + opening INERTIA CONTEXT block. The pre-populated entry demonstrates all four fields at quality-floor level. Inertia framing converts Rule 1's per-surprise counterfactual gate into a document-level pre-write commitment.
