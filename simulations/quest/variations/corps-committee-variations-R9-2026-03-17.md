## corps-committee -- Prompt Variations R9

### Variation Summary

| ID | Axis | Hypothesis |
|----|------|------------|
| **V-01** | Cross-block dependency graph (C-28) | A BLOCK DEPENDENCY MAP that declares inter-block consumption obligations before any block executes makes B4→B2 and B2→B3 consistency verifiable by label comparison. |
| **V-02** | End-to-end token trace (C-29) | A TOKEN TRACE MANIFEST at the Phase 1→2 gate tracks each INERTIA-FINDING-N through all six consumption sites; any dropped token is a missing cell, not invisible prose. |
| **V-03** | AMEND block-level routing (C-30) | An AMEND ROUTING TABLE maps each amendment type to specific blocks to invalidate (participant change → B2+B4; type change → B1+B2) before any phase re-execution. |
| **V-04** | C-28 + C-29 combined | Block graph + token trace as a two-phase self-auditing system: block-level consistency before simulation, token-level chain at the Phase 1→2 gate. |
| **V-05** | Full integration | C-28 + C-29 + C-30 + R8 best (vocabulary contract, tier-sequenced voice, per-tier column grammar, inertia-to-dissent anchor, Challenger block, table-first grammar). |

### Key structural additions vs R8

The three new R9 criteria required adding:

1. **BLOCK DEPENDENCY MAP** (C-28) — a table declaring which blocks consume which other blocks' outputs, with cross-block checks before each block runs. Block 4 Challenger tier must match Block 2 T1 entry; Block 2 T1 Inertia Citation must reference a Block 3 token.

2. **TOKEN TRACE MANIFEST** (C-29) — a gate table at Phase 1→2 boundary tracking each INERTIA-FINDING-N through: B3 origin → T1 cite → CHAIN-3 draft → Phase 2 pre-mortem expansion → Dissent Collection → Dissenting Opinions. A `MISSING` cell halts Phase 2.

3. **AMEND ROUTING TABLE** (C-30) — explicit mapping of amendment types to block invalidation sets before phase re-execution. `AMEND-BLOCKS-INVALIDATED:` declaration required on every AMEND invocation.
