## Variation Design Summary

**What R16 targets** (3 new v13 criteria):

| Criterion | What it upgrades from R15 | Key structural requirement |
|-----------|--------------------------|---------------------------|
| **C-29** | C-28 prose bullets → named append-log table | T-07 pre-declared in scaffold; rows appended *during* trace sequence, not post-trace |
| **C-30** | C-22 inline domination note → standalone table | Separate T-08/T-06 entry in scaffold; `→ Domination analysis: see T-NN` footnote in boundary inventory |
| **C-31** | C-16 step-citation column → Recovery Framing column | Template `Revert to [IB-NN] performed by [Role] at [freq]` — all 3 fields in ONE cell |

**Variation axes:**

| Var | Axis | Hypothesis |
|-----|------|-----------|
| **V-01** | Output format — scaffold column-spec mandate | Declaring exact column names prevents prose substitution before content begins |
| **V-02** | Lifecycle emphasis — per-stage T-07 APPEND CHECKPOINT | Explicit count declaration between stages enforces accumulation, not batch write |
| **V-03** | Phrasing register — imperative commands + DO NOT prohibitions | Naming the failure patterns before the model reaches the decision point eliminates ambiguity |
| **V-04** | Role sequence + lifecycle — Finance first, handoff gate audits T-07 | Role transfer condition makes append-log completeness a named, checkable artifact |
| **V-05** | Inertia framing + output format — baseline-first, Recovery Framing as lead column | IB-NN identifiers established before any recovery row needs them; column ordering creates gravitational pressure toward C-31 |
