# topic-open — Round 6 Variations

**Rubric**: v6 (C-01–C-27, denominator 19)
**Round context**: R5 all 5 variations scored on v5 rubric. Three new criteria: C-25 (tool citations at every exit gate condition and self-check item), C-26 (comparative framing on every ordering instruction universally), C-27 (labeled decision tables at all five phase transitions).

---

## Variation Axes

| Variation | Axis | Hypothesis |
|-----------|------|------------|
| V-01 | C-25 single — exit gate and checklist tool citation at maximum depth | Every gate condition and every checklist item names the enforcement tool and states precisely what it checks; the model cannot execute any verification step without knowing the downstream observer |
| V-02 | C-26 single — comparative framing universally on every ordering instruction | Every "do X before Y" — ROW TEMPLATE copy-vs-reconstruct, prose-before-table, self-check timing — uses explicit if/if contrast in the instruction body, not just in decision tables |
| V-03 | C-27 single — labeled decision tables at all five phase transitions | Five phases, five named decision tables (Collision Check, Content Order, Gate Structure, Output Write, Check Timing) with consistent Path/Action/Consequence columns, each appearing before its phase instruction body |
| V-04 | C-25 + C-26 combination | Gate/checklist tool citations AND universal ordering instruction framing simultaneously — every enforcement point names its tool, every ordering choice has both paths described |
| V-05 | C-25 + C-26 + C-27 full integration | All three new criteria at maximum depth, plus C-24 unified constraint matrix — five phase decision tables + gate/checklist citations + universal if/if framing operating together |

---

## What Distinguishes Each Variation from R5

**V-01 vs. R5 V-01**: R5 V-01 annotated Phase 2 exit gate items and Phase 5 checklist items but not all items were equally specific (some were bare tool names without what-the-tool-checks). V-01 here pushes every gate item and every checklist item to full citation depth — tool name + what it checks for + what happens when the check fails.

**V-02 vs. R5 V-02**: R5 V-02 achieved C-22 (vocab block's own reading-order uses if/if) and had if/if framing in Phase 2 and Phase 5 prose. C-26 requires the ROW TEMPLATE's copy-vs-reconstruct instruction itself to use explicit if/if framing in the instruction body — not just as a decision table. V-02 here adds if/if to the ROW TEMPLATE block directly.

**V-03 vs. R5 V-03**: R5 V-03 already had all five phase decision tables — this is C-27's direct achievement. V-03 here deepens the table contents: every table's Consequence column now includes the specific tool-level outcome (e.g., "`topic-status` loads TOPICS.md on every check; a duplicate creates ambiguous aggregation with no merge path") rather than generic outcomes.

**V-04**: New combination not present in R5 — C-25 + C-26 together. R5 V-04 combined C-21 + C-22. V-04 here combines C-25 + C-26 (the next generation of those same axes).

**V-05**: Full integration of all three new criteria, the most structurally demanding combination. Includes C-24 (unified five-column DEFAULTS table) to achieve constraint-surface compactness simultaneously with the three new criteria.
