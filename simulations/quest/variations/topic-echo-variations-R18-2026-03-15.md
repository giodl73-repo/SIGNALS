## Round 18 — Design Summary

**Three new criteria under test:**

| Criterion | What it tests | Where it lives |
|-----------|--------------|----------------|
| **C-54** | Canonical name at enforcement sites (Stage 4 inline instruction) | Stage 4 body |
| **C-55** | Explicit "names the stage, not merely describes routing" contrast | Intro commitment 3 |
| **C-56** | Scoped cross-section mandate naming failure modes, verdicts, section notes | Standalone Collapse Prohibition block |

**Five variations:**

| Variation | Axis | C-54 | C-55 | C-56 | Notes |
|-----------|------|:----:|:----:|:----:|-------|
| V-01 | Role sequence | YES | no | no | Stage 4 cites canonical name |
| V-02 | Output format | no | YES | no | Intro commitment 3 explicit contrast |
| V-03 | Lifecycle emphasis | no | no | YES | Definition block scoped mandate |
| V-04 | Role sequence + output format | YES | YES | no | Combined, no C-56 |
| V-05 | Full integration + R19 push | YES | YES | YES | Adds: COMMIT-STAGE-4 constraint citation, GATE-CONFIRMED proper-noun stage token |

**Key distinctions between variations:**

- **C-54 present vs absent**: Stage 4 instruction ends with "The Standalone Collapse Prohibition applies: each row is committed before the next begins." (C-54) vs. just "a COMMIT-STAGE-4 does not substitute." (absent)
- **C-55 present vs absent**: Intro commitment 3 says "the token names the stage, not merely describes the routing action" (C-55) vs. "naming its receiving stage before Stage 4 begins" (absent — C-53 satisfied but not C-55)
- **C-56 present vs absent**: Prohibition block says "any failure mode, verdict, or section note in this prompt that references this prohibition is citing the Standalone Collapse Prohibition by name" (C-56, scoped) vs. "so it can be cited by name throughout this prompt" (absent, unscoped)

**R19 candidates in V-05:**
- **C-57 candidate**: COMMIT-STAGE-4 token carries canonical constraint name: `COMMIT-STAGE-4: all entries committed per the Standalone Collapse Prohibition.`
- **C-58 candidate**: GATE-CONFIRMED token delivers stage name as standalone proper noun: `GATE-CONFIRMED-[N]: VALID — Stage 4.` (not embedded in behavioral routing clause)
