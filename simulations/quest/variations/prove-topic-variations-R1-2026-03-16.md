### V-01 — Lifecycle Emphasis
Explicit `GATE-N CHECK` block before every stage with `BLOCKED` emit when precondition unmet. `SCOUT CONFIRMED` must be emitted before Stage 1 opens. Minimal prose around the gate machinery.

### V-02 — Output Format
Path-declaration table at campaign open (all 5 artifact paths declared upfront). Each stage has a frontmatter requirements table. Stages reference "precondition: prior stage artifact written" rather than gate emit strings.

### V-03 — Phrasing Register
Conversational register throughout — "you're running", "in this stage you'll", no CAPS section labels. Tests whether soft register still produces per-stage writes with full paths and explicit scout grounding.

### V-04 — Inertia Framing
`CAMPAIGN OPEN` emits `status_quo` + `inertia` fields extracted from scout artifact. Hypothesis is a displacement claim template. THIN flags name the displacement claim weakened. Synthesis states displacement verdict with adjusted confidence table.

### V-05 — Combined
Combines V-01's gate checks + V-02's path declaration table + V-04's CAMPAIGN OPEN with status_quo/inertia. The integration candidate for scoring: compact (no 37-criteria overhead), but structurally complete for all 10 v14 criteria.
