### V-01 — Output Format: YAML-First, Inline Annotation

**Hypothesis**: Producing the `org.yaml` block immediately makes C-01 unfailable by design — the model can't promise YAML "below." Inline `# detected from:` comments co-locate C-09 with the data. Three explicit amend options at the end cover C-08 mechanically.

**Strongest on**: C-01, C-05 (stated up front as a hard rule), C-09

### V-02 — Phrasing Register: Conversational + Descriptive

**Hypothesis**: A think-aloud register encourages the model to narrate detection reasoning naturally, which produces C-09 and C-10 without requiring explicit per-criterion instructions. The C-05 boundary is front-loaded as a named constraint, then the register relaxes.

**Strongest on**: C-09, C-10 (Inertia Advocate note emerges from narrative framing)

### V-03 — Lifecycle Emphasis: Explicit Phase Gates

**Hypothesis**: Four named phases (SCAN → PIVOT → DRAFT → AMEND), each with a gate confirmation statement, prevent the skip-to-output failure that causes C-02 misses. DRAFT GATE states the draft-only constraint as an entry condition before any YAML is written.

**Strongest on**: C-02, C-05 (as a gate condition), C-06

### V-04 — Role Sequence + Inertia Framing (Combination)

**Hypothesis**: Repo Archaeologist role forces C-02 grounding by producing an inventory the drafter must draw from. Status-Quo Challenger names the stagnation pattern and primes C-10 naturally. Anti-inertia amend options address the specific pattern found, making C-08 the richest of any variation.

**Strongest on**: C-02, C-08 (anti-inertia amend framing), C-09, C-10

### V-05 — Output Format + Lifecycle Emphasis (Combination)

**Strongest on**: C-01, C-03, C-07, C-08 — all essential/recommended criteria present by construction
