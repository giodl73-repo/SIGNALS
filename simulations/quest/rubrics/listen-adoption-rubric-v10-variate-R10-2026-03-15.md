## V-01 — Single Axis: Output Format (C-28 Targeted)

**Variation axis:** Output format — dedicated TABLE 3 (Phase Transition Spread Mechanisms)
inserted after PART 2, with column header "Spread Mechanism: [named signal or artifact — not
generic word-of-mouth]" embedded at generation time. The constraint is in the label; the model
cannot write "word-of-mouth" without violating the slot's own type declaration.

**Hypothesis:** Placing the specificity constraint in the column header of a structural table
makes C-28 pass structurally — no separate instruction is needed and no downstream audit can
catch what the label already excludes. C-07 also passes because the table forces one mechanism
per row per transition. All other R10 new criteria (C-29, C-30) remain at V-04-R9 risk level.

## V-02 — Single Axis: Phrasing Register (C-29 Targeted)

**Variation axis:** Phrasing register — phase headers and champion fields use answer-form
questions for inertia ID citation. "Which Named Inertia ID from SECTION B is driving the chasm?"
is a question whose answer IS the inertia ID; omitting the ID leaves the question unanswered,
which is syntactically incomplete rather than a compliance miss. Replaces V-04-R9 fill-form
fields ("cite I-0X from SECTION B") at the two key locations: PHASE 3 chasm header and PHASE 2
champion entries.

**Hypothesis:** Answer-form phase-header questions make inertia ID omission structurally
incompatible with answering the question at all — C-29 passes structurally. C-13 and C-14 also
benefit: because the ID is the answer to the question, not a value in a slot, the model must
produce a valid ID to satisfy the phase body's own grammar. C-28 and C-30 remain at V-04-R9
risk level (no TABLE 3, no execution stamps).
