Three new criteria added:

**C-20 — UPFRONT FORMAT CONTRACT** (from V-05 Pattern 1): All five table schemas consolidated before the first phase gate. Eliminates schema-recall errors when column names were specified 1,500+ tokens earlier. The structural insight is that every schema violation is a recall failure, not a comprehension failure.

**C-21 — ROLE-NAME LOCK with inline CHECKs** (from V-05 Pattern 2): Extends C-13 from a one-time declaration into an active per-section check. The LOCK declares the contract; the inline CHECKs enforce it at each use site. Without CHECKs, the LOCK is read once and relied on across 2,000+ tokens — the memory-decay failure mode remains open.

**C-22 — Vocabulary-lock + VERIFY co-deployment** (from V-05 Pattern 3): Every vocabulary-governed table gets both a lock (blocks wrong-type values) and a VERIFY (blocks right-type-wrong-structure evasions). These target non-overlapping failure modes and must be deployed together. Also extends the Quorum VERIFY coverage that appeared as "bonus" in V-02.
