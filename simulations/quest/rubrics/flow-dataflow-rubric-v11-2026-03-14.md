---

**Three new criteria added. The progression:**

- **C-36** closes the gap where C-34 requires the baseline token to be cited in every `Incumbent Equiv.` cell but imposes no sequencing constraint on when the baseline artifact is produced. Without C-36, a syntactically correct `Incumbent Equiv.` cell may forward-reference a baseline not yet written at that point in the trace — passing C-34 by form while violating it by production order. With C-36, the baseline is artifact-zero: it is produced before any boundary block can cite it, making every citation structurally complete rather than merely formally complete.

- **C-37** closes the gap where C-34 and C-35 compliance requirements are distributed across BOUNDARY BLOCK SCHEMA and role instructions, requiring an evaluator to cross-reference multiple sections. Without C-37, a prompt can satisfy C-30 and C-32 (register declared, fields mapped) while leaving the C-34/C-35 verification specs scattered. With C-37, the REGISTER DECLARATION contains canonical Parts A and B as the single authoritative source — constraint callbacks point to these Parts, and an evaluator can verify both criteria by reading one section.

- **C-38** closes the gap where C-26 requires non-natural role ordering but does not require any role to skip its immediate predecessor in the sequence. Without C-38, a non-natural ordering can be satisfied by each role citing only the role that precedes it locally — still allowing context to be inherited from adjacent proximity rather than traced. With C-38, the prompt must designate at least one role whose citation must reach back two or more positions, and the phase gate checklist must name the skip-level token explicitly — making citation-gap failures visible as a missing token rather than a prose judgment call.
