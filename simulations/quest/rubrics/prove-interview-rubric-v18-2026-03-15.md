# prove-interview Rubric — v18

`(E/5 × 60) + (R/3 × 30) + (A/46 × 10)` | PARTIAL = 0.5

**v18 changes — three new criteria extracted from Round 19 (V-05):**

| ID | Pattern | Source | Pass Condition |
|----|---------|--------|----------------|
| **C-49** | Chain-critical gate items ordered first in gate block | V-05 Phase 2 + Phase 3 gates | First item in each gate block is the condition whose violation breaks a cross-phase chain — before affirmative completion conditions |
| **C-50** | Propagating gate declares downstream obligations at source | V-05 Phase 0C exit gate | Source gate includes a "downstream propagation obligations" block naming each downstream phase and the specific constraint it must satisfy |
| **C-51** | Contract source named at point of contract declaration | V-05 Phase 0A exit gate vocabulary block | Gate establishing a vocabulary or constraint names itself as the authoritative source and warns against parallel contract creation downstream |

**Denominator: 43 → 46.** Formula: `(E/5 × 60) + (R/3 × 30) + (A/46 × 10)`.

C-49 extends C-32 (which tests gate format) by testing ordering within the gate block — chain-critical items first makes the chain-breaking condition auditable at gate entry without reading through affirmative items. C-50 is the source-side companion to C-30 (which tests the receiving end): declaring obligations at the origin makes propagation traceable from source outward, not just inward. C-51 addresses a distinct failure mode from C-33 (document-head format contract): a gate can establish a vocabulary, distribute a copy downstream, and create a silent parallel contract unless the source gate names itself as the authority.
