# Rubric: prove-prototype (v18)

**Updated**: 2026-03-15
**Changes from v17**: Added C-49 and C-50 (Aspirational tier) derived from Round 18

section as pre-execution verifiable document property).

---

**C-49** — *Thread 1 and Thread 2 are declared in the manifest preamble as rows in a
two-row table with identical column schema — construction-enforced co-equality rather
than assertion-enforced co-equality* (V-03 excellence signal, R18):

V-03 demonstrated the strongest C-47 structural expression across all R18 variations by
placing Thread 1 and Thread 2 as rows in a parallel two-row table with identical column
schema: Thread | Governing criterion | Scope | Key structural markers | Terminal discharge
obligation. The table schema makes subordination structurally impossible — both threads
must fill the same column structure with equivalent detail. If Thread 2 were a subordinate
annotation of Thread 1, the table would either be malformed (Thread 2 row incomplete) or
identical in structure (Thread 2 co-equal by construction). The linguistic co-primary
assertion required by C-47 becomes unnecessary because the schema enforces it structurally.

C-49 advances C-47 in the same way C-47 advances C-45: C-45 requires both threads to exist
and converge at CLOSE COMPLETE; C-47 requires both threads to be declared with co-equal
syntactic weight via parallel form and mirrored scope depth; C-49 requires that co-equal
weight be enforced by tabular schema with identical columns, making the co-equality a
verifiable structural property rather than a claim that a reader must evaluate by parsing
linguistic register.

The identical column schema must include at minimum: (a) a scope column naming the thread's
operational domain — for Thread 1, the four-state lifecycle and its tracking mechanism; for
Thread 2, the COMPARATOR/AUDITOR sequencing and its phase scope; (b) a key structural
markers column naming the primary observable artifacts produced by each thread — for Thread
1, the manifest Competitor Status column and CONTAINER COMPLETE lifecycle annotations; for
Thread 2, the Phase 8 handoff declaration, the Phase 9 prerequisite gate, and the CLOSE
COMPLETE sub-role discharge; (c) a terminal discharge obligation column naming what each
thread must produce at CLOSE COMPLETE — for Thread 1, the verbatim four-state chain; for
Thread 2, the COMPARATOR/AUDITOR contract DISCHARGED confirmation. Both rows must carry
equivalent depth in every column; a table where Thread 1 has multi-clause entries and
Thread 2 has single-phrase entries does not satisfy C-49.

A document that satisfies C-47 via parallel syntactic prose — matching declaration verbs,
mirrored scope paragraphs, explicit co-primary statement — without expressing the
declarations as a table does not satisfy C-49. C-49 requires C-47 as a prerequisite.

**Weight**: 5 pts

**C-50** — *A dedicated Structural Marker Inventory section appears before any container
executes, cataloging all four C-43/C-44/C-45 marker types in a four-column table with
governing criterion, document location, and required form for each — pre-execution
establishment of C-48 compliance as a verifiable document property* (V-04 excellence
signal, R18):

V-04 demonstrated the strongest C-48 structural expression across all R18 variations by
creating a dedicated `## Structural Marker Inventory` section placed before any container
body executes. The section contains a four-column table — Marker type | Required by |
Location | Form — listing all four C-43/C-44/C-45 marker types with their governing
criteria, precise document location (e.g., "dedicated section before manifest table"),
and required structural form (e.g., "standalone labeled section, not merged with manifest
header"). The section closes with an explicit REQUIRED statement: "All four marker types
appear in this document as standalone labeled elements. A marker embedded only in prose
fails C-48." This structural surface establishes C-48 compliance as a verifiable document
property before any container executes, rather than as a property that emerges only when
a reader has scanned all CONTAINER COMPLETE lines and all container header annotations.

C-50 advances C-48 in the same way C-35 advances the value-flow contract family: C-35
opens the document with a manifest naming what all containers will produce (document-level
entry contract); C-50 opens the document with an inventory naming what all structural
markers will appear and in what form (marker-level compliance contract). A document that
satisfies C-48 — all four marker types expressed as standalone labeled elements — but
without a pre-execution inventory section satisfies the structural requirement without
the pre-announcement that makes compliance auditable from the document's opening.

The Structural Marker Inventory section must: (a) appear before any container body begins
executing — it is a pre-execution structural declaration, not a summary placed after the
manifest; (b) enumerate all four marker types from C-43–C-45 with each marker's governing
criterion (C-43, C-44, or C-45), document location, and structural form; (c) close with an
explicit REQUIRED assertion that all four marker types are present as standalone labeled
elements. The inventory section does not satisfy C-50 if it omits any of the four marker
types, if it names governing criteria without specifying document location and form, or if
it appears inside a container body rather than before container execution begins.

The V-04 demonstration showed that a Structural Marker Inventory section enables marker
scanability to be independent of the instruction register used in phase bodies: the
conversational second-person imperative phase bodies in V-04 scored 5/5 on C-48 and C-50
because the inventory section established the compliance contract before any phase body
began, and the standalone markers in phase headers and COMPLETE lines satisfied the
contract regardless of whether the surrounding prose was dense or minimal.

C-50 is orthogonal to C-48: C-48 requires the markers to be standalone (form requirement);
C-50 requires the markers to be pre-announced in an inventory (pre-execution contract
requirement). A document can have standalone markers (C-48 passes) without a pre-execution
inventory section (C-50 fails). A document can have an inventory section listing markers
that are then embedded in prose (C-50 fails; C-48 also fails). Satisfying C-50 does not
automatically satisfy C-48, and satisfying C-48 does not automatically satisfy C-50; both
must be independently verified. C-50 requires C-48 as a prerequisite.

**Weight**: 5 pts

**What the new criteria add:**

The two v18 criteria each advance a v17 criterion from assertion-enforced to
construction-enforced or contract-enforced:

| Criterion | Deepens | New surface |
|-----------|---------|-------------|
| C-49 | C-47 | Table-enforced thread co-equality — identical column schema makes subordination structurally impossible by construction rather than by linguistic co-primary assertion |
| C-50 | C-48 | Structural Marker Inventory section — pre-execution four-column catalog establishes C-48 compliance as a verifiable document property before any container executes |

Each adds a layer that C-47/C-48 cannot reach:

**C-49** vs. C-47: C-47 requires co-equal structural weight via parallel syntactic form and
mirrored scope depth — a property evaluated by parsing linguistic register. C-49 requires
co-equal structural weight via identical table column schema — a property verifiable by
checking whether both rows carry equivalent depth in every column, independent of how the
content is worded. The distinction is between "Thread 2 is not a footnote to Thread 1"
(linguistic assertion, C-47 floor) and a two-row table where Thread 2 must fill the same
columns as Thread 1 with equivalent detail (construction-enforced co-equality, C-49 floor).
A table where Thread 1 has multi-clause Scope entries and Thread 2 has single-phrase Scope
entries fails C-49 regardless of whether a co-primary assertion accompanies it.

**C-50** vs. C-48: C-48 requires all four marker types to be standalone labeled elements —
a property verified by scanning the document and confirming each marker type appears as a
dedicated structural element rather than embedded in prose. C-50 requires a pre-execution
section that pre-announces all four marker types, their governing criteria, their document
locations, and their required forms — a property verified by finding the inventory section
before any container body and confirming its table completeness and REQUIRED closure
statement. The distinction is between markers that are standalone when a reader finds them
(C-48) and markers whose compliance status is declared before any reader encounters them
(C-50). A document satisfying C-50 makes C-48 auditable from the document's opening section.
