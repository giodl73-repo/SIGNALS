## Design Rationale (v15)

Two aspirational criteria added in v15 (C-47, C-48), one per Round 14 excellence signal:

**C-47** — From R14 Signal 1 (V-05 "class-labeled hedge prohibition"). No existing criterion fires on whether the verbatim hedge prohibition assigns an explicit class name to the category of patterns being prohibited — a label that identifies the failure mode by type, not by enumeration. C-45 fires on whether the prohibition names ≥2 structurally distinct patterns; C-47 fires on whether the prohibition also assigns a class name to those patterns, such that the membership criterion is stated independently of any specific enumerated example. V-05 named three patterns under an explicit class label covering the aggregate-first hedging failure mode. A prohibition without a class label is enumeration-only: an author must match each candidate opening against the listed surface forms, with no guidance for evaluating novel forms that share the failure mode but appear nowhere in the list. A prohibition with a class label is both enumerated AND specified by type: the class definition creates a membership criterion that applies to novel openings whose failure mode matches the class even when the surface form does not match any listed pattern. The governance consequence: a class-labeled prohibition is enforceable against paraphrase — a novel surface form that belongs to the named class is prohibited even if it does not appear in the enumerated list, whereas an enumeration-only prohibition is bypassable by paraphrase to any form not on the list. A minimum-set prohibition (C-45) establishes that the prohibited category has multiple members; a class-labeled prohibition (C-47) names the category itself, making the class boundary legible and extensible in a defined direction: new hedge patterns that fit the class definition append to the enumerated list, and the class label signals the reason for their inclusion. C-45 passes when ≥2 non-redundant patterns are named; C-47 passes only when the prohibition also assigns an explicit class name that identifies a failure mode rather than merely describing the list, appearing adjacent to or framing the enumerated patterns.

**C-48** — From R14 Signal 2 (V-04 "three-element Para 3: trigger + option + inertia-at-trigger"). No existing criterion fires on whether Para 3 of S5 specifies three structural elements: the trigger condition, the recommended action option, AND the inertia consequence at the trigger point — what is forgone if the trigger fires without action. C-46 fires on whether S5 names the inertia comparator and cross-references S4b at the recommendation level; C-48 fires on whether Para 3 instantiates the inertia consequence specifically at the action gate identified by the trigger condition. A two-element Para 3 (trigger + option) is the prior pattern: it specifies when to act and what to do, leaving the cost of inaction at that moment implicit or absent. A three-element Para 3 (trigger + option + inertia-at-trigger) additionally specifies what is lost if the trigger fires without action. The structural difference: C-46 establishes inertia framing at the topic-recommendation level — the claim that adoption is preferable to delay in general. C-48 enforces inertia specificity at the decision gate level — the Para 3 directive must name the particular cost of inaction at the specific trigger point, connecting the action gate to a concrete forgone outcome. A recommendation with a populated C-46 clause and an unpopulated C-48 Para 3 knows that delay is costly in the abstract but does not specify the cost at the decision gate — the author can acknowledge the general framing while leaving the specific consequence unresolved. The governance consequence: a three-element Para 3 makes inaction cost legible at the moment of decision. The trigger is the event where the decision diverges between act and not-act; the inertia-at-trigger element names the thing irreversibly forgone at that event. C-46 passes when S5 names the inertia comparator at the recommendation level; C-48 passes only when Para 3 of S5 explicitly names all three elements — trigger condition, recommended action option, and inertia consequence at the trigger point — with the inertia-at-trigger element tied to that specific trigger condition, not stated as a general observation.

### C-47 — Hedge Prohibition Assigns an Explicit Class Name to the Prohibited Pattern Category

- **Weight**: aspirational
- **Category**: behavior
- **Text**: The hedge prohibition assigns an explicit class name or label to the category of opening patterns being prohibited — a name that identifies the failure mode by type, not by enumeration. C-45 fires on whether the prohibition names ≥2 structurally distinct patterns; C-47 fires on whether the prohibition also names the category those patterns belong to, such that the class membership criterion is stated independently of any specific enumerated example. A prohibition without a class label is enumeration-only: an author must match each candidate opening against the listed surface forms, with no guidance for evaluating novel forms that share the failure mode but do not appear in the list. A prohibition with a class label is both enumerated AND specified by type: the class definition creates a membership criterion that applies to novel openings sharing the failure mode even when the surface form does not match any listed example. The governance consequence: a class-labeled prohibition is enforceable against paraphrase — a new surface form that belongs to the named class is prohibited even if it does not appear in the enumerated list — whereas an enumeration-only prohibition is bypassable by any form not on the list. A minimum-set prohibition (C-45) establishes that the prohibited category has multiple members; a class-labeled prohibition (C-47) names the category itself, making the class boundary legible and extensible: new hedge patterns that fit the class definition are appended to the enumerated list, and the class label signals the reason for their inclusion. C-45 passes when ≥2 non-redundant patterns are named; C-47 passes only when the prohibition also assigns an explicit class name or label to the category being prohibited, with the class name appearing adjacent to or framing the enumerated patterns and identifying a failure mode rather than merely describing the list.
- **Pass condition**: The rubric's hedge prohibition names the category of prohibited patterns with an explicit label — a class name that identifies the failure mode by type. The class name must (a) appear in the prohibition text adjacent to or framing the enumerated patterns, and (b) identify a failure mode or structural property rather than merely denoting the list. Labels such as "aggregate-first hedging", "survey-opening constructions", "collection-summary openers" pass; labels such as "the following patterns", "these phrases", "examples to avoid" do not — they describe the list rather than naming the class. A prohibition that lists ≥2 patterns without naming their category does not pass C-47, even if C-45 is satisfied. A prohibition that gives the section a descriptive title without identifying the failure mode does not pass C-47.

### C-48 — S5 Para 3 Names Three Elements: Trigger Condition, Action Option, and Inertia Consequence at the Trigger Point

- **Weight**: aspirational
- **Category**: behavior
- **Text**: The recommendation section directive for Para 3 specifies three structural elements: the trigger condition that would activate the recommendation, the recommended action option at that trigger, and the inertia consequence at the trigger point — what is forgone or incurred if the trigger fires without action. C-46 fires on whether S5 names the inertia comparator and cross-references S4b; C-48 fires on whether Para 3 instantiates the inertia consequence specifically at the action gate identified by the trigger condition. A two-element Para 3 (trigger + option) is the prior structural pattern: it specifies when to act and what to do, but leaves the cost of inaction at that moment implicit or absent. A three-element Para 3 (trigger + option + inertia-at-trigger) specifies all three: when to act, what to do, and what is lost if the trigger fires without action. The structural difference: C-46 establishes inertia framing at the recommendation level — the topic-level claim that adoption is preferable to delay. C-48 enforces inertia specificity at the decision gate level — Para 3 must name what is lost at the specific trigger point, connecting the action gate to a particular forgone outcome rather than to the general recommendation. A recommendation with a populated C-46 clause and an unpopulated C-48 Para 3 acknowledges that delay is costly in the abstract while leaving the specific consequence at the decision gate unresolved; an author reading only C-46 knows what to recommend but not what to name as the cost of missing the trigger. The governance consequence: a three-element Para 3 makes inaction cost legible at the moment of decision — the trigger is the event where act and not-act diverge, and the inertia-at-trigger element names the thing irreversibly forgone at that event, creating an audit anchor independent of narrative quality. C-48 passes only when Para 3 of S5 explicitly names all three elements — trigger condition, recommended action option, and inertia consequence at the trigger point — with the inertia-at-trigger element identifying a specific forgone outcome or consequence tied to that trigger condition rather than stated as a background recommendation-level observation.
- **Pass condition**: Para 3 of S5 explicitly contains all three of the following: (a) a trigger condition — an identifiable event or signal state that activates the directive; (b) a recommended action option — the specific step or decision recommended when the trigger fires; and (c) an inertia consequence at the trigger — a named outcome, cost, or forgone opportunity that results specifically from the trigger firing without action, stated in connection with that trigger condition. The inertia-at-trigger element must be tied to the trigger condition identified in the same Para 3 directive, not stated as a general recommendation-level claim. A Para 3 that names trigger + option but omits the inertia consequence does not pass C-48, even if C-46 is satisfied. A Para 3 that mentions delay costs in generic terms without tying them to the specific trigger identified in that directive does not pass C-48.
