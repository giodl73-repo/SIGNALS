**What C-40 adds over C-21:** C-21 tests that the vocabulary contract enumerates named violation conditions, making the contract self-enforcing at the instance level. C-40 tests typed classification — violations grouped into labeled categories make it possible to audit an entire class of failures (e.g., "are all gate-failure conditions present?") by scanning category headers rather than reading every enumerated condition. Category structure enables class-level completeness checks; flat enumeration does not.

**What C-41 adds over C-35 and C-37:** C-35 requires schemas to be declared upfront; C-37 requires each schema to document its own breach conditions. Neither requires schemas that share constraints to declare the dependency at definition time. C-41 tests binding discipline — a constraint that originates in Schema A but governs Schema B must be declared at Schema B's Phase −1 block, and at every use site the constraint must be labeled as inherited rather than restated. Without this, the same constraint can be silently redefined at use sites, producing schema drift invisible to Phase −1 audit.

**What C-42 adds over C-34:** C-34 requires two audit paths (write-surface spine + lifecycle slots) and tests that both structural layers are present. C-42 tests that a third path — the Schema Spine — is explicitly named alongside the other two in a declared preamble, making the three-path architecture a committed contract rather than an emergent property of passing C-28, C-34, and C-35 individually. A declared three-path list makes structural regression detectable: if any path disappears in a future revision, the declaration itself becomes inconsistent.
