## Changes v13 → v14

Three new aspirational criteria extracted from Round 14 excellence signals:

**C-44 — Withdrawal-aware confirmation gate** (`behavior`)
Extends C-11. The gate must explicitly scope the YES application to "non-withdrawn proposals" — not "all proposals." The withdrawal path lets the reviewer surface one problematic proposal without blocking the full batch. Source signal: C-11 evidence note "Reply **YES** to apply all **non-withdrawn** proposals."

**C-45 — Delta Finding traceability requirement elevated to named COLUMN CONTRACT rule (Rule 4)** (`enforcement`)
Extends C-40 and C-20. The Delta Finding column mandate must be expressed as a CONTRACT rule — not a step-level instruction. Closes the asymmetry where enumerated-column constraints, column independence, and null-case enforcement are all CONTRACT-governed but the traceability chain remains architecturally unbound. Source signal: C-16 evidence note "Rule 4 + Delta Finding mandatory column = proposal→finding→signal chain."

**C-46 — COLUMN CONTRACT minimum rule count: 4 rules covering 4 distinct constraint classes** (`enforcement`)
Extends C-36 and C-45. Rules 1–4 must each govern a distinct class: value enumeration, column independence, null-case enforcement, traceability. A 3-rule CONTRACT — the v13 ceiling — leaves traceability unbound. Source signal: R14 floor item "4-rule COLUMN CONTRACT minimum (Rules 1–4)."
