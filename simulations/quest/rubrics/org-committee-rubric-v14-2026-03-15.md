**What changed v13 → v14**

**C-36 — VALIDATE rules carry ACCEPTABLE/FAILS worked examples** *(new)*

Signal: V-04's C-34 VALIDATE includes both a positive form (`ACCEPTABLE:`) and a negative form (`FAILS:`). V-01/V-02/V-03 each satisfy C-34 with a prohibition only ("a bare-label entry fails"). The bilateral test closes the gap between satisfying the prohibition in form while missing the intent — the ACCEPTABLE example anchors what to produce; FAILS anchors the violation boundary. Applies to any VALIDATE rule where the passing/failing form boundary is narrow (manifest entry format, `CITE:` structure, `RESPONDS-TO:` specificity).

**C-37 — Manifests are named first-class sections in the pre-declared skeleton** *(new)*

Signal: V-04's axis explicitly names "INERTIA RECORD and STANCE MANIFEST elevated as named first-class parallel artifacts." An output satisfying C-33/C-35 by embedding manifest content only inside `PHASE-N-COMMIT:` blocks is correct but architecturally subordinate — the manifest exists as a commit attribute, not a document section. Named headers (`## INERTIA RECORD`, `## STANCE MANIFEST`) make manifests independently addressable by section heading rather than requiring navigation to the commit block.
