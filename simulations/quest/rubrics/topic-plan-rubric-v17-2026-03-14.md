## Changes v16 → v17

Three new aspirational criteria extracted from three R17 variation axes.

**C-56 — Pre-role inertia framing with downstream recurrence markers** (`behavior`)
Extends C-51. C-51 requires co-equal inertia framing in the preamble. C-56 adds two precision requirements: (1) **placement before the role declaration** — before "You are running /topic:plan..." — so the co-equal prior is set before any task identity is adopted; (2) **explicit recurrence markers** at named downstream decision gates (proposals step, Defender Challenge Table, confirmation gate) so the prior is re-activated at each point where change-bias is most likely to operate. A C-51 pass that places the framing after the role line, or has no recurrence markers, fails C-56. Source: V-01 (first-line pre-role + explicit recurrence at three gates) and V-05 (first-paragraph pre-role + "This framing holds at every step" + recurrence at three gates).

**C-57 — Unforgeable calibration slot: pre-printed verdict strings with "select exactly one" instruction** (`enforcement`)
Extends C-52. C-52 requires a calibration audit artifact (one-sentence distribution declaration). C-57 requires it to be expressed as a **pre-printed checkbox slot** — all three verdict strings written out in full, with "select exactly one and write it as output" — not a conditional branch the model composes. An uncompleted slot is structurally visible as a skipped artifact; a conditional branch can be silently omitted or paraphrased. Same structural architecture as C-23/C-40's verbatim null-case templates, applied to the calibration check. Source: V-02 pre-printed checkbox slot.

**C-58 — Analytical narrative gate rows name the degenerate register with a format discriminator** (`format`)
Extends C-53. C-53 requires narrative-completeness gate rows, with a partial pass for describes-step narratives. C-58 requires gate rows to **explicitly name the degenerate register** (step-description narratives) and include a **format discriminator** ("Narrative: conclusion before evidence") that converts the gate from a binary present/absent check into a register-discrimination check. A 2–3 sentence narrative that describes the table still fails. Naming the degenerate form enables prospective avoidance rather than retrospective correction. Source: V-04 (format-validated gate rows) and V-05 (explicit format marker + degenerate form named and illustrated).
