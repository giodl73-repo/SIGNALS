## V-01 — Single Axis: Lifecycle Emphasis (PART 2 Mirrored Subsection Structure)

**Variation axis:** Lifecycle emphasis — PART 2 (Chasm Analysis) decomposes into three named
subsections matching PHASE 3: `### CHASM-A EXPANSION — Incumbent Defense (Full Diagnosis)`,
`### CHASM-B EXPANSION — Bridge Condition (Full Specification)`, `### CHASM-C EXPANSION —
Crossing Signal (Full Identification)`. Each EXPANSION subsection opens with an explicit
back-reference: "Expanding CHASM-A from PHASE 3 —" / "Expanding CHASM-B from PHASE 3 —" /
"Expanding CHASM-C from PHASE 3 —". PHASE 3 introduces the diagnostic sketch; PART 2
EXPANSION subsections expand it to full diagnostic depth.

**Hypothesis:** PART 2 currently uses bold-label items that mirror PHASE 3's CHASM-A/B/C
headers, but structural coupling is one-way: the model fills PART 2 without being
constrained to expand each PHASE 3 subsection individually. Named EXPANSION subsections
close this gap. A model completing PART 2 cannot skip CHASM-A EXPANSION without leaving
a named structural gap — the same enforcement pattern C-39 applied to PHASE 3. Per-element
omission in PART 2 becomes as structurally conspicuous as in PHASE 3 itself.

## V-02 — Single Axis: Role Sequence (TABLE 1 Inertia ID Column)

**Variation axis:** Role sequence — TABLE 1 adds a fourth column: "Inertia ID: [cite I-0X
from SECTION A — the named inertia that explains this role's archetype position]." Every
role in TABLE 1 receives an archetype assignment (C-01), an Incumbent Dependency (C-37),
and an Inertia ID citation (C-11 per-role) at the exact same generation step. SECTION A
remains as the inertia ID source table; TABLE 1 becomes the per-role inertia assignment
location where the C-11 per-archetype IDs are mapped at the role level.

**Hypothesis:** The gap between SECTION A (five inertia IDs, one per archetype) and TABLE 1
(16 roles mapped to archetypes) requires a cross-reference step: role -> archetype ->
inertia ID. Adding the Inertia ID column to TABLE 1 collapses the two-step reference into
a single generation-time fill contract per row. A model filling TABLE 1 must cite the
inertia ID that explains each role's archetype position — making C-11 citations structurally
co-present with C-01 (archetype) and C-37 (Incumbent Dependency) at the table level. The
three contracts (archetype assignment, displacement grounding, inertia identification) become
one simultaneous fill event per TABLE 1 row rather than three separate section visits.
