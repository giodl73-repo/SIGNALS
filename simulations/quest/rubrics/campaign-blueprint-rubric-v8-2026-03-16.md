### v8 Changes

**One new aspirational criterion (+5 pts, new ceiling 173):**

**C-27 — Scout traceability table in CAMPAIGN SETUP (friction-first placement)** (5 pts, FULL or NO CREDIT): C-25 tests whether a four-column traceability table exists with correct column structure; C-27 tests whether it appears in CAMPAIGN SETUP, pre-populated from scout inventory *before* any artifact instruction, with REQUIREMENTS referencing back to the campaign-level table. Pre-artifact placement makes frictions first-class inputs at must-have writing time — a causal-order improvement over post-hoc traceability inside REQUIREMENTS. V-02 (R7) earns FULL; others earn NO CREDIT.

**One diagnostic rule update:**

**D8 (v8 clarification)**: Bracket notation `[INERTIA MODEL Cost → factual record]` satisfies D8's per-site grounding requirement. The distinguishing test is whether the per-site reminder explicitly names an INERTIA MODEL field — not whether it uses prose form. Abstract labels without explicit field reference (e.g., "Factual — assert what is true") remain PARTIAL. Full prose and bracket notation are functionally equivalent when the field name is present. (V-03 R7 finding.)
