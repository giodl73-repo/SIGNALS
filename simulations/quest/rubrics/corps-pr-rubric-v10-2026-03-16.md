## Three new criteria from Round 10

**C-33 — C2 RESULT semantically scoped as READ RECEIPT completeness check** *(C-26 semantic scope gap)*

V-04's fix to the R9 universal gap: C2 RESULT confirms receipt field completeness, not PRE-COMMITMENT existence. PRE-COMMITMENT existence is C1's scope (phase-level); receipt completeness is C2's scope (per-role). A C2 RESULT that targets the wrong scope satisfies C-26's ordering requirement but fails C-33 because the two result lines are not independently auditable at their own scopes.

**C-34 — F-01 typed as IA-RESPONSE in per-role findings template** *(C-31 template-enforcement gap)*

V-04 elevated C-31 from a rubric check to a template invariant by designating F-01 as a typed IA-RESPONSE slot. A role section where F-01 is absent or not typed IA-RESPONSE fails the template, not just C-31. Requires the type designation to appear as a visible output field. A variation that satisfies C-31 by placing counterpoint in F-02 or later without the typed F-01 slot passes C-31 but fails C-34.

**C-35 — READ RECEIPT block precedes C2 RESULT in per-role output** *(receipt-before-findings ordering edge case)*

Closes the edge case where findings appear before the receipt while C2 RESULT is technically post-receipt, leaving the C-23 ordering violation undetected. C-35 and C-23 are complementary: C-23 requires receipt before findings; C-35 requires C2 RESULT after receipt. Together they enforce receipt-first ordering from both directions.

## What changed from v9

**C-33 closes the C-26 semantic scope gap.** C-26 requires C2 RESULT to appear after the IA READ RECEIPT in each role section. R10 V-04 showed a tighter pattern: C2 RESULT is semantically scoped as a receipt completeness confirmation -- it verifies that all required fields (a)-(e) are present in the READ RECEIPT block, not that the PRE-COMMITMENT block was read. The R9 universal gap was that C2 RESULT targeted PRE-COMMITMENT existence (C1's scope) rather than receipt completeness (C2's scope), so the two result lines were not independently auditable. C-33 requires the semantic separation: C1 is a phase-level gate on PRE-COMMITMENT existence; C2 is a per-role gate on receipt field completeness. A C2 RESULT audited against the PRE-COMMITMENT block rather than the receipt's field list fails C-33 even if C-26's ordering requirement is satisfied. The separation makes each result line independently verifiable at its own scope without cross-referencing the other block.

**C-34 closes the C-31 template-enforcement gap.** C-31 requires the IA counterpoint reference to appear within the body text of a specific finding entry. R10 V-04 showed a tighter enforcement mechanism: the per-role findings template designates F-01 as a typed IA-RESPONSE slot. This makes C-31 a template invariant rather than a rubric check applied post-hoc: a role section where F-01 is absent or not typed IA-RESPONSE fails the template. A variation that satisfies C-31 by placing an IA counterpoint in any finding without an F-01 IA-RESPONSE type designation passes C-31 but fails C-34. The type designation must appear as a visible output field (e.g., a Type column entry), making compliance verifiable from the structure alone. The key distinction from C-31: C-31 requires finding-level body text placement; C-34 requires that placement to be structurally enforced at the first finding of every role section by a typed template slot.

**C-35 closes the receipt-before-findings ordering edge case.** C-23 requires the READ RECEIPT block to appear before any finding is written per role. C-26 requires C2 RESULT to appear after the READ RECEIPT. R10 identified a residual edge case: findings could precede the receipt while C2 RESULT is still technically post-receipt, leaving the C-23 ordering violation undetected by C-26 alone. C-35 closes this by requiring C2 RESULT to be preceded in the output by that role's READ RECEIPT block -- if the receipt is absent when C2 RESULT is written, the criterion fails visibly. C-35 and C-23 enforce receipt-first ordering from complementary directions: C-23 requires receipt before findings; C-35 requires C2 RESULT after receipt. A run where findings appear before the receipt would fail C-23 directly; C-35 adds a second structural check that makes the same violation detectable via the C2 RESULT line's position without separately auditing finding order.
