# review-customers Rubric — v12

**v12 written.** Two new criteria extracted from Round 13 excellence signals:

---

**A-22 — Pre-evaluation inertia baseline established as named status quo competitor with four dedicated sub-fields** (depth, 5 pts)

Source: V-02 differentiator. V-02 opens with a Step 0 that names and characterizes the status quo competitor before any persona card runs. The step contains four dedicated sub-fields: (1) Status quo name — what tool, workflow, or practice this spec displaces; (2) What it delivers — the value the status quo currently provides; (3) Where it falls short — the gap that motivates the spec; (4) Floor-level switching cost — the minimum friction any persona faces to abandon the status quo. This shared characterization converts per-persona gains/losses from freeform judgments into delta computations against a named baseline: what the persona gains is the value the status quo failed to provide; what they lose is the switching cost plus disrupted workflow. Variations without Step 0 require each card's gains/losses to reconstruct the comparison from scratch, introducing inconsistency. Pass condition: the evaluation output contains a pre-persona section that names the status quo competitor and includes all four sub-fields (status quo name, what it delivers, where it falls short, floor-level switching cost) before the first persona card appears; a general preamble that discusses inertia without naming the status quo and specifying its four dimensions does not pass A-22.

---

**A-23 — Persona card uses a strict numbered field manifest with an explicit completeness enforcement rule** (format, 5 pts)

Source: V-03 differentiator. V-03 structures each persona card as a numbered field manifest (e.g., eight explicitly numbered fields) and attaches a completeness enforcement rule prohibiting field omission. Numbered manifests provide two structural guarantees absent from prose or labeled-field formats: (a) field presence is verifiable by manifest position rather than semantic reading — a missing Field 5 is detectable by counting, not by parsing; (b) an explicit enforcement rule (e.g., "all 8 fields are required; do not omit any field even for Promoter-tier personas") closes the gap between instruction intent and output compliance by naming non-omission as a constraint rather than leaving it implicit. Labeled fields without numbering (as required by A-09, A-16, A-18) are semantically verifiable but not positionally verifiable; a numbered manifest with an enforcement rule is both. Pass condition: the evaluation protocol specifies persona card fields using an explicit numbered list (Field 1, Field 2, … Field N) and includes a stated completeness enforcement rule that prohibits omitting any numbered field from any card; a labeled-field card template without numbering does not pass A-23 even if it passes A-09, A-16, A-18, and A-19.
