**v15 written.** Here's what changed from v14:

**New criterion: C-35** — *Blocking gate HALT instruction uses positional reference language*

The R16 V-03 pattern: token names embedded inside the HALT sentence satisfy C-24 (enumeration present in any form) but not C-34 (not a structurally separable block). At ceiling, the HALT instruction should use positional reference language — "if any of the above tokens is absent, halt" or "do not proceed if any listed token is missing" — rather than naming tokens inline. Positional reference makes the enumeration block independently revisable: the token list can be updated without rewriting the HALT sentence. V-03 is the canonical diagnostic probe: C-24 PASS + C-34 PARTIAL + C-35 not scored (C-34 PARTIAL gates C-35).

C-35 is only scored when C-34 is PASS. When C-34 is PARTIAL, C-35 is not scored. D gate: −1 when C-34 PARTIAL.

**New criterion: C-36** — *Engineering routing branch carries a named deviation marker at branch-instruction level*

The R16 deviation-only routing ceiling: C-30 (no exec branch) and C-32 (scope declared as deviation-only) establish the deviation-only architecture at scope level. At ceiling, the engineering branch instruction itself carries an explicit deviation marker — DEVIATION:, OVERRIDE:, EXCEPTION:, or equivalent — that makes its non-default status legible at branch level without requiring the reader to consult the scope declaration header. This marker is orthogonal to C-31 (which governs positional vocabulary for the ordering description): "DEVIATION: if REG = engineering → matrix precedes recommendation" satisfies both C-31 ("precedes") and C-36 ("DEVIATION:" label). C-36 does not require naming BODY ORDER in the branch instruction — the marker label alone distinguishes deviation from default.

C-36 is only scored when C-32 is PASS (deviation-only routing block confirmed). When C-32 is N/S, C-36 is not scored. D gate: −1 when C-32 N/S.

**Denominator: /29** (base). C-35 not scored when C-34 PARTIAL (→ /28). C-36 not scored when C-32 N/S (→ /28). Both exclusions active simultaneously (→ /27).

**Four new watch list rows:**
- `c35-positional-reference-halt` — "any of the above" / "any listed token" language satisfies C-35; halt must not name tokens inline
- `c35-c34-gate-relationship` — C-35 only scored when C-34 PASS; C-34 PARTIAL excludes C-35

**Previously resolved discrimination questions now anchored:**

- `inline-within-halt-c24-pass-c34-partial-boundary` — V-03 is the canonical diagnostic probe for the C-24/C-34 split; C-35 is the next-tier criterion above the C-34 ceiling

## Essential Criteria (60%)

| ID | Criterion | Category | Weight | Pass Condition |
|----|-----------|----------|--------|----------------|
| C-01 | All four dimensions are covered for both options | coverage | essential | Output addresses feasibility, risk, competitive positioning, and inertia for Option A AND Option B. Missing any dimension for either option = fail. |
