**Two new criteria extracted from R15, both extending the register-consistency principle established in C-33:**

**C-39 — HALT-AND-CONFIRM block carries explicit sole-instruction-site annotation for each governed enforcement requirement**

Source: V-02/V-04/V-05 C-33 evidence. C-33 prohibits the checklist-strict/prose-advisory split; C-39 closes the *future-drift* version of the same failure mode. A prompt that achieves C-33 passively (no contamination present) places no constraint against a subsequent revision introducing advisory language at a new instruction site. The sole-instruction annotation (`[...sole instruction site...]`) declares the HALT block as exclusive authority, converting the C-33 prohibition from a static check-on-output into a declared architectural constraint at prompt-structure level.

**C-40 — REGISTER-CONSISTENCY AUDIT block embedded at each phase gate before advancing**

Source: V-03 C-33 evidence — "the most thorough C-33 mechanism across all five variations." C-33 + C-39 achieve register consistency at declaration time but provide no mechanism for detecting contamination introduced in section-producing steps *between* gates. The per-gate REGISTER-CONSISTENCY AUDIT requires the model to explicitly confirm register consistency for all active governed requirements at each gate boundary, surfacing contamination at the gate where it is introduced rather than in later scored output.

**Updated scoring formula:** 40 criteria -- 8 essential x 5 pts = 40 pts; 32 aspirational x 2.5 pts = 80 pts; **total = 120 pts**
