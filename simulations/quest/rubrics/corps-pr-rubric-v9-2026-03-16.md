## Five new criteria from Round 8

**C-28 — Three-clause formula line-boundary enforcement** *(C-25 line-boundary gap)*

R8 V-05 showed each of `WORSE rows:`, `BETTER rows:`, `Conclusion:` on its own output line at consistent indentation. C-25 requires the three labeled lines; C-28 requires that each label appears at line start with no other content on the same line. A verdict where two clauses share a line, or where a clause is embedded in a sentence, satisfies C-25's label-presence requirement but fails C-28 because extraction requires context parsing rather than a line-start match.

**C-29 — PRE-COMMITMENT seal token before diff content** *(C-27 sealing gap)*

V-05 closed the PRE-COMMITMENT block with an explicit seal token (`[PRE-COMMITMENT SEALED]`) as a distinct output line before any PR diff file, function, or line is referenced. C-27 requires positional ordering (block precedes findings) but has no explicit closure event. Without the seal, a model can continue appending to the block after diff analysis begins without producing a visible ordering violation. With the seal, the closure is a named output artifact whose position is direct evidence of timing.

**C-30 — Phase C exit condition names PRE-COMMITMENT as compliance item** *(C-18/C-27 phase-gate coupling gap)*

V-05 explicitly listed "PRE-COMMITMENT blocks precede findings tables" in Phase C's exit condition checklist. C-27 and C-18 each exist independently — neither requires the phase-gate to enforce C-27 as a gate. C-30 elevates PRE-COMMITMENT absence from a C-27 format gap to a Phase C exit-condition violation, detectable as a phase failure without separately auditing C-27.

**C-31 — IA counterpoint in finding body text, not standalone section element** *(C-14 finding-level gap)*

All R8 variations scored PARTIAL on C-14: IA references appeared as standalone section anchors ("IA Position:" headers) that satisfy C-14's count requirement but are not attached to any specific finding. C-31 requires the IA reference to appear within the body text of a specific finding entry (F-NN), naming the IA verdict and the disagreeing mechanism together in the same entry.

**C-32 — READ RECEIPT cross-references PRE-COMMITMENT for fields (d) and (e)** *(C-23 completeness gap under C-27)*

All R8 variations scored PARTIAL on C-23 when C-27 was active: the PRE-COMMITMENT block holds fields (d) and (e) correctly, but the READ RECEIPT omits them without a pointer. A reviewer reading the receipt cannot locate those fields without knowing to check the PRE-COMMITMENT block. C-32 requires the receipt to carry explicit cross-references to PRE-COMMITMENT for fields (d) and (e) by label name, or restate them inline.

## What changed from v8

**C-28 closes the C-25 line-boundary gap.** C-25 requires the Budget verdict to appear as exactly three labeled lines. R8 V-05 showed a tighter pattern: each labeled clause appears on its own output line at consistent indentation, with no other content on the same line. This makes each clause independently retrievable by a line-scan tool that matches the label string at line start -- no prose parsing required. A verdict where two clauses share a line, or where a clause is embedded in a sentence, satisfies C-25's string-matchability claim only partially: the labels are present, but extraction requires context parsing. C-28 requires each clause to be a standalone line so the matching predicate is: label-at-line-start, value after separator, end of line.

**C-29 closes the C-27 sealing gap.** C-27 requires the initial position to appear in a PRE-COMMITMENT block that structurally precedes the findings table. R8 V-05 showed a tighter pattern: the block closes with an explicit seal token (e.g., "[PRE-COMMITMENT SEALED]") as a distinct output line before any PR diff content is referenced. Without the seal, C-27's ordering claim is verifiable by reading block positions relative to findings, but the block has no explicit closure -- a model could continue appending to it after diff analysis begins without creating a visible ordering violation. With the seal token, the closure is a named output event: the token's position in the output is direct evidence that the commitment was locked before diff content appeared. C-29 requires the seal as an output artifact, not as a positional inference.

**C-30 closes the C-18/C-27 phase-gate coupling gap.** C-27 requires the PRE-COMMITMENT block to precede findings structurally. C-18 requires a named phase-gate structure with explicit exit conditions. Neither criterion requires the phase-gate to name PRE-COMMITMENT compliance in its exit checklist -- a pipeline that satisfies both can still treat PRE-COMMITMENT absence as a format issue rather than a phase violation. R8 V-05 showed the closing pattern: Phase C's exit condition explicitly names "PRE-COMMITMENT blocks precede findings tables" as a compliance item. This elevates PRE-COMMITMENT absence from a C-27 format gap to a Phase C exit-condition failure, making it detectable as a phase violation without separately checking C-27. C-30 requires this elevation: the phase-gate structure must enforce C-27 as a gate, not merely reference it.

**C-31 closes the C-14 finding-level gap.** C-14 requires each technical role section to contain at least one finding that names the Inertia Advocate's assessment as the counterpoint. R8 scoring showed all variations PARTIAL: implementations provide IA references as standalone section anchors ("IA Position:" headers) that satisfy C-14's count requirement but appear outside any finding body. The counterpoint is visible but not attached to a specific claim -- a reviewer cannot tell which finding the IA reference refutes. C-31 closes this: the IA reference must appear within the body text of a specific finding entry. The finding text names the IA verdict and the technical role's disagreeing mechanism in the same entry, making the counterpoint relationship explicit and finding-scoped rather than section-scoped.

**C-32 closes the C-23 completeness gap under C-27.** C-23 requires the READ RECEIPT to include fields (d) cost row and (e) initial position. C-27 moves the initial position into a PRE-COMMITMENT block that precedes the receipt. When C-27 is active, a receipt that omits (d) and (e) is structurally correct per C-27 but incomplete per C-23 -- a reviewer reading the receipt cannot locate fields (d) and (e) without knowing to look at the PRE-COMMITMENT block. C-32 closes this: the receipt must carry explicit cross-references for (d) and (e) pointing to the PRE-COMMITMENT block by label name, or restate them inline. Either form makes the receipt complete by itself: a reviewer reading the receipt can find all five fields without additional context.
