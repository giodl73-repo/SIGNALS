# Scoring Rubric — corps-committee

Simulates a committee meeting before the real one. Types: ROB (product/service review),
shiproom (go/no-go), arch-board (architecture decision review). Reads participant roles
from `.roles/`. Output: mock meeting minutes with decisions, action items, dissenting
opinions.

## Change Log

**v12 (R11 signals):** Three structural properties identified in R10 analysis are formalized
as criteria. The R11 discriminator matrix validated a three-way dependency pattern across
V-01–V-05:

1. **Re-COMMIT cycle after AMEND (C-34)** — C-30 routes AMEND through the block dependency
   graph; C-33 marks stale output sites. But re-execution blocks must also emit a versioned
   RECOMMIT MANIFEST before downstream phases resume. Phase re-entry must gate on manifest
   currency, not on original v1 seals. V-04 demonstrates the pairing value: routing is
   deterministic, delta-annotated, and version-tracked.

2. **COMMIT seal token manifest (C-35)** — C-31 requires a COMMIT annotation closing each
   block. But the annotation must enumerate *which* tokens are sealed. Without an explicit
   closed set, TOKEN TRACE CONFIRMATION (C-32) can verify tokens it finds but cannot detect
   tokens dropped without trace. V-02/V-03 expose this gap from both sides.

3. **Three-way CONFIRMATION status (C-36)** — C-32 requires TOKEN TRACE CONFIRMATION in
   Phase 2. With a C-35 manifest, each sealed token receives a completion status —
   CONSUMED, NOT-APPLICABLE, or DROPPED — and the table row count must equal the manifest
   token count. Without C-35, the vocabulary is present but the count invariant is
   unenforceable: a token never assigned a row cannot be detected as DROPPED. V-03 exposes
   this: C-36 PASS (weak) because vocabulary exists but closed-set reconciliation is absent.

Aspirational denominator: 25 (v10) → 28 (v12). Total criteria: 36.

## Essential Criteria (60% of composite)

| ID | Criterion | Category | Pass Condition |

| C-01 | **Committee type is correctly instantiated** | correctness | Output identifies the committee type (ROB / shiproom / arch-board) and applies the correct framing: ROB produces a product/service review verdict; shiproom produces a go/no-go decision; arch-board produces an architecture decision record (ADR). | C-03 | **Decisions are explicitly recorded** | correctness | The minutes contain a clearly labeled **Decisions** section with at least one concrete decision stated as an outcome (approved / rejected / deferred / conditional-approval). Vague summaries without a stated outcome = fail. |
| C-04 | **Action items are captured with owners** | correctness | Each action item names both the action and the responsible party. Action items without owners, or minutes with no action items when open questions were raised, = fail. |
| C-05 | **Dissenting opinion is represented when applicable** | coverage | If any participant's role or position creates tension with the majority outcome, a dissenting opinion must appear. If all participants agree, vacuously satisfied. Minutes that smooth over explicit disagreements = fail. |

The three new criteria close the amendment + confirmation loop as a linked system: C-34 ensures re-execution produces versioned evidence, C-35 gives CONFIRMATION a closed set to reconcile against, and C-36 makes dropout structurally detectable via the count invariant. Each criterion is vacuously satisfied if its predecessor is vacuously satisfied, preserving the dependency chain from C-28 downward.
