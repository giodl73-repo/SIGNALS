## flow-lifecycle — Scoring Rubric v20

Two new aspirational criteria formalized from Round 19 scoring evidence: D-ID
cross-referencing of inline gate defect categories to scan-table rows (C-57), and
parenthetical eligibility binding format for typed escape codes (C-58).
Aspirational count: 50. Composite denominator: 58.

---

### Essential Criteria (all must pass)

| ID | Criterion | Category | Pass Condition |
|----|-----------|----------|----------------|
| **C-01** | **State Transition Coverage** | correctness | ≥6 named states; each has explicit entry trigger and exit trigger; no "then X happens" transitions |
| **C-02** | **Exception Flow Traces** | coverage | ≥3 exception flows, each naming: triggering condition, divergence phase/step, recovery or terminal failure state |
| **C-03** | **Terminal State Completeness** | correctness | ≥1 success terminal + ≥1 failure/cancel terminal; every traced path reaches a named terminal |
| **C-04** | **Bottleneck and Gap Identification** | depth | ≥2 bottlenecks (cause + impact); ≥1 process gap (missing step named + consequence stated) |
| **C-05** | **Domain Role Assignment** | correctness | ≥3 domain-qualified roles; every decision gate is owned; no generic placeholders |

### Recommended Criteria

| ID | Criterion | Category | Pass Condition |
|----|-----------|----------|----------------|
| **C-06** | **Parallel Path Modeling** | depth | ≥1 parallel path with named join condition and join owner; or explicit "no parallel" declaration with rationale |
| **C-07** | **Decision Point Explicitness** | format | ≥3 named decision points, each with condition, owner role, and all outcome branches |
| **C-08** | **Edge Case Enumeration** | coverage | ≥2 edge cases distinct from C-02; each names scenario, gap reason, and consequence |

**Two new criteria in v20:**

- **C-57** (Gate Defect D-ID Cross-Reference) — closes the gap between C-55's inline-enumeration requirement (names appear in gate text) and the stronger pattern where each named category carries a parenthetical D-ID that links it to a specific scan-table row; C-55 is satisfied by plain prose category names, but plain names leave the gate-to-taxonomy navigation as a category-match scan; D-IDs make the linkage row-specific and bidirectional — V-04 confirmed that inline category names (C-55 pass) without D-IDs still leave row-level navigation by scan, while V-01/V-02/V-03/V-05 closed that gap with parenthetical D-ID tokens on each category

- **C-58** (Parenthetical Eligibility Binding Format) — closes the gap between C-56's eligibility-condition co-location requirement (format and condition are adjacent) and the stronger pattern where code token and eligibility clause are bound as a single typographic unit by immediate parenthetical adjacency; C-56 is satisfied by any co-located eligibility language including dash-separated imperative forms ("write SECONDARY -- but only when..."), but the dash-separated form places an instruction verb between the code token and its qualifier, creating two parsing units rather than one; V-04 confirmed that eligibility co-location (C-56 pass) without parenthetical binding still produces a separable code + instruction structure, while V-01/V-02/V-03/V-05 closed that gap with the code-token + immediately-adjacent parenthetical form
