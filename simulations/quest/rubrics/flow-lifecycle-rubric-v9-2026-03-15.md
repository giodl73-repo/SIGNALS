**v9 changelog**: Added C-29–C-30 from Round 8 excellence signals.

C-29 captures the Decision Condition threshold-type taxonomy pattern — the structure where the Decision Condition column header operationalizes what "measurable threshold" means rather than leaving the practitioner to infer it. V-01/V-03/V-04 are PARTIAL on C-17 because their column labels say only "(measurable threshold)" — an abstract label that requires interpretation. V-02 achieves PASS via enumeration: the Decision Condition column header reads `measurable threshold (dollar amount, day count, retry count); qualitative condition alone does not count` — naming the threshold type categories explicitly. V-05 achieves PASS via a quoted passing example: its AS-IS ANCHOR for Decision Points reads *"A passing condition names a measurable threshold (e.g., 'Order value > $25,000')"* — a concrete example with comparison operator and units. Both mechanisms close the same gap: the column definition tells the practitioner not just that thresholds must be measurable but what kinds of values qualify. This mirrors the relationship between C-25 and C-27 — C-25 requires the gate's causal mechanism to be named; C-27 requires that mechanism to be anchored in a scan-table defect taxonomy. Analogously, C-17 requires measurable thresholds in cell content; C-29 requires the column definition itself to classify what threshold types qualify. A rubric can pass C-17 without passing C-29 (if cells contain quantified values but the column header carries only an abstract label), but a rubric that passes C-29 necessarily satisfies C-17's intent at the schema level.

C-30 captures the exception-catalog boundary gate pattern — the structure where sequential gates guard the transitions immediately adjacent to the exception catalog, in addition to the early-schema gates (role registry, phase map, state trace) required by C-20. All five variations pass C-20 (≥3 distributed gates), but gate placement reveals a structural distinction. V-01/V-03/V-04 place all gates at early-schema boundaries and the final production gate, leaving the exception catalog's upstream and downstream transitions unenforced. V-02 adds GATE C guarding the exception catalog → terminal declaration boundary, enforcing that terminal declarations may not be authored before exception flows are complete. V-05 adds both GATE C (state trace/decisions → exception catalog) and GATE D (exception catalog → terminal declaration), enforcing both the upstream and downstream exception-catalog boundaries. C-20 requires that gates distribute across the schema at minimum at role registry, phase table, and artifact production; C-30 additionally requires at least one gate at an exception-catalog boundary, closing a mid-schema dependency that C-20's minimum coverage leaves open. A rubric cannot pass C-30 without first passing C-20.
