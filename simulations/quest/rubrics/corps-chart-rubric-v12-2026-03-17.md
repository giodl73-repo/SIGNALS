**Two new excellence patterns extracted from R12 and added as criteria:**

**C-33 — Instruction-site register consistency: no checklist-strict/prose-advisory split**

V-04's C-25 failure reveals a specific register contamination pattern: a HALT-AND-CONFIRM checklist item uses strict mandatory language ("all values from ROLE-NAME LOCK") while a co-located column instruction for the same requirement retains advisory language ("where a role is known"). The model satisfies the checklist check, then interprets the adjacent advisory qualifier as the operative generation instruction — the softening qualifier persists in output despite the strict checklist. C-33 closes this: every instruction site that references a governed enforcement requirement must present a single unambiguous mandatory-register formulation. A HALT-AND-CONFIRM checklist item cannot co-exist with an advisory instruction for the same governed property anywhere in the same prompt.

**C-34 — Per-granularity enforcement sites use procedural action directives, not emphatic importance framing**

V-01's C-28 partial failure reveals the emphatic-framing evasion mode: Key Roles receives "treat it as a first-class enforcement site, not an optional best-effort column" — which signals correct priority without specifying the mechanical check. The procedural directive ("for each cell, verify before populating the next row") is absent. Emphatic language is commentary; procedural language is mechanism. C-34 closes this: per-cell (C-28) and per-row (C-32) enforcement sites must specify both the verification action and the temporal sequencing constraint — emphatic framing alone is insufficient.

| C-33 | Instruction-site register consistency: no checklist-strict/prose-advisory split for any single governed enforcement requirement | behavior | For every governed enforcement requirement expressed in a HALT-AND-CONFIRM checklist item (satisfying C-31) using strict mandatory language -- for example, "all values must appear in the ROLE-NAME LOCK list" -- no other instruction site in the same prompt that references the same requirement may use advisory, hedged, or conditional language. A prompt satisfies C-33 if and only if every instruction site that references a governed enforcement requirement presents a single unambiguous mandatory-register formulation -- no softening qualifiers at any site, including sites outside the checklist itself. A HALT-AND-CONFIRM checklist item cannot co-exist with an advisory instruction for the same governed property at any location in the same prompt. |

Prohibited emphatic forms include "treat as a first-class enforcement site," "not an optional best-effort column," "enforce rigorously," "this column is critical for role coherence," and any framing that characterises the column importance rather than specifying what to check and when. Emphatic language is commentary; procedural language is mechanism.
