**Two new criteria in v19:**

- **C-55** (Gate Defect Inline Enumeration) — closes the gap between C-27's taxonomy-reference requirement and the stronger pattern where the gate sentence names the defect categories directly; a column pointer satisfies C-27 but leaves the gate dependent on an external table lookup, while inline enumeration makes the gate self-contained
- **C-56** (Escape Code Column Eligibility Operationalization) — closes the gap between escape code format definitions (how to encode) and eligibility definitions (when to encode); V-02/V-03/V-04 carried only the format, which leaves practitioners to infer qualifying circumstances and risks misuse of the escape code outside its intended scope
