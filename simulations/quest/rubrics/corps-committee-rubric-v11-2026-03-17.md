Reading the Round 10 scorecard, three new structural properties emerge:

1. **Re-COMMIT cycle after AMEND** — C-30 routes AMEND through the block dependency graph; C-33 marks stale output sites. But nothing currently requires re-execution blocks to re-emit their COMMIT seal before downstream phases resume. V-01 and V-02 expose this gap (COMMIT seals exist, AMEND routing absent), and V-03 exposes it from the other side (AMEND routing declared, no re-COMMIT closing the loop).

2. **COMMIT seal token manifest** — C-31 requires a COMMIT annotation that closes each block before downstream citation. But the annotation asserts completion without enumerating *which* tokens are sealed. Without an explicit manifest, TOKEN TRACE CONFIRMATION (C-32) cannot reconcile against a known closed set — it can only verify tokens it finds, not tokens that were dropped without trace.

3. **Full-coverage TOKEN TRACE CONFIRMATION** — C-32 requires Phase 2 to re-validate inertia tokens at the output layer. But "re-validate" currently means confirming that cited tokens appear in Phase 2 content. With a COMMIT manifest (C-35 below), the confirmation can produce a completion status per sealed token — CONSUMED, NOT-APPLICABLE, or DROPPED — making omission structurally detectable rather than a prose-level absence.

These become C-34 through C-36. Aspirational denominator bumps from 25 → 28.
