Reading the scorecard carefully before writing.

Key patterns from R8:

**V-01 FAIL C-28** — labeled-line blocks satisfy content but not column-existence verification. Gap: no pre-declared schema section; field requirements buried in role instructions.

**V-02 PARTIAL C-28** — tables exist but schema is discovered by reading role instructions, not a declared section. Same gap, softer form.

**V-01 FAIL / V-02 FAIL C-29** — C-09 PARTIAL in both confirms trade-off is never structurally mandated as a named required output.

**V-01 FAIL / V-02 PARTIAL C-30** — neither declares a register explicitly; V-02's compliance markers are implied by column names (register-as-table-format, not register-as-declaration).

**V-03 pattern (conversational register)** — in conversational output there are no column headers to serve as implicit compliance markers. C-30 requires a register declaration but doesn't require per-field marker mapping. Without explicit mapping, an evaluator must read full prose to verify freshness verdict, elapsed total, and error handling — verification degrades silently when register changes.

Three new criteria:

- **C-31**: pre-declared schema section (C-28 requires column tables; C-31 requires the schema to be declared before role output, discoverable without reading role instructions)
- **C-32**: register-specific compliance marker mapping per field (C-30 requires register declaration; C-32 requires the verification method per field to be enumerated explicitly — column name for tabular, sentence-opener or bold keyword for conversational)
- **C-33**: register-invariant declaration when register deviates from tabular default (names which criteria still apply, how each is checked in the new register — prevents silent verifiability degradation)

C-30 requires register declaration; C-32 requires compliance marker mapping per field; C-33 requires that when register deviates from tabular default, the invariants section makes register-switching non-destructive to verifiability. ---

Three new criteria added. The progression:

- **C-31** closes the gap where C-28 tables exist but their schema is only discoverable by reading role instructions — a pre-declared schema section makes field enumeration location-independent.
- **C-32** closes the gap where C-30 declares a register without specifying *how* each field is verified in that register — the per-field marker mapping makes compliance verification executable without rubric knowledge.
- **C-33** closes the gap exposed by V-03: declaring a non-tabular register without a register-invariants section silently degrades verifiability for every criterion that previously relied on column existence — the invariants section makes register-switching explicitly non-destructive.
