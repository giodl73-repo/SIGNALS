**C-57 — Completion-action task noun lexically derived from artifact class name** (extends C-55)

Every C-55-passing variation in R18 uses class-derived task vocabulary: `(FM->ACTOR BRIDGE)` → `ACTOR MAPPING`; `(FM->TRIGGER BRIDGE)` → `TRIGGER MAPPING`. C-55 requires only that a task be named — a generic directive `AND COMPLETE THE REQUIRED WORK` would pass C-55. C-57 closes that gap: the task noun must derive from the discriminating vocabulary of the class annotation. Without derivation, the completion instruction adds no information over what the class annotation already supplies.

- Fail form (C-55 minimum): `AND COMPLETE THE BRIDGE ARTIFACT` — task named, class-generic
- Pass form: `AND COMPLETE THE ACTOR MAPPING` where class is `(FM->ACTOR BRIDGE)` — `ACTOR` flows from class to task noun
- Chain position: C-53 → C-54 → C-55 → **C-57**
- C-54 is a precondition for C-57: class-derived vocabulary requires a class annotation to derive from

**C-58 — Domain-prefix rule labels positionally segregated at non-bridge structural positions** (extends C-42)

V-04 achieves 10.00 in the domain-prefix register for the first time. **What was extracted and why:**

**C-57** — Every C-55-passing variation in R18 uses class-derived task vocabulary: `complete the actor mapping` where the class is `(FM->ACTOR BRIDGE)`, and `complete the trigger mapping` where the class is `(FM->TRIGGER BRIDGE)`. The discriminating noun from the class annotation (`ACTOR`, `TRIGGER`) flows directly into the task noun (`ACTOR MAPPING`, `TRIGGER MAPPING`). This is a genuine structural refinement of C-55: C-55 requires a task to be named; C-57 requires the task to be semantically tied to the artifact class. The practical consequence: without class-derived vocabulary, an author could write a valid C-55 clause (e.g., `AND COMPLETE THE BRIDGE ARTIFACT`) that gives no additional information over what the class annotation already provides. C-57 closes that gap — the task noun must add class-specific semantic content that the class annotation alone does not supply. C-54 is a precondition for C-57 evaluation: class-derived task vocabulary presupposes a class annotation to derive from. The chain C-53 → C-54 → C-55 → C-57 adds one new semantic constraint at each step.

**C-58** — V-04 is the critical R18 result because it achieves 10.00 in the domain-prefix bracket-suffix register for the first time. The enabling structural property: domain-prefix rule labels (`INERTIA THREAT RULE [A-16]`, `INERTIA THREAT CITATION RULE [A-19]`, `INERTIA THREAT FAIL-FIRST RULE [A-31]`) are confined to non-bridge structural analysis positions, while bridge-command labels (`[BRIDGE Q3 COMMAND]`, `[BRIDGE Q4 COMMAND]`) occupy the bridge artifact authoring positions. This positional segregation is not merely incidental — it is what permits Q3/Q4 to be standalone artifact-ID-named sections (enabling C-56) while domain vocabulary threads through the analysis labels (satisfying C-42/C-46). The prior domain-prefix failure in R17 V-04 occurred because that variation used Stage 2A/2B ordinal sub-sections, collapsing Q3/Q4 under a parent stage and preventing C-56. The R18 V-04 scaffold separates the two label classes by function and position — analysis-domain labels at analysis positions, bridge-command labels at bridge positions — and this is a new structural observation worth formalizing as C-58. V-05 (BRIDGE-prefixed labels at bridge/gate positions, no domain-prefix labels at non-bridge analysis positions) fails C-58, giving R19 a discriminating criterion between V-04 and V-05 for the first time.
