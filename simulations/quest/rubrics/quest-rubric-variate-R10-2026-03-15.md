# Rubric: quest-variate (v10)

Evaluates output from the `quest-variate` skill, which generates N distinct prompt variations
of a skill body. Each variation must be complete and runnable, vary along exactly one axis,
and carry a labeled hypothesis.

**v10 changes**: Added C-33, C-34, and C-35 from Round 9 excellence signals. Updated scoring
formula to account for 3 additional aspirational criteria (denominator 24 → 27). No essential
or recommended criteria changed.

Three new cascades:

| New ID | Pattern | Source | Layers on |
|--------|---------|--------|-----------|
| C-33 | **Execution-site invariant labeled self-documentation** — the co-located non-suppression annotation is a labeled block (e.g., `NON-SUPPRESSION INVARIANT CO-LOCATED AT EXECUTION SITE:`) that includes an explanatory sub-clause stating why co-location is necessary ("a model following this template cannot omit checkpoint 2 even if it ignores the preamble invariant or role definition"), making the annotation self-contained and its purpose evident without cross-referencing setup commentary | C-31 V-02 "maximum execution-site coverage" — the fully labeled, self-explanatory annotation block went beyond the terse NOTE pattern of V-01/V-03/V-04, embedding the rationale for co-location inside the annotation itself | C-31 |
| C-34 | **FAIL-directive completion-gate semantic distinction** — when a FAIL-path directive includes an advancement restriction ("do not proceed to checkpoint 2 until X is remediated"), the directive or an adjacent annotation explicitly clarifies that this restriction governs audit completion status rather than execution order, resolving the apparent conflict with the non-suppression invariant and making the two constraints unambiguous at the point of use | C-32 V-03 PASS — FAIL-path directive for 4a said "do not proceed to 4b until Column Contract names the Catalog Source column" while the non-suppression invariant simultaneously said "advance to 4b regardless of this result"; the scorecard resolved this as two constraints operating at different levels (completion gate vs. execution sequence), a distinction that must be explicitly stated to avoid ambiguity | C-32 |
| C-35 | **Per-row axis-cost rationale content quality gate** — a dedicated auditor role or quality-verification step checks per-row axis-cost rationale cell content for substantive quality (e.g., requires a structural reason naming the axis dependency, not a generic or abbreviated value such as "low"), enforcing content quality at the role-gate or step level rather than only at the schema-declaration level where cell presence alone satisfies the requirement | C-30 V-04 PASS (role-sequence variation) — Column Contract enhanced the Evaluation Cost column to require axis-specific structural reasons and an Axis Cost Auditor role verified per-row content quality; this enforcement architecture checks that rationale cells are substantive, a quality guarantee the schema-declaration mechanism (C-26-style) cannot provide | C-30 |

**Full cascade extensions:** `C-31 → C-33` | `C-32 → C-34` | `C-30 → C-35`

The distinction between C-33 and C-31 is self-documentation: C-31 requires the invariant to be co-located at the execution step; C-33 requires that co-located annotation to include a labeled header and an explanatory sub-clause naming the reason co-location is necessary. A terse note ("NOTE: 4a FAIL does not suppress 4b") satisfies C-31; a labeled, self-explanatory block that also explains the purpose of co-location satisfies both.

The distinction between C-34 and C-32 is semantic clarity: C-32 requires a FAIL-path enforcement directive per checkpoint; C-34 requires that directive, when it includes an advancement restriction, to explicitly distinguish completion-gate semantics from execution-sequence semantics. A directive without an advancement restriction (e.g., "FAIL → populate missing rows") has no conflict to resolve and is C-34 exempt; a directive with "do not proceed until X" must carry a clarifying note to pass C-34.

The distinction between C-35 and C-30 is enforcement depth: C-30 requires per-row axis-cost rationale cells in the table; C-35 requires a quality-gate step or role that verifies the content of those cells is substantive — not merely that cells are populated. A table with populated but generic-value rationale cells ("cost: low" with no structural reason) satisfies C-30 only; one where a dedicated role rejects generic values and requires structural reasons satisfies both.

**Scoring:** denominator 24 → 27. C-33/C-34/C-35 are all vacuous-exempt for R9 (born this round). R9 composite scores are unchanged from the v9 scorecard when computed under v10.

## Criteria

### Essential (must all pass — without these the output is not useful)

| ID | Criterion | Category | Weight | Pass Condition |
|----|-----------|----------|--------|----------------|
| C-01 | **Completeness** — Every variation is a full, standalone skill body, not a diff or patch against a base. | correctness | essential | Each V-NN block can be copied into a skill file and executed without referencing any other variation. No "same as V-01 except..." constructs. |
| C-02 | **Count** — Output contains exactly N variations (default 5 when N is unspecified), labeled V-01 through V-0N in sequence. | correctness | essential | Variation count matches the requested N. Labels are sequential with no gaps. |
| C-03 | **Axis declaration** — Each variation declares its variation axis and a hypothesis, both present and non-empty. | format | essential | Every V-NN header or preamble includes an `axis:` field naming one of the six defined axes, and a `hypothesis:` field stating the expected behavioral effect. |
| C-04 | **Single-axis isolation** — Each variation changes exactly one axis relative to a shared baseline. No variation combines multiple axis changes unless the run is explicitly a combination pass. | correctness | essential | Reading two variations side-by-side, the structural difference maps to exactly one named axis. Changes that bleed across axes are a fail. |
| C-05 | **Axis diversity** — No two variations share the same axis (unless N > 6, in which case repeats are allowed only if the hypothesis tests a different setting of the same axis). | coverage | essential | For N <= 6, all variation axes are distinct. |

### Recommended (output is better with these)

| ID | Criterion | Category | Weight | Pass Condition |
|----|-----------|----------|--------|----------------|
| C-06 | **Axis breadth** — Variations collectively sample at least four of the six defined axes: role sequence, output format, lifecycle emphasis, stock role selection, phrasing register, scoring granularity. | coverage | recommended | Count distinct axes present across all variations. Pass if >= 4 of the 6 canonical axes are represented. |
| C-07 | **Hypothesis specificity** — Each hypothesis is falsifiable and names the observable behavioral outcome (not just "will be different"). | depth | recommended | Hypothesis states a concrete prediction: what the evaluator should see change in output, how to detect it, or what scoring dimension it affects. "Output will be more concise" passes; "will vary" fails. |
| C-08 | **Structural fidelity** — Each variation preserves the structural skeleton of a valid Signal skill body (frontmatter, description, steps/body, output contract). | correctness | recommended | All required skill sections are present in every variation even when their content changes. Missing sections in any variation is a fail. |

### Aspirational (raise the bar once essential/recommended are stable)

| ID | Criterion | Category | Weight | Pass Condition |
|----|-----------|----------|--------|----------------|
| C-09 | **Combination readiness** — Output includes a brief note (or appendix) identifying which axis pairs are high-priority candidates for combination runs and why. | depth | aspirational | At least one combination recommendation is present with a stated rationale referencing the hypotheses from the single-axis variations. |
| C-10 | **Evaluation order guidance** — Output suggests a sequencing for which variations to evaluate first to maximize learning per run (e.g., cheapest axis to test, most structurally impactful). | behavior | aspirational | An explicit ordering or priority tier is present, with a rationale connecting to axis cost or hypothesis independence. |
| C-11 | **Hypothesis negation clause** — Each hypothesis includes an explicit failure condition stating what outcome would disprove the prediction, not just what it predicts. | depth | aspirational | At least one hypothesis in the set contains a "if [criterion] does NOT change..." or equivalent negation clause that makes the failure mode concrete and detectable. Scoring considers the weakest hypothesis; a set where every hypothesis has a negation clause is a full pass. |
| C-12 | **Anchor variation designation** — One variation in the set is explicitly named as the structural anchor for all combination runs, with a rationale grounded in structural impact and isolation quality. | behavior | aspirational | A single variation is designated "combination anchor" (or equivalent) and the designation is justified by reference to the axis's structural centrality — not merely listed as a candidate. Presence of a rationale distinguishes this from C-09's combination candidates list. |
| C-13 | **Register-level isolation** — Across all non-phrasing-register variations, the instructional voice register (terse vs. verbose, directive vs. exploratory) is consistent. Terse phrasing in one and verbose in another is a fail unless phrasing-register is the declared axis for one of them. |
| C-14 | **Output-contract axis coherence** — When a variation introduces a new structural element (role, phase, framing), the output contract evolves to include fields that capture the artifact of that element, and those new fields are directly attributable to the variation's declared axis. | format | aspirational | New header fields or label fields present in a variation can be traced to the declared axis. Generic additions with no axis connection are a fail. Variations whose axis does not introduce new structural elements are exempt. |

A failure clause that only says "rate does not increase" does not pass; one that says "rate must exceed the R1 role-sequence baseline" passes. Variations with no prior-round evidence available are exempt. |
| C-16 | **Combination-pass structured declaration** — Combination variations carry a machine-readable `pass-type: combination` field (or equivalent) in their header, not only a prose statement. | format | aspirational | Every combination variation has a `pass-type:` field in its header block. This field is what allows automated tools (and evaluators) to trigger C-04 and C-05 exceptions without reading the body. Single-axis variations are exempt. A combination run with no combination variations is a full pass by vacuous truth. |
| C-17 | **Hypothesis-traceable combination rationale** — Each entry in the combination candidates table (C-09) cites the specific V-NN hypotheses that motivate the pairing, making the recommendation traceable to evidence rather than intuition. | depth | aspirational | Every combination candidate entry names at least one source variation by label (e.g., "based on V-02 mechanism + V-04 failure condition"). A rationale that says "these axes complement each other" without citing a variation fails. Exempt if no combination candidates table is present (C-09 already gates this). |
| C-18 | **Rubric-genealogy failure baseline** — At least one hypothesis failure condition names not only the prior-round result (C-15) but also the specific variation and rubric criterion that was born from that event, making the criterion's origin story traceable within the failure clause itself. | depth | aspirational | The failure clause names: (1) a round label, (2) a variation label (V-NN), and (3) the criterion (C-NN) created or changed as a consequence of that event. A clause that cites the round and result but not the genealogy does not pass this criterion (it passes C-15 only). Variations with no genealogy-level evidence available are exempt. |
| C-19 | **Compound-effects combination analysis** — Each combination candidate entry in the C-09 table includes a structured compound-effects model: which failure modes each contributing axis targets, the residual weakness remaining after only the first axis's improvement is applied, the expected compound criterion effect when both axes are active simultaneously, and a priority rating. | depth | aspirational | Every combination candidate entry contains all four sub-elements: (a) failure modes targeted per axis, (b) residual weakness after first axis, (c) compound criterion effect when both active, (d) priority rating. An entry with only a pairing rationale (C-09 pass) does not pass this criterion. Exempt if no combination candidates table is present. |
| C-20 | **Cross-round dependency sequencing** — The evaluation order table (C-10) explicitly names cross-round evidence dependencies for any variation whose claim requires prior-round scoring results before it can be verified, and sequences those variations after the rounds they depend on. | behavior | aspirational | At least one entry in the evaluation order states that verification depends on a specific prior-round result (e.g., "verifiable only after R2 V-01 is scored") and is ranked accordingly. An evaluation order based solely on axis cost or structural independence does not pass this criterion (it passes C-10 only). Exempt if no variation's claim depends on prior-round evidence. |
| C-21 | **Hypothesis mechanism-necessity clause** — At least one hypothesis mechanism is framed as a structural precondition using necessity language: the axis change makes the target behavior not merely more likely but unavoidable ("X is a required input to Y", "Y cannot be produced without X first"). | depth | aspirational | The mechanism clause in at least one hypothesis states a structural necessity, not a probability or tendency. Mechanism language of "tends to", "is expected to", or "improves" does not pass this criterion (it passes C-07 only); language such as "cannot be written without", "is a required input to", or equivalent necessity framing passes. Exempt if no hypothesis includes a mechanism clause at all. Layered on C-07: C-07 requires a mechanism, C-21 requires the mechanism to assert structural necessity. |
| C-22 | **Evaluative-label failure baseline** — The failure clause names round + variation + criterion (the C-18 triple) AND the evaluative ranking label that was applied to that source evidence in its scoring round (e.g., "most complete in round", "strongest in set"), making the comparison threshold the specific instance of scored excellence rather than an anonymous result. | depth | aspirational | At least one failure clause contains all four elements: (1) round label, (2) variation label (V-NN), (3) criterion (C-NN), and (4) the evaluative ranking label applied to that instance in the source round. A clause that meets C-18 requirements but omits the ranking label does not pass this criterion. Exempt if no evidence from prior rounds carried an evaluative ranking label. Layered on C-18: C-18 requires round + variation + criterion born, C-22 requires those three plus the evaluative ranking label distinguishing the source instance. |
| C-23 | **Structured-phase dependency catalog** — When the set has cross-round evidence dependencies (C-20 is applicable), the skill body includes a dedicated structural phase that catalogs dependency claims before the evaluation order phase, and the evaluation order table is explicitly derived from that catalog — not generated ad hoc or embedded in-line within the evaluation order. | behavior | aspirational | A dedicated dependency-cataloging phase (or step) is present with its own section, its output includes per-variation claims paired with the prior-round results required to verify them, and the evaluation order preamble or table notes name the catalog phase as their source. An evaluation order table that names dependencies in-line without a prior catalog phase does not pass (it passes C-20 only). Exempt if no variation's claim depends on prior-round evidence (same exemption as C-20). Layered on C-20: C-20 requires naming cross-round dependencies in the order table, C-23 requires a dedicated catalog phase whose output drives the table. |

A table whose preamble names the catalog but has no per-row attribution column does not pass this criterion (it passes C-23 only). Exempt if C-23 is exempt (i.e., no cross-round evidence dependencies). Layered on C-23: C-23 requires the preamble to name the catalog as the derivation source, C-24 requires the table itself to carry per-row source attribution. |
| C-25 | **Evaluative-label quality-dimension descriptor** — The evaluative ranking label in at least one failure clause names the *quality dimension* of the source instance (e.g., "cleanest demonstration of X", "most rigorous application of Y") — expressing *why* the instance was excellent in methodological or pedagogical terms — rather than only naming its comparative rank within the scoring round. | depth | aspirational | At least one evaluative label passes C-22 AND names a quality dimension of the instance (what made it excellent methodologically or pedagogically), not only a round-scoped rank ("most complete", "strongest in set") or milestone position ("first to clear all criteria"). A directive with an advancement restriction but no clarifying note does not pass this criterion; one that both restricts advancement and explicitly scopes the restriction to completion status passes. Exempt if C-32 is exempt or if no FAIL-path directive in the output contains an advancement restriction. Layered on C-32: C-32 requires a FAIL-path enforcement directive per checkpoint; C-34 requires that directive, when it includes an advancement restriction, to carry a semantic clarification that resolves the apparent conflict with the non-suppression invariant. |

Layered on C-30: C-30 requires per-row axis-cost rationale cells; C-35 requires a quality-gate mechanism that verifies the content of those cells is substantive. |

## Scoring

```
composite = (essential_pass / 5 * 60)
          + (recommended_pass / 3 * 30)
          + (aspirational_pass / 27 * 10)
```

Partial credit on essential criteria (e.g., C-04 PARTIAL) counts as 0.5 toward the
essential numerator. Aspirational partial credit counts as 0.5 toward the aspirational
numerator.

**Golden threshold**: all 5 essential criteria pass (no partials) AND composite >= 80.

| Band | Score | Interpretation |
|------|-------|----------------|
| Golden | all essential + >= 80 | Ship-ready output |
| Acceptable | all essential + 70-79 | Minor gaps, usable with review |
| Marginal | all essential + < 70 | Structural gaps, iterate before use |
| Fail | any essential fails | Not usable regardless of total score |
