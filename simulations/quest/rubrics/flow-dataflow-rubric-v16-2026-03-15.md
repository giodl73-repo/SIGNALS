The distinction between C-40 and C-36 is the same structural logic as C-31/C-32 vs. C-36 in v15: per-stage verification ≠ cross-stage arithmetic parity. C-40 adds the same distinction at phase granularity.
ded "CMD Binding column on every subsequent scaffold row." C-37 establishes a compliance register; C-24 establishes a scaffold; neither requires the two to be linked. V-01's signal is the column that binds them: every T-NN, PG-NN, and T-PARITY row carries a "CMD Binding" column naming which CMD-NN entries govern that artifact. Without this column, the two structures are parallel but unlinked -- a CMD violation is not automatically a scaffold violation. The binding column closes that gap.

**C-40 — Per-phase running total column in PG-NN gate blocks** (from R18 V-02)
V-02's axis was "Every PG-NN gate carries a T-07 Running Total column." C-36 requires a post-trace T-PARITY gate asserting cross-stage sum parity. V-02's signal is an intermediate verification tier: each PG-NN gate block carries an additional column showing the T-07 append-log running total at that phase boundary, creating a two-tier parity audit -- phase-close running totals at each gate plus arithmetic sum verification at T-PARITY. A gate with GO/NO-GO but no running total column satisfies C-34 but does not qualify here.

**C-41 — Dual-count checkpoint (this-stage + cumulative running total)** (from R18 V-02)
V-02's axis included "CMD-06 requires both [N] (this-stage) and [M] (running total) in checkpoint." C-32 requires CHECKPOINT [Stage ID] COMPLETE -- [N] rows appended. V-02's signal is [M] -- the cumulative running total through and including this stage -- yielding the form: CHECKPOINT [Stage ID] COMPLETE -- [N] rows appended, [M] total. The dual count makes each checkpoint self-verifying against prior checkpoints: [M] at stage k must equal [M] at stage k-1 plus [N] at stage k. A checkpoint with only [N] satisfies C-32 but fails this criterion.
