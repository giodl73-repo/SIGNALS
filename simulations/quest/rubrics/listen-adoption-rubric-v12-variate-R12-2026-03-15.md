## V-01 — Single Axis: Role Sequence (Incumbent-Integrated TABLE 1)

**Variation axis:** Role sequence — THE INCUMBENT is identified inside the TABLE 1 header
preamble, not in a standalone SECTION B. TABLE 1 adds a third column: "Incumbent Dependency:
[the workflow step this role currently relies on THE INCUMBENT to perform]." Every role's
archetype assignment is grounded simultaneously in displacement probability: low Incumbent
Dependency roles cluster in Innovator/Early Adopter; high Incumbent Dependency roles cluster
in Late Majority/Laggard. SECTION B collapses into the TABLE 1 header preamble.

**Hypothesis:** Merging THE INCUMBENT identification with role archetype assignment forces
the model to surface displacement probability at TABLE 1 level, making C-26 structurally
co-present with C-01. A model cannot complete TABLE 1 without naming THE INCUMBENT (it is in
the table header preamble) and cannot assign archetypes without grounding them in Incumbent
Dependency (it is a required column). The structural gap where C-01 passes and C-26 fails via
disconnected sections is closed at generation time.

## V-02 — Single Axis: Output Format (SLOT-KEY Row Labels in TABLE 3)

**Variation axis:** Output format — TABLE 3 row labels use an explicit `SLOT-KEY:` prefix,
extending the C-36 typed-header contract from the header preamble down to each individual row.
The row label reads "SLOT-KEY: Innovator → Early Adopter" rather than "Innovator → Early
Adopter." The TABLE 3 header preamble is updated to reference the SLOT-KEY notation. The column
header for spread mechanisms retains the C-28 inline constraint.

**Hypothesis:** A `SLOT-KEY:` prefix in the row label makes the slot-type contract visible at
two independent structural locations simultaneously — the table header preamble (C-36 baseline)
and each individual row label. A model filling the table cannot conflate the row label with a
generic description because the SLOT-KEY: prefix signals a type contract at the point of
generation. The structural gap where a model reads the header preamble once and then ignores
it during row completion is closed by embedding the type contract in every row's own label.
