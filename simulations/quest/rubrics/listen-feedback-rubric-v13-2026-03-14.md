# review-customers Rubric — v13

**v13 written.** Three new criteria extracted from Round 14 excellence signals:

---

**A-24 — Gains and losses field instructions cross-reference named Step 0 inertia sub-fields** (depth, 5 pts)

Source: V-01/V-04/V-05 differentiator. These variations structure the `Gains from this spec:` and `Losses and switching costs:` field instructions so they explicitly name the Step 0 sub-fields they draw from — e.g., "Gains: what this spec provides that the status quo (Step 0, sub-field 3 — 'Where it falls short') failed to deliver"; "Losses: the floor-level switching cost (Step 0, sub-field 4) plus any persona-specific workflow disruption." The cross-reference converts gains/losses from freeform bilateral assessments into delta computations: the evaluator cannot complete Fields 2 and 3 without consulting the named Step 0 sub-fields, making bilateral coverage structurally enforced rather than instructionally implied. A-17 requires bilateral coverage in prose; A-21 requires conjunctive framing in the instruction; A-24 adds the third layer — the instruction names the specific Step 0 sub-fields that the gains/losses fields draw from, creating a structural dependency that eliminates omission of either side. Variations without cross-reference (V-02/V-03) pass A-17 and A-21 only when both sides happen to appear; variations with cross-reference (V-01/V-04/V-05) cannot suppress either side without also suppressing the named baseline sub-field reference. Pass condition: the evaluation protocol's `Gains from this spec:` and `Losses and switching costs:` field instructions each name at least one Step 0 sub-field by label (e.g., "What it delivers," "Where it falls short," "Floor-level switching cost") that the field is required to draw from; field instructions that reference gains/losses in general terms without naming specific Step 0 sub-fields do not pass A-24.

---

**A-25 — UX-before-PM role ordering accompanied by an explicit epistemic rationale** (depth, 5 pts)

Source: V-03 differentiator. V-03 earns a full PASS on A-12 with an "explicit UX-before-PM epistemic rationale"; V-01/V-02/V-04 earn only PARTIAL because role order is unspecified or ordering is stated without explanation. Specifying UX before PM satisfies A-12's positional requirement; attaching an epistemic rationale makes the order a principled constraint rather than an arbitrary sequence choice. A rationale of the form "UX observes behavioral patterns before PM synthesizes strategic impact — behavioral observation must precede strategic synthesis to prevent PM framing from anchoring UX reading" means a reviewer cannot swap the order without also invalidating the stated reasoning. Without a rationale, the UX-before-PM constraint is positionally verifiable but epistemically arbitrary — a later variation author could reverse it without contradiction. With a rationale, reversing the order introduces an explicit logical inconsistency. Pass condition: the evaluation protocol specifies UX Read before PM Read (satisfying A-12) AND includes a stated epistemic or procedural reason for that ordering — naming why UX observation should precede PM synthesis — as part of the professional lens section instructions; a bare ordering instruction without rationale does not pass A-25 even if A-12 passes.

---

**A-26 — Aggregate section uses a numbered field manifest with positional verifiability** (format, 5 pts)

Source: V-02/V-04/V-05 differentiator. A-23 applies a numbered field manifest with completeness enforcement to persona cards; V-02/V-04/V-05 extend the same structural guarantee to the aggregate section by specifying aggregate fields as a numbered manifest (e.g., A1 — Mean NPS; A2 — Band distribution; A3 — Dominant Detractor objection; A4 — Threshold verdict). A-10 requires the `Dominant Detractor objection:` labeled field to be present and is verified semantically — a reviewer reads field names to detect absence. A numbered aggregate manifest makes every aggregate field positionally verifiable: A3 is detectable by counting, not by parsing labels. An explicit completeness enforcement rule attached to the manifest closes the compliance gap. The structural parallel with A-23 means the same verification property holds at both levels of the output — persona card and aggregate summary — making the entire output positionally auditable without semantic reading. An output can pass A-10, A-23, and all other aggregate criteria while failing A-26 if the aggregate section uses labeled fields without numbering. Pass condition: the evaluation protocol specifies aggregate section fields using an explicit numbered list (A1, A2, … AN, or an equivalent sequential numbering scheme) and includes a stated completeness enforcement rule that prohibits omitting any numbered aggregate field; a labeled-field aggregate template without sequential numbering does not pass A-26 even if A-10 passes.
