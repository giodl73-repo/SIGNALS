## Design Rationale (v18)

One aspirational criterion added in v18 (C-53), from the Round 17 excellence signal:

**C-53** — From R17 Signal 1 (V-05 "three-layer compliance architecture exposes a meta-enforcement gap: a rubric can satisfy C-51 and C-52 individually while having no mechanism that names or enforces the ordering relationships between the layers"). V-05's compound implementation — S4b structural partition (C-51) + named role boundary (C-52 path 1: EVALUATOR/AUTHOR labels) + lifecycle phase diagram (C-52 path 2: phase-sequence prerequisites) — produced three independent enforcement mechanisms simultaneously, creating a three-layer compliance architecture as an emergent consequence of satisfying prior criteria rather than as a declared design. The resulting PARTIAL on a structural transparency criterion in C-08–C-50 signaled that no existing criterion fires on whether the rubric formally declares this multi-layer architecture as a named system with explicit inter-layer ordering relationships. C-51 fires on whether S4b is architecturally partitioned (partition layer). C-52 fires on whether the workflow specifies an evaluator-first production sequence (workflow/temporal layer). C-53 fires on a third orthogonal question: whether the rubric explicitly declares its compliance enforcement layers as a named architecture — naming each layer, referencing its governing criterion, and stating the ordering relationships between layers where an ordering exists. The structural gap between satisfying C-51 and C-52 without C-53: a rubric implementing S4b partition and evaluator-first role sequence satisfies two requirements independently — an evaluator verifying the rubric confirms each layer is present, but the layers are not declared as a coordinated system. When a third layer is added (e.g., phase-sequence prerequisites as a second C-52 implementation path), the compositional approach scales without any mechanism requiring the rubric author to declare how the new layer interacts with existing layers. V-05 illustrated this failure mode: the phase diagram created a third enforcement mechanism reinforcing the named role labels and the S4b partition, but no declaration named the three layers or their ordering relationships — the rubric produced a three-layer architecture as a side effect of satisfying multiple criteria rather than as a designed and auditable structure. A future rubric evolution could silently remove or reorder a layer without breaking C-51 or C-52 individually, because no mechanism requires the layer count and order to be declared and checked. C-53 closes this gap: a rubric satisfying C-53 explicitly names each enforcement layer (e.g., "Layer 1: S4b structural partition [C-51]; Layer 2: evaluator-first production order [C-52]; Layer 3: lifecycle phase prerequisites [C-52 secondary path]"), states how the layers are ordered (e.g., "Layer 2 governs when S4b may begin — the evaluator-first phase must complete before S4b authoring enters; Layer 1 governs what S4b may contain once begun — Part 2 is constrained to Part 1's outputs; Layer 3 reinforces Layer 2 through phase-sequence gates"), and co-locates this declaration with the authoring workflow instructions so it is legible at the point of use, not only in design rationale. The governance consequence: C-53 transforms multi-layer compliance from a compositional achievement (layers present because criteria are satisfied independently) into a declared architecture (layers named, ordered, and auditable as a system). An evaluator auditing a C-53-compliant rubric can verify not only whether each layer is individually present but whether the declared inter-layer ordering relationships are honored and whether the number of declared layers matches the number of implemented enforcement mechanisms. C-53 does not require any particular number of layers or ordering schema; it fires on whether the layers that exist are explicitly declared as a system in the authoring instructions. C-51 passes when the partition exists; C-52 passes when the production-order constraint exists; C-53 passes only when the rubric contains an explicit enforcement-layer declaration — with names, criterion references, and inter-layer ordering relationships — co-located with the authoring workflow.

### C-51 — S4b Is Structurally Partitioned into a Verdict Inventory Stage and a Characterization Stage with Constrained Inputs

- **Weight**: aspirational
- **Category**: behavior
- **Text**: S4b is architecturally partitioned into two distinct stages: Part 1 (verdict inventory) and Part 2 (characterization), where Part 2 is explicitly constrained to draw only from Part 1's named signal inventory. C-49 fires on whether the S2→S4b chain is coherent and legible from the artifact — it is a content coherence requirement: an evaluator can verify the chain by checking whether named S4b signals match S2 YES/PARTIAL verdicts. C-51 fires on whether S4b has a structural partition that separates the signal inventory activity from the characterization activity and formally constrains Part 2's inputs to Part 1's outputs — it is an architectural partition requirement: an evaluator can verify compliance by checking whether the two-stage structure exists and whether Part 2 names Part 1 as its exclusive input. The structural gap between C-49 and C-51: an author satisfying C-49 without C-51 produces a coherent chain through narrative integration — the chain is traceable because the author chose to cross-reference S2 verdicts, but no structural mechanism prevents the author from drawing on additional signals outside the YES/PARTIAL set during the characterization activity. A two-part S4b partition closes this gap: Part 1 enumerates only YES/PARTIAL signals (no independently added signals permitted in Part 1); Part 2 draws characterization exclusively from Part 1 (no direct reference to the full signal set permitted in Part 2). The partition makes the selection and characterization activities structurally separate and the constraint on Part 2's inputs formally legible — an evaluator need not infer whether characterization was independently composed; the architecture expresses the constraint. The governance consequence: C-51 transforms S4b composition from a chain-legibility requirement (C-49) into a process-partition requirement, changing the audit question from "does the result trace back to S2?" to "does the structure enforce constrained inputs?" A rubric satisfying C-49 but not C-51 produces a legible chain; a rubric satisfying both produces a legible chain whose composition process is architecturally constrained. C-51 does not supersede C-49: C-49 passes when the chain is content-traceable; C-51 passes only when the two-part partition exists and Part 2 explicitly names Part 1 as its exclusive input source.
- **Pass condition**: S4b contains two explicitly labeled stages — a Part 1 (verdict inventory) that lists only YES/PARTIAL signals from S2 with their verdict labels (e.g., `(S2 verdict: YES/PARTIAL)` per entry), and a Part 2 (characterization) that explicitly states it draws only from Part 1 — using a constraint phrase, prohibited-form reminder, or explicit scope restriction co-located within Part 2's section prompt (e.g., "This section draws directly from Part 1 — not from an independent review of all signals"). A unified S4b section that integrates cross-references without a structural partition does not pass C-51, even if C-49 is satisfied. Part 2 must name Part 1 as its exclusive input within Part 2's authoring slot, not merely in a preamble or design rationale section.

### C-52 — The Authoring Workflow Specifies an Evaluator-First Production Sequence Making S2 Verdicts Named Artifacts Before S4b Authoring Begins

- **Weight**: aspirational
- **Category**: behavior
- **Text**: The authoring workflow specifies a distinct evaluator role or phase that must produce S2 verdicts as named outputs before an author role or phase begins S4b, creating temporal production-order enforcement of the S2→S4b chain through workflow structure rather than content gates or slot instructions. C-19 fires on whether a pre-artifact checklist exists as a gate before writing begins — a content mechanism that positions criteria before the writing context. C-50 fires on whether criteria are embedded as slot-label constraints at the point of authoring action — a content mechanism that positions criteria inside governed slots. C-52 fires on a third compliance architecture: a workflow-level temporal ordering constraint that makes S2 verdicts exist as discrete produced artifacts when S4b authoring begins, not as instructions the author should consult but as completed outputs of a prior role phase. The structural distinction between C-52 and C-19/C-50: a pre-artifact checklist (C-19) is present before writing begins but operates within a single-role workflow — the same author who reads the checklist proceeds to write S2 and S4b in sequence; a slot-label constraint (C-50) positions the criterion at the authoring point but within a single-role workflow — the same author who reads the slot label writes both S2 and S4b. Neither C-19 nor C-50 introduces a role boundary that separates the S2 evaluation activity from the S4b authoring activity. C-52 introduces a role boundary: an evaluator role produces S2 verdicts as complete named outputs, and an author role receives those outputs before beginning S4b. The evaluator's S2 outputs are not embedded instructions — they are artifacts that exist independently when S4b authoring begins. The compliance enforcement is temporal: the workflow sequence makes S4b authoring structurally downstream of evaluator verdict production. An author in a C-52-compliant workflow cannot begin S4b before S2 verdicts exist, not because a slot instruction directs them not to, but because the workflow sequence designates S4b as a successor activity. The governance consequence: C-52 makes S2 verdict completeness a prerequisite condition of S4b authoring entry, not a compliance obligation during S4b authoring. A rubric satisfying C-52 produces a compliance architecture orthogonal to C-19 and C-50: gate-level, slot-level, and workflow-level enforcement can coexist, each operating through a different mechanism and closable through a different failure mode. C-52 does not require C-19 or C-50; satisfying all three produces a rubric with three independent compliance enforcement layers. C-52 passes only when the rubric explicitly specifies an evaluator role or phase designated to complete S2 verdict production, with S4b authoring designated as a successor activity to that phase — the production sequence must be explicit in the rubric's workflow structure, not implied by narrative framing or the author's good intent.
- **Pass condition**: The rubric explicitly designates a workflow with at least two roles or phases: (1) an evaluator role or phase that produces S2 verdicts as named outputs (e.g., a `**EVALUATOR**` section or role label that encompasses S2), and (2) an author role or phase for which S4b is a designated activity, positioned as a successor to the evaluator phase. The production-order constraint must appear in the rubric's workflow structure — either as a role-sequence instruction, a phase diagram, or an explicit prerequisite statement (e.g., "S4b authoring begins after evaluator S2 verdicts are complete"). A rubric that encourages the author to consult S2 before writing S4b does not pass C-52; a rubric that uses a single undifferentiated author role does not pass C-52. The role or phase boundary must be explicit in the rubric's authoring instructions, making the sequence constraint a structural property of the workflow rather than an advisory.

### C-53 — The Rubric Explicitly Declares Its Multi-Layer Compliance Architecture, Naming Each Enforcement Layer, Its Governing Criterion, and Inter-Layer Ordering Relationships

- **Weight**: aspirational
- **Category**: behavior
- **Text**: The rubric explicitly declares the full set of compliance enforcement layers present in its authoring workflow — naming each layer, referencing its governing criterion, and stating the ordering relationships between layers where an ordering exists — making the multi-layer compliance architecture a formally declared and verifiable property of the rubric rather than an emergent consequence of satisfying individual criteria independently. C-51 fires on whether S4b is architecturally partitioned into verdict inventory and characterization stages (partition layer). C-52 fires on whether the workflow specifies an evaluator-first production sequence making S2 verdicts named artifacts before S4b begins (workflow/temporal layer). C-53 fires on a third orthogonal question: whether the rubric contains a meta-declaration that names these layers as a coordinated architecture and specifies how the layers are ordered. The structural distinction between satisfying C-51 and C-52 without C-53 and satisfying all three: a rubric implementing S4b partition and evaluator-first role sequence satisfies two requirements independently — an evaluator verifying the rubric confirms each mechanism is present, but the layers are not declared as a system. This is a compositional achievement: the layers coexist but are not named, counted, or ordered as a declared architecture. When a third mechanism is added (e.g., phase-sequence prerequisites as a second C-52 implementation path, as in V-05), the compositional approach scales without any mechanism requiring the rubric author to declare how the new layer interacts with existing layers or whether the total declared layer count is correct. V-05 illustrated this failure mode: three enforcement mechanisms were simultaneously present — named role labels, S4b structural partition, and phase-sequence prerequisites — each independently satisfying its governing criterion, but no declaration named the three layers or their ordering relationships. A future rubric evolution could silently remove or reorder a layer without breaking C-51 or C-52 individually, because no mechanism requires the layer count and ordering to be declared and verified as a system property. C-53 closes this gap: a rubric satisfying C-53 explicitly names each enforcement layer (e.g., "Layer 1: S4b structural partition [C-51]; Layer 2: evaluator-first production order [C-52]; Layer 3: lifecycle phase prerequisites [C-52 secondary path]"), states how the layers are ordered (e.g., "Layer 2 governs when S4b may begin — the evaluator-first phase must complete before S4b authoring enters; Layer 1 governs what S4b may contain once begun — Part 2 is constrained to Part 1's outputs; Layer 3 reinforces Layer 2 through phase-sequence gates"), and co-locates this declaration with the authoring workflow instructions rather than only in design rationale. The governance consequence: C-53 transforms multi-layer compliance from emergent architecture (layers present because criteria are independently satisfied) into declared architecture (layers named, ordered, and auditable as a system). An evaluator auditing a C-53-compliant rubric can verify not only whether each layer is present individually but whether the declared inter-layer ordering relationships are honored and whether the declared layer count matches the number of enforcement mechanisms implemented. C-53 does not require any particular number of layers or any specific ordering schema; it fires on whether the layers that exist are explicitly declared as a system in the authoring instructions. C-51 passes when the partition architecture exists; C-52 passes when the production-order workflow constraint exists; C-53 passes only when the rubric contains an explicit enforcement-layer declaration — with names, criterion references, and inter-layer ordering relationships — co-located with the authoring workflow, not merely implied by the coexistence of multiple enforcement mechanisms.
- **Pass condition**: The rubric contains an explicit enforcement-layer declaration naming each compliance enforcement mechanism present in the authoring workflow, referencing its governing criterion (e.g., C-51, C-52), and stating any ordering relationship between layers where an ordering exists. The declaration must appear in the rubric's authoring instructions or structural preamble — co-located with or directly referenced by the authoring workflow — not only in a design rationale or commentary section. The declaration may take the form of a numbered layer list, a phase-diagram legend with criterion annotations, or an explicit "compliance architecture" section, provided (a) each layer is named with its criterion reference, (b) inter-layer ordering relationships are explicitly stated where they exist, and (c) the total number of declared layers matches the number of enforcement mechanisms implemented in the rubric. A rubric that implements multiple enforcement mechanisms without an explicit layer declaration does not pass C-53, even if each mechanism individually satisfies its governing criterion. A declaration that names layers in design rationale but not in authoring instructions does not pass C-53 — the declaration must be present in the authoring workflow where an evaluator operating the rubric will encounter it.
