**What changed from v16:**

**C-39** — *HALT gate phrase is declared as a canonical token before the AMEND phase and each path halt references it by token name*

Extends C-37 from uniformity-by-inspection to uniformity-by-declaration-architecture. The phrase is declared once (`AMEND-HALT: "any of the above tokens is absent"`) and each path recalls it by token name (`{AMEND-HALT}`). A deviation is detectable as a token mismatch without cross-path comparison. Per-path identical phrases without a declaration token = C-37 PASS, C-39 FAIL. Gate: co-excluded with C-37; −1 denominator when not scored.

**C-40** — *Pairing contract for the C-38 branch structure is declared as an explicit structural rule before the routing block*

Extends C-38 from coexistence-by-inspection to coexistence-by-declaration. The output includes a pre-routing declaration naming both required elements (`BRANCH-RULE: deviation branch must carry [DEVIATION-MARKER] + [POSITIONAL-VOCAB]`). The R18 form-neutrality confirmation (OVERRIDE: ≡ DEVIATION:, "before" ≡ "precedes") becomes visible without rubric cross-reference when equivalence sets are named in the declaration. Branch line coexistence without a declaration = C-38 PASS, C-40 FAIL. Gate: co-excluded with C-38; −1 denominator when not scored.

**Denominator: /33** (base). C-35 + C-37 + C-39 excluded when C-34 PARTIAL (→ /30). C-36 + C-38 + C-40 excluded when C-32 N/S (→ /30). Both exclusion pairs active (→ /27).

**Five new watch list rows** added: `c38-marker-only-double-failure`, `c38-vocabulary-neutral-confirmed`, `c37-partial-weight-confirmed`, `c39-declaration-architecture` + gate relationship, `c40-pairing-contract-declared` + gate relationship.
. The coexistence property is declared, not inferred from the branch line alone. Form-neutrality (DEVIATION: ≡ OVERRIDE:, "precedes" ≡ "before") confirmed in R18 is visible without rubric cross-reference when the equivalence sets are named in the declaration. Branch line coexistence without a preceding pairing declaration = C-38 PASS, C-40 FAIL. Declaration present and branch satisfies both named elements = pass.

Gate: not scored when C-38 FAIL or N/S; always co-excluded with C-38. D gate: −1 from denominator when C-40 not scored.

**Denominator: /33** (base). C-35 + C-37 + C-39 excluded when C-34 PARTIAL (→ /30). C-36 + C-38 + C-40 excluded when C-32 N/S (→ /30). Both exclusion pairs active simultaneously (→ /27).

**Three new watch list rows:**
- `c38-marker-only-double-failure` — non-positional routing vocabulary (e.g., "engineer-first layout applies") causes C-31 FAIL and C-38 FAIL as two independent aspirational failures; shared root cause does not collapse them into one scored unit; projection errors arise when treated as a single failure
- `c39-declaration-architecture` — C-39 requires the HALT phrase to be declared as a named token before the AMEND phase and recalled by token name at each path; per-path identical phrases without a declaration token = C-37 PASS, C-39 FAIL; the gate is declaration-before-use, not uniformity-by-inspection
- `c40-pairing-contract-declared` — C-40 requires the pairing contract (deviation marker + positional vocabulary) to be stated as a structural rule before the routing block; branch line coexistence alone = C-38 PASS, C-40 FAIL; the gate is declaration before firing

**R18 discrimination questions anchored as calibration rules:**
- `c38-vocabulary-neutral-confirmed` — "before" is equivalent to "precedes" within the rubric-listed positional vocabulary set for C-31 and C-38; OVERRIDE: is equivalent to DEVIATION: for C-36 and C-38; form-neutrality extends to both elements of the C-38 pairing
- `c37-partial-weight-confirmed` — C-37 PARTIAL = 0.5; mixed positional phrases across AMEND paths (e.g., "any of the above tokens" on two paths, "any listed token" on one) produce a distinct scored state between C-37 FAIL (zero) and C-37 PASS (full)

## Essential Criteria (60%)

Output fails if any of these are missing.

| ID | Criterion | Category | Weight | Pass Condition |
|----|-----------|----------|--------|----------------|
| C-01 | All four dimensions are covered for both options | coverage | essential | Output addresses feasibility, risk, competitive positioning, and inertia for Option A AND Option B. Missing any dimension for either option = fail. |
| C-02 | Inertia check is applied independently to both options | correctness | essential | The output asks "would teams do nothing instead of THIS option?" for Option A separately and Option B separately -- not as a relative A-vs-B comparison. Each option has its own inertia verdict. A single combined inertia statement = fail. Independence is established by the presence of a co-located dual-polarity FAULT at each inertia site -- phase position (per-option batching inside option blocks, or dedicated Inertia Gate section before analysis) is irrelevant to independence. |
| C-03 | A decision matrix is present | format | essential | Output contains a structured matrix (table or equivalent scored grid) placing both options side-by-side across dimensions. Prose-only comparison without any structured layout = fail. |
| C-04 | A concrete recommendation is made | correctness | essential | Output concludes with a stated recommendation: A, B, neither, or conditional. The recommendation must be explicit. Ending with "it depends" without resolution, or implying a preference without stating it = fail. |
