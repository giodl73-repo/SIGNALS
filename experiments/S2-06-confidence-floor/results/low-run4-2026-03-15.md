## Bugs

**F-01: `SPEC_FILE` defined but never used, and hardcodes "scout"**
`SPEC_FILE` is set on line 16 with `signal-scout-$TODAY.md` but is never referenced anywhere in the script. The "scout" is also hardcoded rather than derived from `$SKILL_ID`, so it would be wrong for any non-scout skill.

**F-02: `VARS_CONTENT` defined but never used**
Line 251 assigns `VARS_CONTENT=$(cat "$LAST_VARIATIONS" | head -100)` but the QU5 Python script reads `$LAST_VARIATIONS` directly via `open(sys.argv[3])`. Dead assignment.

**F-03: QU5 fallback copies the full variations file, not the winning variation**
When `claude --print` fails in the QU5 step, the fallback does `cp "$LAST_VARIATIONS" "$SIMPLIFIED_FILE"`. `$LAST_VARIATIONS` is the full 5-variation file from the last round, not the winning variation. The golden step downstream then tries to extract the winner from what is now an unfiltered dump.

**F-04: Shell injection in inline Python via `$SKILL_ID`**
In Steps 2 and 5, Python source code is constructed by directly interpolating `$SKILL_ID` inside a single-quoted Python string literal. A skill ID containing a single quote (e.g., `foo'bar`) would produce a Python syntax error. Low probability given the ID format, but fragile.

**F-05: JSON regex fragile — `{[^}]+}` breaks if pattern strings contain `}`**
All three parse blocks use `r'\`\`\`json\s*({[^}]+})\s*\`\`\`'`. If any string in `new_patterns` contains a literal `}`, the regex terminates at the first occurrence. The expected format is safe most of the time, but this is a silent parse failure: the except path silently returns `0` / `false`.

## Logic Gaps

**F-06: No recovery when `all_essential=false` and `new_patterns=0`**
The plateau reset happens in the `else` branch, which fires when either condition fails. The rubric evolution fires only when `new_patterns > 0`. So when essential criteria are failing but Claude found no new patterns, the script loops silently to `MAX_ROUNDS` with no rubric changes and no convergence. There is no escalation path (e.g., try harder variations, widen the rubric scope).

**F-07: Golden file written as `status: GOLDEN` regardless of final score**
After the loop, the script always writes `status: GOLDEN` and updates `signal.skills.yaml` with `quest_status: GOLDEN`. If the run hit `MAX_ROUNDS` without meeting the threshold (composite < 80 or essential not all passing), the result is misleadingly labeled. Should be `BEST_EFFORT` or `TIMEOUT` with the actual score.

**F-08: Score threshold defined in rubric prompt but never checked in convergence logic**
The rubric prompt specifies "Golden threshold: all essential pass + composite >= 80." The convergence check in the loop only tests `all_essential_pass` and `new_patterns == 0`. A run that plateaus at score 65 (all essential pass, no new patterns) would converge and be written as golden.

## Reliability Issues

**F-09: Rubric truncated at 80 lines in score, variate, and evolve prompts**
`head -80` is used when embedding the rubric into VARIATE_PROMPT, SCORE_PROMPT, and EVOLVE_PROMPT. A rubric with frontmatter, preamble, and 8+ criteria can easily exceed 80 lines. Truncated criteria mean scoring is done against an incomplete rubric, silently undermining the entire loop.

**F-10: `mktemp` used directly in QU5 but deliberately avoided in `claude_ask`**
`claude_ask` explicitly avoids `mktemp` due to Windows path issues in the relay context (`/tmp/claude-ask-$$.txt` instead). Line 250 uses `mktemp /tmp/qu5-prompt-XXXXXX.txt` directly. If `mktemp` returns a Windows-style path in this context, the QU5 step breaks while `claude_ask` calls succeed.

**F-11: Temp file not cleaned up on `set -e` exit from `claude_ask`**
If `claude --print` fails and `set -e` propagates the error before reaching `rm -f "$tmp_prompt"`, the temp file leaks. Low impact in practice but accumulates under repeated failures.

**F-12: No guard if `LAST_VARIATIONS` or `LAST_SCORECARD` is empty entering QU5/golden**
Both are initialized to `""` and only set inside the loop body. `MAX_ROUNDS=20` makes this safe today, but if the loop were parameterized to 0 or skipped, `cat ""` fails under `set -e` with no diagnostic message.

## Minor

**F-13: No check that `claude` CLI is available in PATH**
The script fails with a cryptic "command not found" error if the relay context doesn't have `claude` on PATH. A preflight `command -v claude || { echo "claude not found"; exit 1; }` would make the failure obvious.

**F-14: Temp file name collision if `claude_ask` called concurrently**
`/tmp/claude-ask-$$.txt` uses PID for uniqueness. Multiple concurrent invocations of the script from the same PID (subshell patterns) would collide. Not an issue in the current linear flow, but relevant if the script is ever parallelized.
