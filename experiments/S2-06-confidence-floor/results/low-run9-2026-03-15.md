## Findings

### F-01 — Dead variable: `SPEC_FILE` defined but never used
`SPEC_FILE` is assigned at the top and never referenced. It also hardcodes `signal-scout-` instead of using `$SKILL_ID`, so it would be wrong for every skill except scout variants.

---

### F-02 — Dead computation: `VARS_CONTENT` and `RUBRIC_CONTENT` unused
In the QU5 section, both variables are computed with `cat | head` but then discarded — the Python script reads those files directly via `sys.argv`. Wasted subprocess and memory.

---

### F-03 — Convergence never checks the composite score threshold
The rubric defines golden as "all essential pass AND composite >= 80." The loop only checks `ALL_ESSENTIAL=true AND NEW_PATTERNS=0`. A run with score 65, all essential passing, and no new patterns converges and writes a golden file — even though composite is below threshold.

---

### F-04 — JSON regex breaks on `}` inside string values
The pattern `{[^}]+}` used in all three Python parsing blocks fails if any `new_patterns` entry contains a `}` character (e.g. `"pattern with {braces}"`). A real JSON parser on the extracted block would be safer.

---

### F-05 — Golden file always gets `status: GOLDEN`, even on max-rounds timeout
The loop exits naturally at `MAX_ROUNDS` if convergence never happens. The golden file is still written with `status: GOLDEN` and `score: $LAST_SCORE` (potentially low). There's no differentiation between a true convergence and hitting the cap.

---

### F-06 — YAML round-trip via PyYAML destroys formatting
`yaml.dump()` in Step 5 re-serializes the entire file. Any comments, blank-line grouping, custom key ordering beyond `sort_keys=False`, or multi-line string style in `signal.skills.yaml` is lost on every run. This is a well-known PyYAML round-trip hazard.

---

### F-07 — Rubric truncated to 80 lines in scoring and evolve prompts
`$(cat "$RUBRIC_FILE" | head -80)` is the rubric injected into the score prompt, the evolve prompt, and the golden prompt. If the rubric grows beyond 80 lines (which it will as it evolves), the model is scoring against an incomplete rubric. Same issue with variations capped at 200 lines for scoring.

---

### F-08 — `ls` used for trace file glob
`ls "$TRACE_DIR"/plugin-${SKILL_ID}-*.md` is the antipattern for file discovery. A shell glob (`for f in "$TRACE_DIR"/plugin-${SKILL_ID}-*.md`) handles spaces and avoids `ls` output parsing. The `|| true` suppresses the error correctly, but the underlying pattern is fragile.

---

### F-09 — Useless `cat | head` (UUOC throughout)
All prompt injections use `$(cat "$FILE" | head -N)`. The `cat` is unnecessary — `$(head -N "$FILE")` is equivalent and avoids a subprocess.

---

### F-10 — `mktemp` inconsistency between `claude_ask` and QU5
`claude_ask` avoids `mktemp` explicitly because it "may return Windows paths in relay context." QU5 then uses `mktemp /tmp/qu5-prompt-XXXXXX.txt` anyway. Either the mktemp concern applies everywhere (fix `claude_ask` or use an explicit path pattern) or it doesn't (use `mktemp` everywhere). The inconsistency undermines the documented reasoning.

---

### F-11 — `SKILL_ID` injected into Python `-c` inline strings
`python -c "... if s['id'] == '$SKILL_ID' ..."` — if SKILL_ID ever contains a single quote or backslash, it breaks the Python. All three Python inline calls in Steps 2 and the JSON parsing blocks share this. The QU5 block correctly uses `sys.argv` to pass values; the earlier blocks don't.

---

### F-12 — QU5 fallback copies all variations, not the winning one
When QU5 fails, `cp "$LAST_VARIATIONS" "$SIMPLIFIED_FILE"` copies the full variations file (all 5 V-01 through V-05). The golden prompt then reads 80 lines of this and asks Claude to "use the winning variation" — correct instruction, but the model has to re-identify the winner from the blob rather than receiving it directly.

---

### F-13 — No final exit status for max-rounds non-convergence
When the loop hits `MAX_ROUNDS` without converging, the script continues silently to write golden artifacts. A warning to stdout (or a non-zero exit) would let callers distinguish a successful quest from a timed-out one.

---

### F-14 — `TOP_SCORE` never gates convergence — only `ALL_ESSENTIAL` and `NEW_PATTERNS` do
`TOP_SCORE`/`LAST_SCORE` is extracted each round and logged, then written to the golden frontmatter. But it plays no role in the convergence check. Score could be 50 and the loop still converges if essentials pass and patterns stop. The score is computed for display but has no behavioral effect.
