## Review: `quest-run-one.sh`

---

### F-01 — Bug: `SPEC_FILE` hardcodes `signal-scout-` regardless of skill

```bash
SPEC_FILE="$REPO/simulations/draft/specs/signal-scout-$TODAY.md"
```

The filename bakes in `signal-scout-` rather than `$SKILL_ID`. Running with `draft-proposal` or `flow-resilience` will point at the wrong file. Should be `signal-${SKILL_ID}-$TODAY.md` (or at minimum extract the namespace from the ID).

---

### F-02 — Bug: `SPEC_FILE` is defined but never used

`SPEC_FILE` is set on the line above yet never referenced again anywhere in the artifact. Either the variable is dead code, or it was meant to be passed into the rubric prompt alongside `TRACE_CONTENT` and was accidentally omitted.

---

### F-03 — Security: shell injection via `SKILL_ID` embedded in Python source

```python
python -c "
...
    if s['id'] == '$SKILL_ID':
```

`$SKILL_ID` is taken directly from `$1` and interpolated into Python source code. A value like `'; import os; os.system('rm -rf /')  #` would execute arbitrary code. At minimum, pass it as an environment variable and read it with `os.environ` inside the Python snippet.

---

### F-04 — Silent data loss: trace truncation has no warning

```bash
${TRACE_CONTENT:0:3000}
```

Trace content is silently truncated to 3,000 characters with no log message. If the trace is larger, the rubric prompt gets incomplete ground truth without any indication. Add a size check and an `echo "WARNING: trace truncated"` so the operator knows.

---

### F-05 — Fragile file discovery with `ls`

```bash
TRACE_FILE=$(ls "$TRACE_DIR"/plugin-${SKILL_ID}-*.md 2>/dev/null | head -1 || true)
```

`ls` output is locale-sorted and breaks on filenames containing spaces or newlines. `head -1` silently picks one file when multiple matches exist. Prefer a glob directly in bash:

```bash
files=("$TRACE_DIR"/plugin-${SKILL_ID}-*.md)
TRACE_FILE="${files[0]}"
```

This also avoids the `ls` antipattern.

---

### F-06 — Prompt expansion risk: `SKILL_DESC` and `TRACE_CONTENT` in a double-quoted string

```bash
RUBRIC_PROMPT="You are running /quest:rubric for the Signal plugin skill: $SKILL_ID
...
$SKILL_DESC
...
${TRACE_CONTENT:0:3000}
..."
```

If `SKILL_DESC` or `TRACE_CONTENT` contain backticks (`` ` ``) or `$(...)` sequences, the shell will attempt to evaluate them during assignment. This is unlikely in practice but is a latent correctness hazard. Use a `printf` with `%s` or a heredoc with a quoted delimiter (`<<'EOF'`) to prevent expansion of external data.

---

### F-07 — `MAX_ROUNDS` is declared but unused in this artifact

```bash
MAX_ROUNDS=20  # experimental -- let the rubric discover
```

`MAX_ROUNDS` is never referenced anywhere in the provided script. If it drives the variate loop in later steps (not shown), it should be validated to be a positive integer. If it's dead code in the current draft, removing it reduces confusion.
