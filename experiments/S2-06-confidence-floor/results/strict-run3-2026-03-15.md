## Review: `tools/quest-run-one.sh`

---

### F-01 — Shell injection via `$SKILL_ID` in Python inline code [BUG, SEVERITY: HIGH]

**Evidence:**
```bash
SKILL_DESC=$(python -c "
...
for s in data.get('skills', []):
    if s['id'] == '$SKILL_ID':
```

`$SKILL_ID` comes from `"${1:-scout-feasibility}"` with no sanitization. A value containing a single quote (e.g., `foo' ; import os; os.system('rm -rf /') #`) breaks Python string context and executes arbitrary code. In a relay context where the input is JSON-driven, this is reachable.

**Fix:** Write the YAML lookup as a Python script file, or pass `$SKILL_ID` as an environment variable: `SKILL_ID="$SKILL_ID" python -c "import os, yaml; ..."` and read `os.environ['SKILL_ID']` inside Python.

---

### F-02 — `SPEC_FILE` hardcodes `signal-scout` instead of `$SKILL_ID` [BUG]

**Evidence:**
```bash
SPEC_FILE="$REPO/simulations/draft/specs/signal-scout-$TODAY.md"
```

The script is generic (`SKILL_ID="${1:-scout-feasibility}"`), but `SPEC_FILE` is hardwired to `signal-scout`. For any skill other than a scout variant, this points to the wrong file. Should be `signal-${SKILL_ID}-$TODAY.md`.

---

### F-03 — `MAX_ROUNDS` defined but never used [DEAD CODE]

**Evidence:**
```bash
MAX_ROUNDS=20  # experimental -- let the rubric discover
```

`MAX_ROUNDS` appears nowhere else in the artifact. Either the loop termination logic was never implemented, or the variable was orphaned during a refactor. The comment "experimental -- let the rubric discover" suggests intent exists but code does not.

---

### F-04 — Concurrent `claude_ask` calls from the same PID share the same temp file [LATENT BUG]

**Evidence:**
```bash
tmp_prompt="/tmp/claude-ask-$$.txt"
```

`$$` is the PID of the running shell. The header states *"Writes all artifacts to simulations/quest/ independently (P-01 concurrent by default)"*, suggesting future parallel execution. If `claude_ask` is ever called in background subshells of the same script (`&`), they will race on `/tmp/claude-ask-<same-pid>.txt`. Use `mktemp` or incorporate `$RANDOM`/a counter: `/tmp/claude-ask-$$-${RANDOM}.txt`.

---

### F-05 — Python errors silently replaced with misleading fallback [RELIABILITY]

**Evidence:**
```bash
" 2>/dev/null || echo "Skill not found in registry")
```

`2>/dev/null` suppresses all stderr including YAML parse errors, import errors, and the F-01 injection failure. The fallback `"Skill not found in registry"` will then be passed to the LLM as the skill description, producing a plausible-looking but incorrect rubric with no signal that anything went wrong.

**Fix:** Redirect only the expected "not found" path; let actual errors surface, or capture stderr and check it separately.

---

### F-06 — Rubric composite score formula is ambiguous in the prompt [QUALITY]

**Evidence:**
```
Composite score = (essential_pass/N * 60) + (recommended_pass/N * 30) + (aspirational_pass/N * 10)
```

`N` is used as the denominator for all three tiers, but each tier has a different count. The formula should use per-tier denominators (e.g., `essential_pass/essential_N`). As written, the LLM is likely to interpret this inconsistently across runs, producing rubrics with non-comparable scores.
