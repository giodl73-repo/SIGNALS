Here are the findings:

---

## Bugs

**B-01: `set -e` will abort on non-zero `claude_ask` exit**
`claude_ask` captures `$rc` and returns it, but `set -e` means any non-zero exit from `claude` will kill the whole script mid-run. If claude returns a non-zero code on a partial/warning exit, you lose all progress with no recovery path.

**B-02: `$ROUND` used after the loop without `break` guarantee**
At Step 4, `$ROUND` is referenced in the golden frontmatter (`rounds: $ROUND`). If the loop exhausts all `MAX_ROUNDS` without hitting `break`, `$ROUND` retains the last iteration value — which is correct in bash — but the intent is ambiguous. No distinction is made between "converged at round N" and "hit the cap at round N."

**B-03: Python inline string injection via `$SKILL_ID`**
In Step 2, `$SKILL_ID` is interpolated directly into a Python `-c` string (`if s['id'] == '$SKILL_ID'`). A skill ID containing a single quote (e.g., `foo'bar`) would break the Python syntax. Same issue in Step 5. These should use environment variables read inside Python (`os.environ`) rather than shell interpolation.

**B-04: `$LAST_VARIATIONS` and `$LAST_SCORECARD` may be empty**
If the loop runs zero iterations (somehow `MAX_ROUNDS=0` or `seq` produces nothing), `$LAST_VARIATIONS` and `$LAST_SCORECARD` remain empty strings. All subsequent `cat "$LAST_VARIATIONS"` calls silently cat nothing, and the QU5/golden prompts get garbage context. No guard exists.

**B-05: JSON regex fails on multi-field objects**
The regex `r'\`\`\`json\s*({[^}]+})\s*\`\`\`'` uses `[^}]+` which stops at the first `}`. If the JSON block contains nested objects (e.g., `"new_patterns": [{"label": "..."}]`), the match truncates. This is also why `re.DOTALL` is on the outer match but `[^}]+` already excludes newlines inside the braces — the flag is not doing the work intended here.

**B-06: `SPEC_FILE` is assigned but never used**
`SPEC_FILE` at line 16 is set to `signal-scout-$TODAY.md` (hardcoded `scout`, not `$SKILL_ID`). It is never referenced again. Either it should be `signal-${SKILL_ID}-$TODAY.md` and actually read, or the variable should be removed.

## Design / Reliability Issues

**D-01: Truncated rubric in all subsequent prompts (`head -80`)**
The rubric is always fed truncated to 80 lines for variate, score, and evolve prompts. A rubric with many criteria would be silently cut. The scoring model is asked to evaluate against a rubric it may only see half of.

**D-02: Trace content truncated to 3000 chars in rubric, 800 chars in scoring**
The scoring prompt gets only 800 chars of trace content as ground truth. That may be less than one complete skill output example, making the ground-truth anchor nearly useless by round 2+.

**D-03: QU5 pass reads `$LAST_VARIATIONS` (all 5 variations) not the winning variation**
The QU5 simplification prompt says "winning variation" but feeds the entire variations file (head -100). Claude has to infer which variation won. The scorecard ranking is never explicitly extracted and threaded through.

**D-04: Plateau logic resets on any new pattern, even noise**
If the model hallucinates a spurious "new pattern" every other round, `PLATEAU_COUNT` resets to 0 indefinitely and the loop runs to `MAX_ROUNDS`. There is no minimum score threshold to enter convergence checking. A run can plateau-loop with score=45 forever.

**D-05: Rubric evolution only adds aspirational criteria, never removes or tightens**
The evolve prompt says "add them as new aspirational criteria." Rubric criteria that turn out to be wrong, overlapping, or never-failing are never pruned. By round 10+ the rubric can be bloated with aspirational criteria that always pass, deflating the signal of the composite score.

**D-06: `signal.skills.yaml` is rewritten with `yaml.dump` which loses formatting/comments**
The update in Step 5 does `yaml.safe_load` → mutate → `yaml.dump`. This discards all inline comments, custom ordering, and any block-style formatting in the original YAML. On next run, the file looks nothing like the hand-authored version.

## Minor Issues

**M-01: `ls` used instead of `glob`/`find` for trace file discovery**
`ls "$TRACE_DIR"/plugin-${SKILL_ID}-*.md 2>/dev/null | head -1` is fine functionally, but if filenames contain spaces or special characters the pipeline breaks silently.

**M-02: `mktemp` used in QU5 but explicitly avoided in `claude_ask`**
The header comment for `claude_ask` says `mktemp may return Windows paths in relay context`, then QU5 (line 250) uses `mktemp /tmp/qu5-prompt-XXXXXX.txt` directly. The workaround is applied inconsistently.

**M-03: `cat "$FILE" | head -N` is useless use of cat**
Should be `head -N "$FILE"` throughout. Minor style issue but consistent across the file.

**M-04: No check that `claude` binary is available**
The script fails with an opaque error if `claude` is not on `$PATH`. A preflight check at the top would give a clear error message.

**M-05: `RUBRIC_CONTENT` and `VARS_CONTENT` assigned but unused**
Lines 251–252 assign `VARS_CONTENT` and `RUBRIC_CONTENT` via shell, but the QU5 Python block reads the files directly via `sys.argv`. The shell variables are dead code.
