## Competitive Brief: Signal Plugin

**Strategic verdict**: Signal has no direct competitor. It occupies an uncontested lane — multi-namespace evidence accumulation before spec authorship. The spec-driven development wave is the pressure to watch: the window to define this category is 12-18 months.

---

### The Primary Competitor: Inertia — THREAT: HIGH

The real competition is "do nothing." Teams have no visible cost for building the wrong thing. The cost is invisible at commit time and appears at retrospective time, 6-12 weeks later. Signal's `--mode risk` framing is the primary lever to break this.

What makes inertia stick: successful teams won't adopt. GTM entry point is post-incident recovery, not greenfield.

---

### Key Findings

**F-01** — Inertia is the primary competitor. GTM is post-incident, not greenfield.

**F-02** — GitHub Spec Kit is the highest-named threat. 72.7k stars, official Claude Code support. Gap today: scaffold vs. evidence accumulator. The gap is closeable. (Source: augmentcode.com/tools/best-spec-driven-development-tools)

**F-03** — feature-dev skill (89k installs) is the in-ecosystem benchmark. If it adds a pre-build evidence phase, it wins on install base alone.

**F-04** — **Naming risk**: "Feature decision tools" is LaunchDarkly's mental model territory. Signal must use "pre-commit evidence" language, not "feature decision" language.

**F-05** — **Blue ocean: `/prove:` namespace.** No tool runs structured pre-build hypothesis testing for feature decisions. Zero competition. Strongest PM differentiation story.

**F-06** — Signal passes all table stakes (native, repo artifacts, auto-detect, fast first output, non-disruptive). No named competitor currently beats all five.

---

### --mode risk (exec framing)

A 2-engineer, 3-sprint feature costs ~$60-90k loaded. Industry rework cost is 2-5x when assumptions aren't validated before build. Signal is pre-build insurance: 2-4 hour premium, claim event = any missed feature. Teams that have filed that claim know the math.
