## Competitive Brief: Signal Plugin

### 1. The Primary Competitor: Inertia (HIGH)

The stickiest threat isn't any named tool -- it's the comfort of "we already know what to build." Inertia beats Signal when the feature feels small, the deadline is tight, or the PM has a spec. The counter: those four conditions are exactly when pre-build evidence matters most. Signal's job is to make the cost of skipping visible before the architecture locks.

---

### Finding 1 - The Whitespace Is Vacant

No existing tool owns all three simultaneously: **pre-build** + **repo-committed artifacts** + **multi-namespace**. The Claude Code ecosystem has 120+ plugins, every one execution-focused (write, review, test, deploy). Signal would be the first research-class plugin. Verified: composio.dev, ccplugins/awesome-claude-code-plugins.

### Finding 2 - AWS Kiro Is the Closest Threat (MEDIUM-HIGH)

Kiro launched mid-2025 as a VS Code fork with a spec-driven phase (requirements.md + design.md + task checklist). Covers ~3 of Signal's 9 namespaces. Key moat: Signal's artifacts commit into your repo with full git history; Kiro's spec is Kiro-managed. Kiro is also an IDE (hard distribution switch), not a plugin drop-in. Source: kiro.dev.

### Finding 3 - Linear AI Is Downstream, Not Competitive (MEDIUM)

Linear's 2026 MCP integration brings it into the Claude Code orbit. Its Project Intelligence assesses incoming work and makes build suggestions -- which sounds like Signal's topic namespace. But Linear manages what to build after the decision is made. Signal gathers evidence before the decision. They occupy adjacent phases of the same pipeline, which opens an integration story rather than a fight. Source: linear.app/ai, eesel.ai/blog/linear-ai.

### Finding 4 - Feature Flags Are Post-Commit (Adjacent)

LaunchDarkly and Split.io (now Harness) answer "should we release?" Signal answers "should we build?" They are adjacent in the decision chain, not competing. Signal sits at the earliest decision point; flags sit at the latest before production. This is a clean positioning wedge: Signal is what you run *before* there's a flag to flip. Sources: launchdarkly.com/pricing, harness.io.

### Finding 5 - Repo-Committed Artifacts Are Architecturally Distinctive

Notion, Pieces, Swimm, and Windsurf all store knowledge outside the repo (SaaS silo, session memory, CI dashboard). Signal's `{topic}-{signal}-{date}.md` model commits evidence alongside code -- versioned, diffable, reviewable in PRs, team-owned not vendor-owned. No competitor owns this model. This is both the table stake and the deepest moat.

### Finding 6 - The CC Plugin Ecosystem Has No Comparable Tool

Superpowers (the largest CC lifecycle plugin) covers drafting, review, and flow but is a general productivity tool, not a feature-decision product. AB Method handles spec workflows in one namespace. Neither touches scout, prove, listen, or topic. The pre-build research space in the Claude Code ecosystem is unoccupied.
