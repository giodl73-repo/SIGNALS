# Competitive Brief: Signal Plugin

**17 findings across 10 named competitors.**

---

## Primary Competitor: Inertia (HIGH)

**F-01** — Inertia is not passivity. It is an active, socially reinforced choice backed by velocity culture, survivorship bias, and no named owner for pre-build research. Signal wins by being faster than "is this worth the time?" — not by being more comprehensive.

**F-02** — The cost of skipping is invisible at skip time. The METR study (July 2025) found developers using AI tools took 19% longer than they perceived — same failure mode. Signal must make the cost visible before it's incurred.

---

## Named Competitors

**F-03 (Whitespace)** — No competitor owns all four: pre-build + repo-native artifacts + multi-namespace coverage + topic narrative synthesis. Every named tool owns at most one.

**F-04 (Table stakes)** — Zero-setup first run is the developer filter. Tools requiring configuration before producing a first artifact lose the developer before trust is established.

**F-05 — Anthropic /feature-dev (HIGH)** — First-party, bundled in the default Claude Code catalog, seen by every Claude Code user before Signal. It's a 7-phase execution workflow (requirements → codebase exploration → architecture → implementation → testing → review → documentation). It starts *after* the build decision is made. Signal must position as "pre-/feature-dev" — the tool that feeds /feature-dev with the right inputs — or risk being bypassed by users who think the problem is already solved.

**F-06 — GitHub Copilot Workspace (HIGH)** — Generates a Specification and Plan before code runs, across 100M+ GitHub developer accounts. The Specification is a build spec — "how to implement" — not a decision brief. No competitive analysis, no failure-mode simulation, no persona evaluation. The distribution moat is the threat.

**F-07 — Cursor Plan Mode (MEDIUM)** — Composer 2.0 (Cursor 2.5, Feb 2026) produces editable repo-native Markdown plans — the closest structural overlap with Signal's `/draft:spec`. Single-axis: it asks "how should we build this?" never "should we?" Differentiation is multi-axis depth.

**F-08 — Devin / Cognition (LOW)** — Integration target, not threat. Signal upstream, Devin execution. Risk: if Devin adds a research harness before Signal ships, the integration opportunity closes.

**F-09 — Linear AI (LOW-MEDIUM)** — "From drafting PRDs to pushing PRs" — the closest named workflow to Signal's scout-to-draft pipeline. Watch for a `/research` namespace addition. MCP server integration with Cursor and Claude means it's moving toward Signal's handoff point.

**F-10 — Notion AI (MEDIUM)** — Overlaps with `/prove:intelligence` and `/draft:proposal`. The structural moat: Notion artifacts live in Notion, outside git, invisible to the team two years from now. Signal artifacts are committed to the repo alongside the code they informed.

**F-11 — Atlassian Rovo / Confluence (LOW-MEDIUM)** — PRD-to-ticket pipeline formalized. Covers `/draft:` overlap only. The entire Signal namespace surface outside `/draft:` — competitive intelligence, system simulation, customer reaction, compliance — is uncovered.

**F-12 — LaunchDarkly (LOW)** — Operates in the post-build decision window. Zero overlap on the research problem. Most powerful in sequence: Signal before the build decision, LaunchDarkly after.

**F-13 — Roo Code / Cline (LOW-MEDIUM)** — Same developer footprint, different job. Risk is roadmap convergence, not current overlap. As Plan Mode matures, "should we?" is the natural next question any agentic coding tool gets asked.

**F-14 — Raw LLM prompting (MEDIUM)** — Zero friction, already installed, no learning curve. A direct prompt produces one answer once. Signal produces accumulated, repo-persistent, topic-threaded evidence. The qualitative difference is invisible until Signal has been used at least once. The first-run demo must surface a gap a raw prompt misses.

---

## Risk Frame (exec)

**F-15** — Signal's strongest exec argument is the decision audit trail. In regulated industries, knowing *why* a feature was built — with traceable evidence — is a compliance requirement, not a nice-to-have. `/prove:publish` and `/scout:compliance` own this directly.

---

## Six-Month Watch

**F-16** — The watch trigger across all HIGH-threat competitors is identical: does any competitor add a pre-build "should we?" stage that produces structured, durable artifacts? /feature-dev gaining a `/scout:` equivalent is the single move that compresses Signal's whitespace most directly.

**F-17** — Distribution is the highest strategic risk. /feature-dev is bundled. Copilot Workspace ships with GitHub. Signal must earn the "install first" slot with a cold-start demo that makes the whitespace visible in under five minutes. That window narrows every month.
