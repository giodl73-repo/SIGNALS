## Competitive Brief: Signal Plugin — Summary

### The Primary Competitor: Inertia (always HIGH)

Teams choose nothing because the cost of skipping pre-build research is invisible until a feature fails. The ad hoc process (Slack + Claude.ai chat + gut feel) already exists, produces no artifacts, and feels sufficient. Signal competes with the dopamine of shipping.

### The Whitespace

**No competitor owns the territory between "I have an idea" and "I have a spec worth trusting."**

- Coding tools (Copilot, Cursor, Gemini CLI) own implementation — not pre-build evidence
- Spec-driven IDEs (Kiro) start *after* a build decision — Signal starts *before*
- PM tools (Productboard) own roadmap — not developer terminal workflow
- General AI chat owns informal research — artifacts evaporate at session end

### Named Competitors

| Competitor | Threat | Why |
|---|---|---|
| AWS Kiro | MEDIUM-HIGH | Closest structural competitor — spec-driven, but starts post-decision |
| General AI chat (Claude.ai, ChatGPT) | MEDIUM | The "good enough" substitute — zero friction, zero artifacts |
| GitHub Copilot | MEDIUM | 20M+ users, could enter this space with one product decision |
| Cursor | MEDIUM | Dominating agentic IDE; no namespace/artifact model for pre-build |
| Gemini CLI | LOW-MEDIUM | Free tier, Google Search, 1M context — no structured evidence system |
| Devin/Windsurf | LOW-MEDIUM | Enterprise execution platform, not pre-build evidence |
| Productboard | LOW | PM tool — different audience entirely |

### The Risk Frame (for execs)

$29.5B wasted annually on features rarely or never used. Signal is not a research tool — it is a risk-reduction system. Chat logs don't constitute evidence. Signal artifacts do.
